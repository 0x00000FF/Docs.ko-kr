# <a name="stackalloc-c-reference"></a><span data-ttu-id="e4188-102">stackalloc(C# 참조)</span><span class="sxs-lookup"><span data-stu-id="e4188-102">stackalloc (C# Reference)</span></span>
<span data-ttu-id="e4188-103">`stackalloc` 키워드는 스택에 메모리 블록을 할당하기 위해 안전하지 않은 코드 컨텍스트에서 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-103">The `stackalloc` keyword is used in an unsafe code context to allocate a block of memory on the stack.</span></span>  
  
```  
int* block = stackalloc int[100];  
```  
  
## <a name="remarks"></a><span data-ttu-id="e4188-104">주의</span><span class="sxs-lookup"><span data-stu-id="e4188-104">Remarks</span></span>  
 <span data-ttu-id="e4188-105">이 키워드는 지역 변수 이니셜라이저에서만 유효합니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-105">The keyword is valid only in local variable initializers.</span></span> <span data-ttu-id="e4188-106">다음 코드를 실행하면 컴파일러 오류가 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-106">The following code causes compiler errors.</span></span>  
  
```  
int* block;  
// The following assignment statement causes compiler errors. You  
// can use stackalloc only when declaring and initializing a local   
// variable.  
block = stackalloc int[100];  
```  
  
 <span data-ttu-id="e4188-107">포인터 형식이 사용되므로 `stackalloc`에 [안전하지 않은](../../../csharp/language-reference/keywords/unsafe.md) 컨텍스트가 필요합니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-107">Because pointer types are involved, `stackalloc` requires [unsafe](../../../csharp/language-reference/keywords/unsafe.md) context.</span></span> <span data-ttu-id="e4188-108">자세한 내용은 [안전하지 않은 코드 및 포인터](../../../csharp/programming-guide/unsafe-code-pointers/index.md)를 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="e4188-108">For more information, see [Unsafe Code and Pointers](../../../csharp/programming-guide/unsafe-code-pointers/index.md).</span></span>  
  
 <span data-ttu-id="e4188-109">`stackalloc`는 C 런타임 라이브러리의 [_alloca](/cpp/c-runtime-library/reference/alloca)와 유사합니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-109">`stackalloc` is like [_alloca](/cpp/c-runtime-library/reference/alloca) in the C run-time library.</span></span>  
  
 <span data-ttu-id="e4188-110">다음 예제에서는 피보나치 시퀀스의 처음 20개 숫자를 계산하고 표시합니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-110">The following example calculates and displays the first 20 numbers in the Fibonacci sequence.</span></span> <span data-ttu-id="e4188-111">각 숫자는 이전 두 숫자의 합계입니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-111">Each number is the sum of the previous two numbers.</span></span> <span data-ttu-id="e4188-112">코드에서 `int` 형식의 요소 20개를 포함하기에 충분한 크기의 메모리 블록이 힙이 아니라 스택에 할당됩니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-112">In the code, a block of memory of sufficient size to contain 20 elements of type `int` is allocated on the stack, not the heap.</span></span> <span data-ttu-id="e4188-113">블록의 주소는 `fib` 포인터에 저장됩니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-113">The address of the block is stored in the pointer `fib`.</span></span> <span data-ttu-id="e4188-114">이 메모리에는 가비지 수집이 적용되지 않으므로 [fixed](../../../csharp/language-reference/keywords/fixed-statement.md)를 사용하여 고정할 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-114">This memory is not subject to garbage collection and therefore does not have to be pinned (by using [fixed](../../../csharp/language-reference/keywords/fixed-statement.md)).</span></span> <span data-ttu-id="e4188-115">메모리 블록의 수명은 이 수명을 정의하는 메서드의 수명으로 제한됩니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-115">The lifetime of the memory block is limited to the lifetime of the method that defines it.</span></span> <span data-ttu-id="e4188-116">메서드가 반환되기 전에는 메모리를 해제할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="e4188-116">You cannot free the memory before the method returns.</span></span>
