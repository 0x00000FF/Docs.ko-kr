# <a name="using-static-directive-c-reference"></a><span data-ttu-id="8f9e9-102">using 정적 지시문(C# 참조)</span><span class="sxs-lookup"><span data-stu-id="8f9e9-102">using static Directive (C# Reference)</span></span>

<span data-ttu-id="8f9e9-103">`using static` 지시문은 형식 이름을 지정하지 않고 정적 멤버에 액세스할 수 있는 형식을 지정합니다.</span><span class="sxs-lookup"><span data-stu-id="8f9e9-103">The `using static` directive designates a type whose static members you can access without specifying a type name.</span></span> <span data-ttu-id="8f9e9-104">사용되는 구문은 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="8f9e9-104">Its syntax is:</span></span>

```csharp
using static <fully-qualified-type-name>
```

<span data-ttu-id="8f9e9-105">여기서 *fully-qualified-type-name*은 형식 이름을 지정하지 않고 정적 멤버를 참조할 수 있는 형식의 이름입니다.</span><span class="sxs-lookup"><span data-stu-id="8f9e9-105">where *fully-qualified-type-name* is the name of the type whose static members can be referenced without specifying a type name.</span></span> <span data-ttu-id="8f9e9-106">정규화된 형식 이름(전체 네임스페이스 및 형식 이름)을 제공하지 않으면 C#은 컴파일러 오류 CS0246, "'<type-name>' 형식 또는 네임스페이스 이름을 찾을 수 없습니다."를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="8f9e9-106">If you do not provide a fully qualified type name (the full namespace name along with the type name), C# generates compiler error CS0246: "The type or namespace name '<type-name>' could not be found."</span></span>

<span data-ttu-id="8f9e9-107">`using static` 지시문은 정적 멤버가 있는 모든 형식에 적용됩니다(인스턴스 멤버가 있는 경우에도).</span><span class="sxs-lookup"><span data-stu-id="8f9e9-107">The `using static` directive applies to any type that has static members, even if it also has instance members.</span></span> <span data-ttu-id="8f9e9-108">그러나 인스턴스 멤버는 형식 인스턴스를 통해서만 호출할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="8f9e9-108">However, instance members can only be invoked through the type instance.</span></span>

<span data-ttu-id="8f9e9-109">`using static` 지시문은 C# 6에서 도입되었습니다.</span><span class="sxs-lookup"><span data-stu-id="8f9e9-109">The `using static` directive was introduced in C# 6.</span></span>

 <span data-ttu-id="8f9e9-121">같은 컴파일 단위 또는 네임스페이스에서 여러 `using static` 지시문을 통해 다양한 형식에서 가져온 같은 이름을 사용하는 메서드는 메서드 그룹을 구성합니다.</span><span class="sxs-lookup"><span data-stu-id="8f9e9-121">Methods with the same name imported from different types by different `using static` directives in the same compilation unit or namespace form a method group.</span></span>  <span data-ttu-id="8f9e9-122">이들 메서드 그룹 내에서 오버로드 확인은 일반 C# 규칙을 따릅니다.</span><span class="sxs-lookup"><span data-stu-id="8f9e9-122">Overload resolution within these method groups follows normal C# rules.</span></span>
