# <a name="targetwinmdobj-c-compiler-options"></a><span data-ttu-id="0c9b5-102">/target:winmdobj(C# 컴파일러 옵션)</span><span class="sxs-lookup"><span data-stu-id="0c9b5-102">/target:winmdobj (C# Compiler Options)</span></span>
<span data-ttu-id="0c9b5-103">**/target:winmdobj** 컴파일러 옵션을 사용하는 경우 컴파일러는 사용자가 Windows 런타임 이진(.winmd) 파일로 변환할 수 있는 중간 .winmdobj 파일을 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-103">If you use the **/target:winmdobj** compiler option, the compiler creates an intermediate .winmdobj file that you can convert to a Windows Runtime binary (.winmd) file.</span></span> <span data-ttu-id="0c9b5-104">그런 다음 관리되는 언어 프로그램뿐만 아니라 JavaScript 및 C++ 프로그램에서도 .winmd 파일을 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-104">The .winmd file can then be consumed by JavaScript and C++ programs, in addition to managed language programs.</span></span>  
  
## <a name="syntax"></a><span data-ttu-id="0c9b5-105">구문</span><span class="sxs-lookup"><span data-stu-id="0c9b5-105">Syntax</span></span>  
  
```console  
/target:winmdobj  
```  
  
## <a name="remarks"></a><span data-ttu-id="0c9b5-106">설명</span><span class="sxs-lookup"><span data-stu-id="0c9b5-106">Remarks</span></span>  
 <span data-ttu-id="0c9b5-107">**winmdobj** 설정이 컴파일러에 중간 모듈이 필요하다는 신호를 보냅니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-107">The **winmdobj** setting signals to the compiler that an intermediate module is required.</span></span> <span data-ttu-id="0c9b5-108">이에 대한 응답으로 Visual Studio에서 C# 클래스 라이브러리를 .winmdobj 파일로 컴파일합니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-108">In response, Visual Studio compiles the C# class library as a .winmdobj file.</span></span> <span data-ttu-id="0c9b5-109">그런 다음 <xref:Microsoft.Build.Tasks.WinMDExp> 내보내기 도구를 통해 .winmdobj 파일을 공급하여 Windows 메타데이터(.winmd) 파일을 만들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-109">The .winmdobj file can then be fed through the <xref:Microsoft.Build.Tasks.WinMDExp> export tool to produce a Windows metadata (.winmd) file.</span></span> <span data-ttu-id="0c9b5-110">.winmd 파일에는 원본 라이브러리의 코드와 JavaScript 또는 C++ 및 Windows 런타임에서 사용하는 WinMD 메타데이터가 모두 들어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-110">The .winmd file contains both the code from the original library and the WinMD metadata that is used by JavaScript or C++ and by the Windows Runtime.</span></span>  
  
 <span data-ttu-id="0c9b5-111">**/target:winmdobj** 컴파일러 옵션을 사용하여 컴파일된 파일의 출력은 WimMDExp 내보내기 도구의 입력으로만 사용하도록 설계되었습니다. .winmdobj 파일 자체는 직접 참조되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-111">The output of a file that’s compiled by using the **/target:winmdobj** compiler option is designed to be used only as input for the WimMDExp export tool; the .winmdobj file itself isn’t referenced directly.</span></span>  
  
 <span data-ttu-id="0c9b5-112">[/out](../../../csharp/language-reference/compiler-options/out-compiler-option.md) 옵션을 사용하지 않으면 첫 번째 입력 파일의 이름이 출력 파일의 이름으로 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-112">Unless you use the [/out](../../../csharp/language-reference/compiler-options/out-compiler-option.md) option, the output file name takes the name of the first input file.</span></span> <span data-ttu-id="0c9b5-113">[Main](../../../csharp/programming-guide/main-and-command-args/index.md) 메서드는 필요하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-113">A [Main](../../../csharp/programming-guide/main-and-command-args/index.md) method isn’t required.</span></span>  
  
 <span data-ttu-id="0c9b5-114">명령 프롬프트에서 /target:winmdobj 옵션을 지정하면 Windows 프로그램을 만드는 데 다음 **/out** 또는 [/target:module](../../../csharp/language-reference/compiler-options/target-module-compiler-option.md) 옵션까지의 모든 파일이 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-114">If you specify the /target:winmdobj option at a command prompt, all files until the next **/out** or [/target:module](../../../csharp/language-reference/compiler-options/target-module-compiler-option.md) option are used to create the Windows program.</span></span>

2.  <span data-ttu-id="0c9b5-117">**응용 프로그램** 탭을 선택합니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-117">Choose the **Application** tab.</span></span>  
  
3.  <span data-ttu-id="0c9b5-118">**출력 형식** 목록에서 **WinMD 파일**을 선택합니다.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-118">In the **Output type** list, choose **WinMD File**.</span></span>

 <span data-ttu-id="0c9b5-120">이 컴파일러 옵션을 프로그래밍 방식으로 설정하는 방법에 대한 자세한 내용은 <xref:VSLangProj80.ProjectProperties3.OutputType%2A>을 참조하십시오.</span><span class="sxs-lookup"><span data-stu-id="0c9b5-120">For information about how to set this compiler option programmatically, see <xref:VSLangProj80.ProjectProperties3.OutputType%2A>.</span></span>
