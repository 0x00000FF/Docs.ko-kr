<span data-ttu-id="132d9-103">안전하지 않은 형식 'type name'은 개체를 만드는 데 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="132d9-103">Unsafe type 'type name' cannot be used in object creation.</span></span>  
  
 <span data-ttu-id="132d9-104">`new` 연산자는 관리되는 힙에서만 개체를 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="132d9-104">The `new` operator creates objects only on the managed heap.</span></span> <span data-ttu-id="132d9-105">그러나 언어의 상호 운용성 기능을 사용하여 포인터를 반환하는 네이티브 메서드를 호출하는 방식으로 간접적으로 관리되지 않는 메모리에서 개체를 만들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="132d9-105">However, you can create objects in unmanaged memory indirectly by using the interoperability capabilities of the language to call native methods that return pointers.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="132d9-106">이 오류를 해결하려면</span><span class="sxs-lookup"><span data-stu-id="132d9-106">To correct this error</span></span>  
  
1.  <span data-ttu-id="132d9-107">새 개체 만들기 식에서 안전한 형식을 사용하세요.</span><span class="sxs-lookup"><span data-stu-id="132d9-107">Use a safe type in the new object creation expression.</span></span> <span data-ttu-id="132d9-108">예를 들어 `char*` 또는 `int*` 대신 `char` 또는 `int`를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="132d9-108">For example, use `char` or `int` instead of `char*` or `int*`.</span></span>  
  
2.  <span data-ttu-id="132d9-109">관리되지 않는 메모리에서 개체를 만들어야 할 경우 Win32 또는 COM 메서드를 사용하거나 C 또는 C++에서 자체 함수를 작성하고 C#에서 호출합니다.</span><span class="sxs-lookup"><span data-stu-id="132d9-109">If you must create objects in unmanaged memory, use a Win32 or COM method or else write your own function in C or C++ and call it from C#.</span></span>  
  
## <a name="example"></a><span data-ttu-id="132d9-110">예제</span><span class="sxs-lookup"><span data-stu-id="132d9-110">Example</span></span>
