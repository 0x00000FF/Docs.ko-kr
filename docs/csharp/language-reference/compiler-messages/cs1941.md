## <a name="example"></a><span data-ttu-id="07d04-108">예제</span><span class="sxs-lookup"><span data-stu-id="07d04-108">Example</span></span>  
 <span data-ttu-id="07d04-109">다음 코드에서는 `equals` 연산자에 `int`와 `string`의 비교를 요청 중이므로 CS1941 오류가 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="07d04-109">The following code generates CS1941 because the `equals` operator is being asked to compare an `int` to a `string`.</span></span>  
  
```csharp  
// cs1941.cs  
using System.Collections;  
using System.Linq;  
class Test  
{  
    static int Main()  
    {  
        var nums = new[] { 1, 2, 3, 4, 5, 6 };  
        var words = new string[] { "lake", "mountain", "sky" };  
        IEnumerable e = from n in nums  
                        join w in words on n equals w // CS1941  
                        select w;  
        return 0;  
    }  
}  
```  
  
 <span data-ttu-id="07d04-110">형식 유추에 실패한 메서드는 컴파일 시간에 쿼리 절이 변환되는 메서드입니다.</span><span class="sxs-lookup"><span data-stu-id="07d04-110">The method in which type inference fails is the method that the query clause is translated to at compile time.</span></span>
