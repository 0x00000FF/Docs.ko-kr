# <a name="compiler-warning-level-2-cs0467"></a><span data-ttu-id="9d15f-102">컴파일러 경고(수준 2) CS0467</span><span class="sxs-lookup"><span data-stu-id="9d15f-102">Compiler Warning (level 2) CS0467</span></span>
<span data-ttu-id="9d15f-103">메서드 ‘method’와 메서드가 아닌 ‘non-method’ 사이에 모호성이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9d15f-103">Ambiguity between method 'method' and non-method 'non-method'.</span></span> <span data-ttu-id="9d15f-104">메서드 그룹을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="9d15f-104">Using method group.</span></span>

## <a name="example"></a><span data-ttu-id="9d15f-106">예제</span><span class="sxs-lookup"><span data-stu-id="9d15f-106">Example</span></span>  
 <span data-ttu-id="9d15f-107">다음 예제에서는 CS0467을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="9d15f-107">The following example generates CS0467.</span></span>  
  
```csharp  
// CS0467.cs  
interface IList
{  
    int Count { get; set; }  
}  
  
interface ICounter  
{  
    void Count(int i);  
}  
  
interface IListCounter : IList, ICounter {}  
  
class Driver
{  
    void Test(IListCounter x)  
    {  
        // The following line causes the warning. The assignment also  
        // causes an error because you can't assign a value to a method.  
        x.Count = 1;  
        x.Count(3);
        // To resolve the warning, you can change the name of the method or
        // the property.  
  
        // You also can disambiguate by specifying IList or ICounter.  
        ((IList)x).Count = 1;  
        ((ICounter)x).Count(3);  
    }  
  
    static void Main()
    {  
    }  
}  
```
