<span data-ttu-id="e317a-105">이 오류는 일반적으로 범위 변수에 잘못된 명시적 형식을 지정하는 경우에 생성됩니다.</span><span class="sxs-lookup"><span data-stu-id="e317a-105">This error is typically generated when the range variable has been given an incorrect explicit type.</span></span>  

## <a name="to-correct-this-error"></a><span data-ttu-id="e317a-106">이 오류를 해결하려면</span><span class="sxs-lookup"><span data-stu-id="e317a-106">To correct this error</span></span>

1. <span data-ttu-id="e317a-107">범위 변수가 명시적으로 형식화된 경우, 해당 형식이 반복 컬렉션 요소의 형식과 동일하거나 이 형식으로부터 암시적으로 변환 가능하도록 합니다.</span><span class="sxs-lookup"><span data-stu-id="e317a-107">If the range variable is explicitly typed, make sure that the type is either the same as, or implicitly convertible from, the type of the elements in the collection it iterates.</span></span> <span data-ttu-id="e317a-108">범위 변수 앞에 `var` 키워드가 있는 경우 `var`를 제거합니다.</span><span class="sxs-lookup"><span data-stu-id="e317a-108">If the range variable is preceded with the `var` keyword, remove `var`.</span></span>  
  
## <a name="example"></a><span data-ttu-id="e317a-109">예제</span><span class="sxs-lookup"><span data-stu-id="e317a-109">Example</span></span>

<span data-ttu-id="e317a-110">다음 코드에서는 CS1942 오류가 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="e317a-110">The following code generates CS1942:</span></span>  

```csharp
// cs1942.cs  
class Program  
    {  
        static void Main(string[] args)  
        {  
            var x = from var i in Enumerable.Range(1, 100) // CS1949  
                    select i; //CS1942  
        }  
    }  
```

<span data-ttu-id="e317a-111">CS1942는 CS1949와 관련이 있습니다. 범위 변수와 `var`를 함께 사용하면 `Cast<T>`가 형식이 아니므로 기본 `var` 작업이 실패하기 때문입니다.</span><span class="sxs-lookup"><span data-stu-id="e317a-111">CS1942 is related to CS1949 because the use of `var` with a range variable causes the underlying `Cast<T>` operation to fail because `var` is not a type.</span></span>
