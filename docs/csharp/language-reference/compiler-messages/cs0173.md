# <a name="compiler-error-cs0173"></a><span data-ttu-id="dd267-102">컴파일러 오류 CS0173</span><span class="sxs-lookup"><span data-stu-id="dd267-102">Compiler Error CS0173</span></span>

<span data-ttu-id="dd267-103">'class1'과(와) 'class2' 사이에 암시적 변환이 없으므로 조건식의 형식을 확인할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="dd267-103">Type of conditional expression cannot be determined because there is no implicit conversion between 'class1' and 'class2'</span></span>

<span data-ttu-id="dd267-104">클래스 간의 변환은 서로 다른 클래스의 개체에서 동일한 코드로 작업하려는 경우에 유용합니다.</span><span class="sxs-lookup"><span data-stu-id="dd267-104">Conversions between classes are useful when you want objects of different classes to work with the same code.</span></span> <span data-ttu-id="dd267-105">그러나 함께 작동하는 두 클래스는 상호 중복 변환이나 암시적 변환을 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="dd267-105">However, two classes that work together cannot have mutual and redundant conversions, or no implicit conversions.</span></span> <span data-ttu-id="dd267-106">`class1` 및 `class2`의 형식은 독립적으로 결정되며, 보다 일반적인 형식이 조건식의 형식으로 선택됩니다.</span><span class="sxs-lookup"><span data-stu-id="dd267-106">The types of `class1` and `class2` are determined independently, and the more general type is selected as the type of the conditional expression.</span></span> <span data-ttu-id="dd267-107">형식이 결정되는 방식에 대한 자세한 내용은 [조건부 연산자가 암시적으로 캐스팅할 수 없음](https://stackoverflow.com/questions/2215745/conditional-operator-cannot-cast-implicitly/2215959#2215959)을 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="dd267-107">For more information about how types are determined, see [Conditional operator cannot cast implicitly](https://stackoverflow.com/questions/2215745/conditional-operator-cannot-cast-implicitly/2215959#2215959).</span></span>

## <a name="example"></a><span data-ttu-id="dd267-110">예</span><span class="sxs-lookup"><span data-stu-id="dd267-110">Example</span></span>

<span data-ttu-id="dd267-111">다음 예제는 컴파일러 오류 CS0173을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="dd267-111">The following examples generate compiler error CS0173:</span></span>

```csharp
public class C {}

public class A
{
    // The following code defines an implicit conversion operator from
    // type C to type A.
    //public static implicit operator A(C c)
    //{
    //    A a = new A();
    //    a = c;
    //    return a;
    //}
}

public class MyClass
{
    public static void F(bool b)
    {
        A a = new A();
        C c = new C();

        // The following line causes CS0173 because there is no implicit
        // conversion from a to c or from c to a.
        object o = b ? a : c;

        // To resolve the error, you can cast a and c.
        // object o = b ? (object)a : (object)c;

        // Alternatively, you can add a conversion operator from class C to
        // class A, or from class A to class C, but not both.
    }

   public static void Main()
   {
      F(true);
   }
}
```

```csharp
class M
{
    static int Main ()
    {
        int X = 1;
        // The following line causes CS0173.
        object o = (X == 0) ? null : null;
        return -1;
    }
}
```
