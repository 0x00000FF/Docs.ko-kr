# <a name="compiler-error-cs0173"></a><span data-ttu-id="a955c-102">컴파일러 오류 CS0173</span><span class="sxs-lookup"><span data-stu-id="a955c-102">Compiler Error CS0173</span></span>

<span data-ttu-id="a955c-103">'class1'과(와) 'class2' 사이에 암시적 변환이 없으므로 조건식의 형식을 확인할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="a955c-103">Type of conditional expression cannot be determined because there is no implicit conversion between 'class1' and 'class2'</span></span>

## <a name="example"></a><span data-ttu-id="a955c-110">예</span><span class="sxs-lookup"><span data-stu-id="a955c-110">Example</span></span>

<span data-ttu-id="a955c-111">다음 예제는 컴파일러 오류 CS0173을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="a955c-111">The following examples generate compiler error CS0173:</span></span>

```csharp
public class C {}

public class A
{
    // The following code defines an implicit conversion operator from
    // type C to type A.
    //public static implicit operator A(C c)
    //{
    //    A a = new A();
    //    a = c;
    //    return a;
    //}
}

public class MyClass
{
    public static void F(bool b)
    {
        A a = new A();
        C c = new C();

        // The following line causes CS0173 because there is no implicit
        // conversion from a to c or from c to a.
        object o = b ? a : c;

        // To resolve the error, you can cast a and c.
        // object o = b ? (object)a : (object)c;

        // Alternatively, you can add a conversion operator from class C to
        // class A, or from class A to class C, but not both.
    }

   public static void Main()
   {
      F(true);
   }
}
```

```csharp
class M
{
    static int Main ()
    {
        int X = 1;
        // The following line causes CS0173.
        object o = (X == 0) ? null : null;
        return -1;
    }
}
```
