# <a name="compiler-error-cs0269"></a><span data-ttu-id="1eae1-102">컴파일러 오류 CS0269</span><span class="sxs-lookup"><span data-stu-id="1eae1-102">Compiler Error CS0269</span></span>
<span data-ttu-id="1eae1-103">할당되지 않은 'parameter' out 매개 변수를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="1eae1-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="1eae1-104">컴파일러는 사용되기 전에 out 매개 변수에 값이 할당되었는지 확인할 수 없습니다. 할당 시 해당 값이 정의되어 있지 않을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="1eae1-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="1eae1-105">값에 액세스하기 전에 호출된 메서드의 `out` 매개 변수에 값을 할당해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="1eae1-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="1eae1-106">전달된 변수의 값을 사용해야 하는 경우 `ref` 매개 변수를 대신 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="1eae1-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="1eae1-107">자세한 내용은 [매개 변수 전달](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md)을 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="1eae1-107">For more information, see [Passing Parameters](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="1eae1-108">예제</span><span class="sxs-lookup"><span data-stu-id="1eae1-108">Example</span></span>  
 <span data-ttu-id="1eae1-109">다음 샘플에서는 CS0269를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="1eae1-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="1eae1-110">예제</span><span class="sxs-lookup"><span data-stu-id="1eae1-110">Example</span></span>  
 <span data-ttu-id="1eae1-111">이 오류는 컴파일러가 성공적으로 실행될지 확인할 수 없는 try 블록에서 변수를 초기화하는 경우에도 발생할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="1eae1-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
