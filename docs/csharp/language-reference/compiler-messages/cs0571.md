<span data-ttu-id="1c336-103">‘function’: 연산자나 접근자를 명시적으로 호출할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="1c336-103">'function' : cannot explicitly call operator or accessor</span></span>  
  
 <span data-ttu-id="1c336-104">특정 연산자에는 내부 이름이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="1c336-104">Certain operators have internal names.</span></span> <span data-ttu-id="1c336-105">예를 들어 **op_Increment**는 ++ 연산자의 내부 이름입니다.</span><span class="sxs-lookup"><span data-stu-id="1c336-105">For example, **op_Increment** is the internal name of the ++ operator.</span></span> <span data-ttu-id="1c336-106">이러한 메서드 이름을 사용하거나 명시적으로 호출해서는 안 됩니다.</span><span class="sxs-lookup"><span data-stu-id="1c336-106">You should not use or explicitly call such method names.</span></span>  
  
 <span data-ttu-id="1c336-107">다음 샘플에서는 CS0571을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="1c336-107">The following sample generates CS0571:</span></span>  
  
```csharp  
// CS0571.cs  
public class MyClass  
{  
   public static MyClass operator ++ (MyClass c)  
   {  
      return null;  
   }  
  
   public static int prop  
   {  
      get  
      {  
         return 1;  
      }  
      set  
      {  
      }  
   }  
  
   public static void Main()  
   {  
      op_Increment(null);   // CS0571  
      // use the increment operator as follows  
      // MyClass x = new MyClass();  
      // x++;  
  
      set_prop(1);      // CS0571  
      // try the following line instead  
      // prop = 1;  
   }  
}  
```
