# <a name="compiler-error-cs0188"></a><span data-ttu-id="92bb8-102">컴파일러 오류 CS0188</span><span class="sxs-lookup"><span data-stu-id="92bb8-102">Compiler Error CS0188</span></span>

<span data-ttu-id="92bb8-103">모든 필드에 값을 할당하기 전에는 ‘this’ 개체를 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="92bb8-103">The 'this' object cannot be used before all of its fields are assigned to</span></span>  
  
 <span data-ttu-id="92bb8-104">생성자가 `struct`의 모든 필드를 할당해야 생성자가 `struct`의 메서드를 호출할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="92bb8-104">All fields in a `struct` have to be assigned by a constructor before the constructor can call a method in the `struct`.</span></span>  
  
 <span data-ttu-id="92bb8-105">구조체 생성자에서 속성을 초기화하려고 할 때 이 오류가 나타날 경우의 해결 방법은 속성 자체를 초기화하는 대신 해당 속성의 지원 필드를 직접 초기화하는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="92bb8-105">If you see this error when trying to initialize a property in a struct constructor, the solution is to initialize the backing field of that property directly instead of the property itself.</span></span>

## <a name="example"></a><span data-ttu-id="92bb8-106">예제</span><span class="sxs-lookup"><span data-stu-id="92bb8-106">Example</span></span>

 <span data-ttu-id="92bb8-107">다음 샘플에서는 CS0188을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="92bb8-107">The following sample generates CS0188:</span></span>  

```csharp
// CS0188.cs  
// compile with: /t:library  
namespace MyNamespace  
{  
    class MyClass  
    {  
        struct S  
        {  
            public int a;  
  
            void MyMethod()  
            {  
            }  
  
            S(int i)  
            {  
                // a = i;  
                MyMethod();  // CS0188  
            }  
        }  
        public static void Main()  
        { }  
  
    }  
}  
```
