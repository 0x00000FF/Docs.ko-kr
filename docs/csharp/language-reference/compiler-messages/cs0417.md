# <a name="compiler-error-cs0417"></a><span data-ttu-id="b87f1-102">컴파일러 오류 CS0417</span><span class="sxs-lookup"><span data-stu-id="b87f1-102">Compiler Error CS0417</span></span>

<span data-ttu-id="b87f1-103">'identifier': 변수 형식의 인스턴스를 만들 때는 인수를 지정할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="b87f1-103">'identifier': cannot provide arguments when creating an instance of a variable type</span></span>  
  
 <span data-ttu-id="b87f1-104">이 오류는 형식 매개 변수에서 `new` 연산자를 호출할 때 인수를 사용하는 경우에 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="b87f1-104">This error occurs if a call to the `new` operator on a type parameter has arguments.</span></span> <span data-ttu-id="b87f1-105">알 수 없는 매개 변수 형식에서 `new` 연산자를 사용하여 호출할 수 있는 생성자는 인수가 없는 생성자뿐입니다.</span><span class="sxs-lookup"><span data-stu-id="b87f1-105">The only constructor that can be called by using the `new` operator on an unknown parameter type is a constructor that has no arguments.</span></span> <span data-ttu-id="b87f1-106">다른 생성자를 호출해야 하는 경우 클래스 형식 제약 조건이나 인터페이스 제약 조건을 사용하는 것이 좋습니다.</span><span class="sxs-lookup"><span data-stu-id="b87f1-106">If you need to call another constructor, consider using a class type constraint or interface constraint.</span></span>  
  
## <a name="example"></a><span data-ttu-id="b87f1-107">예</span><span class="sxs-lookup"><span data-stu-id="b87f1-107">Example</span></span>  

 <span data-ttu-id="b87f1-108">다음 예제에서는 CS0417을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="b87f1-108">The following example generates CS0417:</span></span>  

```csharp
// CS0417  
class ExampleClass<T> where T : new()  
{  
    // The following line causes CS0417.  
    T instance1 = new T(1);
  
    // The following line doesn't cause the error.  
    T instance2 = new T();  
}  
```
