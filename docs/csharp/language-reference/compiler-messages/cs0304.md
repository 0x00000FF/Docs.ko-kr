# <a name="compiler-error-cs0304"></a><span data-ttu-id="0b1ac-102">컴파일러 오류 CS0304</span><span class="sxs-lookup"><span data-stu-id="0b1ac-102">Compiler Error CS0304</span></span>
<span data-ttu-id="0b1ac-103">'type' 변수 형식에 new() 제약 조건이 없으므로 이 변수 형식의 인스턴스를 만들 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="0b1ac-103">Cannot create an instance of the variable type 'type' because it does not have the new() constraint</span></span>  
  
 <span data-ttu-id="0b1ac-104">제네릭 클래스를 구현하고 `new` 키워드를 사용하여 형식 매개 변수 `T`에 대해 제공되는 형식의 새 인스턴스를 만들려는 경우 다음 예제와 같이 클래스 선언에서 [new() 제약 조건](../../../csharp/language-reference/keywords/new.md)을 `T`에 적용해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="0b1ac-104">When you implement a generic class, and you want to use the `new` keyword to create a new instance of any type that is supplied for a type parameter `T`, you must apply the [new() constraint](../../../csharp/language-reference/keywords/new.md) to `T` in the class declaration, as shown in the following example.</span></span>  
  
```csharp  
class C<T> where T : new()  
```  
  
 <span data-ttu-id="0b1ac-105">`new()` 제약 조건은 `T`에 대해 제공된 구체적인 형식에 매개 변수가 없는 기본 생성자가 있는지 확인하여 형식 안전성을 적용합니다.</span><span class="sxs-lookup"><span data-stu-id="0b1ac-105">The `new()` constraint enforces type safety by guaranteeing that any concrete type that is supplied for `T` has a default, parameterless constructor.</span></span> <span data-ttu-id="0b1ac-106">`T`가 `new()` 제약 조건을 지정하지 않는 경우 클래스 본문에 `new` 연산자를 사용하여 형식 매개 변수 `T`의 인스턴스를 만들려고 하면 CS0304가 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="0b1ac-106">CS0304 occurs if you attempt to use the `new` operator in the body of the class to create an instance of type parameter `T` when `T` does not specify the `new()` constraint.</span></span> <span data-ttu-id="0b1ac-107">클라이언트 쪽에서 코드를 통해 기본 생성자가 없는 형식으로 제네릭 클래스를 인스턴스화하려고 하면 해당 코드에서 [컴파일러 오류 CS0310](../../../csharp/language-reference/compiler-messages/cs0310.md)이 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="0b1ac-107">On the client side, if code attempts to instantiate the generic class with a type that has no default constructor, that code will generate [Compiler Error CS0310](../../../csharp/language-reference/compiler-messages/cs0310.md).</span></span>  
  
 <span data-ttu-id="0b1ac-108">다음 예제에서는 CS0304를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="0b1ac-108">The following example generates CS0304.</span></span>  
  
```csharp  
// CS0304.cs  
// Compile with: /target:library.  
class C<T>  
{  
    // The following line generates CS0304.  
    T t = new T();  
}  
```  
  
 <span data-ttu-id="0b1ac-109">또한 `new` 연산자는 클래스 메서드에서 허용되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="0b1ac-109">The `new` operator also is not allowed in methods of the class.</span></span>  
  
```csharp  
// Compile with: /target:library.  
class C<T>  
{  
    public void ExampleMethod()  
    {  
        // The following line generates CS0304.  
        T t = new T();  
    }  
}  
```  
  
 <span data-ttu-id="0b1ac-110">오류를 방지하려면 다음 예제와 같이 `new()` 제약 조건을 사용하여 클래스를 선언합니다.</span><span class="sxs-lookup"><span data-stu-id="0b1ac-110">To avoid the error, declare the class by using the `new()` constraint, as shown in the following example.</span></span>  
  
```csharp  
// Compile with: /target:library.  
class C<T> where T : new()  
{  
    T t = new T();  
  
    public void ExampleMethod()  
    {  
        T t = new T();  
    }  
}  
```
