<span data-ttu-id="4926e-113">다행히도 반드시 필요하지는 않습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-113">Thankfully, this isn't necessary.</span></span> <span data-ttu-id="4926e-114">.NET Core 프레임워크에는 대리자 형식이 필요할 때마다 재사용할 수 있는 여러 가지 형식이 포함되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-114">The .NET Core framework contains several types that you can reuse whenever you need delegate types.</span></span> <span data-ttu-id="4926e-115">이러한 형식은 [제네릭](programming-guide/generics/index.md) 정의이므로 새 메서드 선언이 필요할 때 사용자 지정을 선언할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-115">These are [generic](programming-guide/generics/index.md) definitions so you can declare customizations when you need new method declarations.</span></span> 

<span data-ttu-id="4926e-116">이러한 형식 중 첫 번째는 <xref:System.Action> 형식이며 여러 가지 변형이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-116">The first of these types is the <xref:System.Action> type, and several variations:</span></span>


<span data-ttu-id="4926e-117">제네릭 형식 인수에 대한 `in` 한정자는 공변성(Covariance)에 대한 문서에서 설명합니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-117">The `in` modifier on the generic type argument is covered in the article on covariance.</span></span>

<span data-ttu-id="4926e-118">이름의 변형이 `Action` 같이 최대 16 개 사이의 인수를 포함 하는 대리자 <xref:System.Action%6016>합니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-118">There are variations of the `Action` delegate that contain up to 16 arguments such as <xref:System.Action%6016>.</span></span>
<span data-ttu-id="4926e-119">이러한 정의에서는 각 대리자 인수에 대해 서로 다른 제네릭 인수를 사용하여 최대한의 유연성을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-119">It's important that these definitions use different generic arguments for each of the delegate arguments: That gives you maximum flexibility.</span></span> <span data-ttu-id="4926e-120">메서드 인수는 같은 형식일 필요가 없지만 같은 형식일 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-120">The method arguments need not be, but may be, the same type.</span></span>

<span data-ttu-id="4926e-121">void 반환 형식을 갖는 대리자 형식에 대해 `Action` 형식 중 하나를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-121">Use one of the `Action` types for any delegate type that has a void return type.</span></span>

<span data-ttu-id="4926e-122">또한 프레임워크에는 값을 반환하는 대리자 형식에 사용할 수 있는 여러 가지 제네릭 대리자 형식이 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-122">The framework also includes several generic delegate types that you can use for delegate types that return values:</span></span>

<span data-ttu-id="4926e-123">결과 제네릭 형식 인수에 대한 `out` 한정자는 공변성(Covariance)에 대한 문서에서 설명합니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-123">The `out` modifier on the result generic type argument is covered in the article on covariance.</span></span>

<span data-ttu-id="4926e-124">이름의 변형이 `Func` 와 같은 최대 16 개의 입력된 인수와 함께 위임 <xref:System.Func%6017>합니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-124">There are variations of the `Func` delegate with up to 16 input arguments such as <xref:System.Func%6017>.</span></span>
<span data-ttu-id="4926e-125">규칙에 따라 결과의 형식은 모든 `Func` 선언에서 항상 마지막 형식 매개 변수입니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-125">The type of the result is always the last type parameter in all the `Func` declarations, by convention.</span></span>

<span data-ttu-id="4926e-126">값을 반환하는 모든 대리자 형식에 대해 `Func` 형식 중 하나를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-126">Use one of the `Func` types for any delegate type that returns a value.</span></span>

<span data-ttu-id="4926e-128">모든 `Predicate` 형식에 대해 구조적으로 동일한 `Func` 형식이 있다는 것을 알 수 있습니다. 예를 들면 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-128">You may notice that for any `Predicate` type, a structurally equivalent `Func` type exists For example:</span></span>

<span data-ttu-id="4926e-129">이러한 두 형식이 동일하다고 생각할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-129">You might think these two types are equivalent.</span></span> <span data-ttu-id="4926e-130">그러나 동일하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-130">They are not.</span></span>
<span data-ttu-id="4926e-131">이러한 두 변수는 서로 교환해서 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-131">These two variables cannot be used interchangeably.</span></span> <span data-ttu-id="4926e-132">한 형식의 변수에 다른 형식을 할당할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-132">A variable of one type cannot be assigned the other type.</span></span> <span data-ttu-id="4926e-133">C# 형식 시스템에서는 구조체가 아니라 정의된 형식의 이름을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="4926e-133">The C# type system uses the names of the defined types, not the structure.</span></span>
