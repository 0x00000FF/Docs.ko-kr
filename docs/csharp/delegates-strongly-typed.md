<span data-ttu-id="0c3b4-115">이러한 형식 중 첫 번째는 <xref:System.Action> 형식이며 여러 가지 변형이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-115">The first of these types is the <xref:System.Action> type, and several variations:</span></span>


<span data-ttu-id="0c3b4-116">제네릭 형식 인수에 대한 `in` 한정자는 공변성(Covariance)에 대한 문서에서 설명합니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-116">The `in` modifier on the generic type argument is covered in the article on covariance.</span></span>

<span data-ttu-id="0c3b4-117"><xref:System.Action%6016> 같이 최대 16개의 인수가 포함된 `Action` 대리자의 변형이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-117">There are variations of the `Action` delegate that contain up to 16 arguments such as <xref:System.Action%6016>.</span></span>
<span data-ttu-id="0c3b4-118">이러한 정의에서는 각 대리자 인수에 대해 서로 다른 제네릭 인수를 사용합니다. 그러면 최대한의 유연성이 제공됩니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-118">It's important that these definitions use different generic arguments for each of the delegate arguments: That gives you maximum flexibility.</span></span> <span data-ttu-id="0c3b4-119">메서드 인수는 같은 형식일 필요가 없지만 같은 형식일 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-119">The method arguments need not be, but may be, the same type.</span></span>

<span data-ttu-id="0c3b4-120">void 반환 형식을 갖는 대리자 형식에 대해 `Action` 형식 중 하나를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-120">Use one of the `Action` types for any delegate type that has a void return type.</span></span>

<span data-ttu-id="0c3b4-121">또한 프레임워크에는 값을 반환하는 대리자 형식에 사용할 수 있는 여러 가지 제네릭 대리자 형식이 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-121">The framework also includes several generic delegate types that you can use for delegate types that return values:</span></span>

<span data-ttu-id="0c3b4-122">결과 제네릭 형식 인수에 대한 `out` 한정자는 공변성(Covariance)에 대한 문서에서 설명합니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-122">The `out` modifier on the result generic type argument is covered in the article on covariance.</span></span>

<span data-ttu-id="0c3b4-123"><xref:System.Func%6017> 같이 최대 16개의 입력 인수가 포함된 `Func` 대리자의 변형이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-123">There are variations of the `Func` delegate with up to 16 input arguments such as <xref:System.Func%6017>.</span></span>

<span data-ttu-id="0c3b4-125">값을 반환하는 모든 대리자 형식에 대해 `Func` 형식 중 하나를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-125">Use one of the `Func` types for any delegate type that returns a value.</span></span>

<span data-ttu-id="0c3b4-127">있습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-127">type for a delegate that returns a test on a single value:</span></span>

<span data-ttu-id="0c3b4-128">모든 `Predicate` 형식에 대해 구조적으로 동일한 `Func` 형식이 있다는 것을 알 수 있습니다. 예를 들면 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-128">You may notice that for any `Predicate` type, a structurally equivalent `Func` type exists For example:</span></span>

<span data-ttu-id="0c3b4-129">이러한 두 형식이 동일하다고 생각할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-129">You might think these two types are equivalent.</span></span> <span data-ttu-id="0c3b4-130">그러나 동일하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-130">They are not.</span></span>
<span data-ttu-id="0c3b4-131">이러한 두 변수는 서로 교환해서 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-131">These two variables cannot be used interchangeably.</span></span> <span data-ttu-id="0c3b4-132">한 형식의 변수에 다른 형식을 할당할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-132">A variable of one type cannot be assigned the other type.</span></span> <span data-ttu-id="0c3b4-133">C# 형식 시스템에서는 구조체가 아니라 정의된 형식의 이름을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-133">The C# type system uses the names of the defined types, not the structure.</span></span>

<span data-ttu-id="0c3b4-138">그러면 시간이 절약되고 대리자로 작업하기 위해 만들어야 하는 새로운 형식 수가 최소화됩니다.</span><span class="sxs-lookup"><span data-stu-id="0c3b4-138">This should save time, and minimize the number of new types that you need to create in order to work with delegates.</span></span>
