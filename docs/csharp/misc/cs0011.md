## <a name="example"></a><span data-ttu-id="19ec2-108">예제</span><span class="sxs-lookup"><span data-stu-id="19ec2-108">Example</span></span>  
 <span data-ttu-id="19ec2-109">두 번째 파일은 이전 예제에서 만든 `C` 클래스에서 파생된 `B` 클래스를 정의하는 DLL을 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="19ec2-109">The second file creates a DLL that defines a class `C` that is derived from the class `B` that was created in the previous example.</span></span>

```  
// compile with: /target:library /reference:CS0011_1.dll  
// post-build command: del /f CS0011_1.dll

```  
  
## <a name="example"></a><span data-ttu-id="19ec2-110">예제</span><span class="sxs-lookup"><span data-stu-id="19ec2-110">Example</span></span>  
 <span data-ttu-id="19ec2-111">세 번째 파일은 첫 번째 단계에서 만든 DLL을 대체하고 내부 클래스 `B`의 정의를 생략합니다.</span><span class="sxs-lookup"><span data-stu-id="19ec2-111">The third file replaces the DLL created by the first step, and omits the definition of the inner class `B`.</span></span>

## <a name="example"></a><span data-ttu-id="19ec2-112">예제</span><span class="sxs-lookup"><span data-stu-id="19ec2-112">Example</span></span>
