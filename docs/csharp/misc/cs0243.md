 <span data-ttu-id="13074-104">[조건부](http://msdn.microsoft.com/library/e1c4913b-74d0-421a-8a6d-c14b3f0e68fb) 특성으로 표시 된 메서드가 허용 되지 않습니다는 [재정의](../../csharp/language-reference/keywords/override.md) 키워드입니다.</span><span class="sxs-lookup"><span data-stu-id="13074-104">The [Conditional](http://msdn.microsoft.com/library/e1c4913b-74d0-421a-8a6d-c14b3f0e68fb) attribute is not allowed on a method that is marked with the [override](../../csharp/language-reference/keywords/override.md) keyword.</span></span> <span data-ttu-id="13074-105">자세한 내용은 참조 [알고 있으면 때 사용 하 여 재정의 하 고 새 키워드](../../csharp/programming-guide/classes-and-structs/knowing-when-to-use-override-and-new-keywords.md)합니다.</span><span class="sxs-lookup"><span data-stu-id="13074-105">For more information, see [Knowing When to Use Override and New Keywords](../../csharp/programming-guide/classes-and-structs/knowing-when-to-use-override-and-new-keywords.md).</span></span>  
  
 <span data-ttu-id="13074-106">컴파일러는 재정의 메서드에 바인딩되지 않습니다. 컴파일러는 기본 메서드에만 바인딩되며, 공용 언어 런타임에서 재정의를 적절하게 호출합니다.</span><span class="sxs-lookup"><span data-stu-id="13074-106">The compiler never binds to override methods; it only binds to the base method and the common language runtime calls the override, as appropriate.</span></span>  
  
 <span data-ttu-id="13074-107">다음 샘플에서는 CS0243을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="13074-107">The following sample generates CS0243:</span></span>  
  
```  
// CS0243.cs  
// compile with: /target:library  
public class MyClass  
{  
   public virtual void M() {}  
}  
  
public class MyClass2 : MyClass  
{  
   [System.Diagnostics.ConditionalAttribute("MySymbol")]   // CS0243  
   // remove Conditional attribute or remove override keyword  
   public override void M() {}  
}  
```
