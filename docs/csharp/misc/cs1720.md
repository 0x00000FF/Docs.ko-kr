# <a name="compiler-warning-level-1-cs1720"></a><span data-ttu-id="b3132-102">컴파일러 경고(수준 1) CS1720</span><span class="sxs-lookup"><span data-stu-id="b3132-102">Compiler Warning (level 1) CS1720</span></span>
<span data-ttu-id="b3132-103">'generic type'의 기본값이 null이므로 식에서 항상 System.NullReferenceException이 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="b3132-103">Expression will always cause a System.NullReferenceException because the default value of 'generic type' is null</span></span>  
  
 <span data-ttu-id="b3132-104">참조 형식(예: 클래스)인 제네릭 형식 변수의 기본값을 포함하는 식을 작성하면 이 오류가 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="b3132-104">If you write an expression involving the default of a generic type variable that is a reference type (for example, a class), this error will occur.</span></span> <span data-ttu-id="b3132-105">다음 식을 살펴보세요.</span><span class="sxs-lookup"><span data-stu-id="b3132-105">Consider the following expression:</span></span>  
  
```csharp

```  
  
 <span data-ttu-id="b3132-106">`T` 는 참조 형식이기 때문에 해당 기본값이 null이므로 <xref:System.Object.ToString%2A> 메서드를 적용하려고 하면 <xref:System.NullReferenceException>이 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="b3132-106">Since `T` is a reference type, its default value is null, and so attempting to apply the <xref:System.Object.ToString%2A> method to it will throw a <xref:System.NullReferenceException>.</span></span>  
  
## <a name="example"></a><span data-ttu-id="b3132-107">예제</span><span class="sxs-lookup"><span data-stu-id="b3132-107">Example</span></span>  
 <span data-ttu-id="b3132-108">`T` 형식의 클래스 참조 제약 조건에 따라 `T` 는 참조 형식이 됩니다.</span><span class="sxs-lookup"><span data-stu-id="b3132-108">The class reference constraint on type `T` ensures that `T` is a reference type.</span></span>

```csharp  
// CS1720.cs

public class Tester   
{  
    public static void GenericClass<T>(T t1) where T : class   
    {  
        Console.WriteLine(default(T).ToString());  // CS1720  
    }  
    public static void Main() {}  
}  
```
