# <a name="compiler-warning-level-2-cs0435"></a><span data-ttu-id="54d27-102">컴파일러 경고(수준 2) CS0435</span><span class="sxs-lookup"><span data-stu-id="54d27-102">Compiler Warning (level 2) CS0435</span></span>
<span data-ttu-id="54d27-103">'assembly'의 'namespace' 네임스페이스가 'assembly'에서 가져온 형식 'type'과 충돌합니다.</span><span class="sxs-lookup"><span data-stu-id="54d27-103">The namespace 'namespace' in 'assembly' conflicts with the imported type 'type' in 'assembly'.</span></span> <span data-ttu-id="54d27-104">'어셈블리'에 정의된 네임스페이스 사용</span><span class="sxs-lookup"><span data-stu-id="54d27-104">Using the namespace defined in 'assembly'.</span></span>  
  
 <span data-ttu-id="54d27-105">이 경고는 원본 파일(file_2)의 네임스페이스가 file_1의 가져온 형식과 충돌할 때 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="54d27-105">This warning is issued when a namespace in a source file (file_2) conflicts with an imported type in file_1.</span></span> <span data-ttu-id="54d27-106">컴파일러에서는 원본 파일에 있는 네임스페이스를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="54d27-106">The compiler uses the one in the source file.</span></span>  
  
 <span data-ttu-id="54d27-107">다음 예제에서는 CS0435를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="54d27-107">The following example generates CS0435:</span></span>  
  
 <span data-ttu-id="54d27-108">먼저 아래 파일을 컴파일합니다.</span><span class="sxs-lookup"><span data-stu-id="54d27-108">Compile this file first:</span></span>  
  
```csharp  
// CS0435_1.cs  
// compile with: /t:library  
public class Util
{  
   public class A { }  
}  
```  
  
 <span data-ttu-id="54d27-109">그런 다음 아래 파일을 컴파일합니다.</span><span class="sxs-lookup"><span data-stu-id="54d27-109">Then, compile this file:</span></span>  
  
```csharp  
// CS0435_2.cs  
// compile with: /r:CS0435_1.dll  
  
using System;  
  
namespace Util
{  
   public class A { }  
}  
  
public class Test
{  
   public static void Main()
   {  
      Console.WriteLine(typeof(Util.A)); // CS0435  
   }  
}  
```
