 <span data-ttu-id="30324-104">이러한 제약 조건을 두 번째 형식 매개 변수에서 제약 조건으로 사용할 수 없도록 값 형식 제약 조건이 암시적으로 봉인됩니다.</span><span class="sxs-lookup"><span data-stu-id="30324-104">Value type constraints are implicitly sealed so those constraints cannot be used as constraints on a second type parameter.</span></span> <span data-ttu-id="30324-105">이는 값 형식을 재정의할 수 없기 때문입니다.</span><span class="sxs-lookup"><span data-stu-id="30324-105">This is because value types cannot be overridden.</span></span> <span data-ttu-id="30324-106">이 오류를 해결하려면 첫 번째 형식 매개 변수를 사용하여 간접적으로 수행하는 대신 두 번째 형식 매개 변수에 직접 값 형식 제약 조건을 저장합니다.</span><span class="sxs-lookup"><span data-stu-id="30324-106">To resolve this error, put a value type constraint directly on the second type parameter, instead of doing so indirectly by means of the first type parameter.</span></span>  
  
## <a name="example"></a><span data-ttu-id="30324-107">예</span><span class="sxs-lookup"><span data-stu-id="30324-107">Example</span></span>  
 <span data-ttu-id="30324-108">다음 샘플에서는 CS0456을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="30324-108">The following sample generates CS0456.</span></span>  
  
```csharp  
// CS0456.cs

{  
   public class G5<T> where T : struct  
   {  
      public class N<U> where U : T {}   // CS0456  
      public class N2<U> where U : struct {}   // OK  
   }  
}  
```
