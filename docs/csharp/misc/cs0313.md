<span data-ttu-id="493bb-103">제네릭 형식 또는 메서드 'type2'에서 'type1' 형식을 형식 매개 변수 'parameter name'으로 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="493bb-103">The type 'type1' cannot be used as type parameter 'parameter name' in the generic type or method 'type2'.</span></span> <span data-ttu-id="493bb-104">nullable 형식 'type1'이 'type2'의 제약 조건을 만족하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="493bb-104">The nullable type 'type1' does not satisfy the constraint of 'type2'.</span></span> <span data-ttu-id="493bb-105">nullable 형식은 인터페이스 제약 조건을 충족할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="493bb-105">Nullable types cannot satisfy any interface constraints.</span></span>  
  
 <span data-ttu-id="493bb-106">nullable 형식은 nullable이 아닌 상응 항목과 동일하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="493bb-106">A nullable type is not equivalent to its non-nullable counterpart.</span></span> <span data-ttu-id="493bb-107">아래에 있는 예제에서 `ImplStruct` 는 `BaseInterface` 제약 조건을 충족하지만 `ImplStruct?` 는 `Nullable<ImplStruct>` 에서 `BaseInterface`를 구현하지 않기 때문에 충족하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="493bb-107">In the example that follows, `ImplStruct` satisfies the `BaseInterface` constraint but `ImplStruct?` does not because `Nullable<ImplStruct>` does not implement `BaseInterface`.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="493bb-108">이 오류를 해결하려면</span><span class="sxs-lookup"><span data-stu-id="493bb-108">To correct this error</span></span>  
  
1.  <span data-ttu-id="493bb-109">아래에 있는 코드를 예제로 사용할 경우 한 가지 해결 방법은 `ImplStruct` 호출의 첫 번째 형식 인수로 일반 `TestMethod`를 지정하는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="493bb-109">Using the code that follows as an example, one solution is to specify an ordinary `ImplStruct` as the first type argument in the call to `TestMethod`.</span></span> <span data-ttu-id="493bb-110">그런 다음 return 문에서 `TestMethod` 의 nullable 버전을 만들도록 `Implstruct` 를 수정합니다.</span><span class="sxs-lookup"><span data-stu-id="493bb-110">Then modify `TestMethod` to create a nullable version of `Implstruct` in its return statement:</span></span>  
  
    ```csharp  
    return new Nullable<T>(t);  
    ```  
  
## <a name="example"></a><span data-ttu-id="493bb-111">예제</span><span class="sxs-lookup"><span data-stu-id="493bb-111">Example</span></span>  
 <span data-ttu-id="493bb-112">다음 코드에서는 CS0313을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="493bb-112">The following code generates CS0313:</span></span>  
  
```csharp  
// cs0313.cs  
public interface BaseInterface { }  
public struct ImplStruct : BaseInterface { }  
  
public class TestClass  
{  
    public T? TestMethod<T, U>(T t) where T : struct, U  
    {  
        return t;  
    }  
}  
  
public class NullableTest  
{  
    public static void Run()  
    {  
  
        TestClass tc = new TestClass();  
        tc.TestMethod<ImplStruct?, BaseInterface>(new ImplStruct?()); // CS0313  
    }  
    public static void Main()  
    { }  
}  
```
