 <span data-ttu-id="4a4c4-104">클래스를 선언할 때 기본 클래스에서 메서드가 아닌 것을 재정의하려고 했습니다.</span><span class="sxs-lookup"><span data-stu-id="4a4c4-104">A class declaration attempted to override a non-method in a base class.</span></span> <span data-ttu-id="4a4c4-105">재정의는 멤버 유형과 일치해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="4a4c4-105">Overrides must match the member type.</span></span> <span data-ttu-id="4a4c4-106">기본 클래스의 메서드로 동일한 이름이 있는 메서드를 사용하려면 기본 클래스의 메서드 선언에서 [새로 만들기](../../csharp/language-reference/keywords/new-modifier.md) 를 사용합니다( [재정의](../../csharp/language-reference/keywords/override.md)사용 안 함).</span><span class="sxs-lookup"><span data-stu-id="4a4c4-106">If a method with the same name as a method in a base class is desired, use [new](../../csharp/language-reference/keywords/new-modifier.md) (and not [override](../../csharp/language-reference/keywords/override.md)) on the method declaration in the base class.</span></span>  
  
 <span data-ttu-id="4a4c4-107">다음 샘플에서는 CS0505를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="4a4c4-107">The following sample generates CS0505:</span></span>  
  
```csharp

{  
   public int i;  
}

{  
   public override int i() { return 0; }   // CS0505  
}  
```
