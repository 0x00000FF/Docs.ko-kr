# <a name="compiler-error-cs0017"></a><span data-ttu-id="15057-102">컴파일러 오류 CS0017</span><span class="sxs-lookup"><span data-stu-id="15057-102">Compiler Error CS0017</span></span>
<span data-ttu-id="15057-103">'output file name' 프로그램에 둘 이상의 진입점이 정의되었습니다.</span><span class="sxs-lookup"><span data-stu-id="15057-103">Program 'output file name' has more than one entry point defined.</span></span> <span data-ttu-id="15057-104">/main으로 컴파일하여 진입점을 포함하는 형식을 지정합니다.</span><span class="sxs-lookup"><span data-stu-id="15057-104">Compile with /main to specify the type that contains the entry point.</span></span>  
  
 <span data-ttu-id="15057-105">프로그램에는 [Main](../programming-guide/main-and-command-args/index.md) 메서드가 하나만 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="15057-105">A program can only have one [Main](../programming-guide/main-and-command-args/index.md) method.</span></span>  
  
 <span data-ttu-id="15057-106">이 오류를 해결 하려면 코드에서 Main 메서드를 하나만 남기고 모두 삭제 하거나 [-main](../language-reference/compiler-options/main-compiler-option.md) 컴파일러 옵션을 사용 하 여 사용 하려는 main 메서드를 지정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="15057-106">To resolve this error, you can either delete all Main methods in your code, except one, or you can use the [-main](../language-reference/compiler-options/main-compiler-option.md) compiler option to specify which Main method you want to use.</span></span>  
  
 <span data-ttu-id="15057-107">다음 샘플에서는 CS0017을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="15057-107">The following sample generates CS0017:</span></span>  
  
```csharp  
// CS0017.cs  
// compile with: /target:exe  
public class clx  
{  
   static public void Main()  
   {  
   }  
}  
  
public class cly  
{  
   public static void Main()   // CS0017, delete one Main or use /main  
   {  
   }  
}  
```
