<span data-ttu-id="51ca2-103">double 형식의 리터럴을 암시적으로 'type' 형식으로 변환할 수 없습니다. 이 형식의 리터럴을 만들려면 'suffix' 접미사를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="51ca2-103">Literal of type double cannot be implicitly converted to type 'type'; use an 'suffix' suffix to create a literal of this type</span></span>  
  
 <span data-ttu-id="51ca2-104">할당을 완료할 수 없습니다. 접미사를 사용하여 지침을 수정합니다.</span><span class="sxs-lookup"><span data-stu-id="51ca2-104">An assignment could not be completed; use a suffix to correct the instruction.</span></span> <span data-ttu-id="51ca2-105">각 형식에 대한 설명서는 형식에 대한 해당 접미사를 식별합니다.</span><span class="sxs-lookup"><span data-stu-id="51ca2-105">The documentation for each type identifies the corresponding suffix for the type.</span></span> <span data-ttu-id="51ca2-106">변환에 대 한 자세한 내용은 [캐스팅 및 형식 변환](../programming-guide/types/casting-and-type-conversions.md)을 참조 하세요.</span><span class="sxs-lookup"><span data-stu-id="51ca2-106">For more information on conversions, see [Casting and Type Conversions](../programming-guide/types/casting-and-type-conversions.md).</span></span>  
  
 <span data-ttu-id="51ca2-107">다음 샘플에서는 CS0664를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="51ca2-107">The following sample generates CS0664:</span></span>  
  
```csharp  
// CS0664.cs  
class Example  
{  
    static void Main()  
    {  
        decimal d1 = 1.0;   // CS0664, because 1.0 is interpreted  
                            // as a double.  
  
        // Try the following line instead.  
        decimal d2 = 1.0M;  // The M tells the compiler that 1.0 is a  
                            // decimal.  
        Console.WriteLine(d2);  
    }  
}  
```
