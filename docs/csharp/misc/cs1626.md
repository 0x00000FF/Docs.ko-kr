# <a name="compiler-error-cs1626"></a><span data-ttu-id="3253b-102">컴파일러 오류 CS1626</span><span class="sxs-lookup"><span data-stu-id="3253b-102">Compiler Error CS1626</span></span>
<span data-ttu-id="3253b-103">catch 절이 포함된 try 블록의 본문에서는 값을 생성할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="3253b-103">Cannot yield a value in the body of a try block with a catch clause</span></span>  
  
 <span data-ttu-id="3253b-104">yield 문은 try 블록과 연결된 catch 절이 있는 경우 try 블록에 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="3253b-104">A yield statement is not allowed in a try block if there is a catch clause associated with the try block.</span></span> <span data-ttu-id="3253b-105">이 오류를 방지 하려면 yield 문을 try/catch/finally 블록 내부에서 외부로 이동 하거나 catch 블록을 제거 합니다.</span><span class="sxs-lookup"><span data-stu-id="3253b-105">To avoid this error, either move the yield statement out of the try/catch/finally block, or remove the catch block.</span></span>
  
 <span data-ttu-id="3253b-106">다음 샘플에서는 CS1626을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="3253b-106">The following sample generates CS1626:</span></span>  
  
```csharp  
// CS1626.cs  
using System.Collections;  
  
class C : IEnumerable  
{  
   public IEnumerator GetEnumerator()  
   {  
      try  
      {  
         yield return this;  // CS1626  
      }  
      catch  
      {  
  
      }  
      finally
      {
      
      }
   }  
}  
  
public class CMain  
{  
   public static void Main() { }  
}  
```
