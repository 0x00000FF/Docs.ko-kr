# <a name="compiler-error-cs1626"></a><span data-ttu-id="e1851-102">컴파일러 오류 CS1626</span><span class="sxs-lookup"><span data-stu-id="e1851-102">Compiler Error CS1626</span></span>
<span data-ttu-id="e1851-103">catch 절이 포함된 try 블록의 본문에서는 값을 생성할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="e1851-103">Cannot yield a value in the body of a try block with a catch clause</span></span>  
  
 <span data-ttu-id="e1851-104">yield 문은 try 블록과 연결된 catch 절이 있는 경우 try 블록에 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="e1851-104">A yield statement is not allowed in a try block if there is a catch clause associated with the try block.</span></span> <span data-ttu-id="e1851-105">이 오류를 방지하려면 yield 문을 catch 절 밖으로 이동합니다.</span><span class="sxs-lookup"><span data-stu-id="e1851-105">To avoid this error, move the yield statement out of the catch clause.</span></span>  
  
 <span data-ttu-id="e1851-106">다음 샘플에서는 CS1626을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="e1851-106">The following sample generates CS1626:</span></span>  
  
```csharp  
// CS1626.cs  
using System.Collections;  
  
class C : IEnumerable  
{  
   public IEnumerator GetEnumerator()  
   {  
      try  
      {  
         yield return this;  // CS1626  
      }  
      catch  
      {  
  
      }  
   }  
}  
  
public class CMain  
{  
   public static void Main() { }  
}  
```
