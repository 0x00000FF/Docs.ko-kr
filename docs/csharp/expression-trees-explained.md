# <a name="expression-trees-explained"></a><span data-ttu-id="9f617-103">식 트리 설명</span><span class="sxs-lookup"><span data-stu-id="9f617-103">Expression Trees Explained</span></span>

<span data-ttu-id="9f617-105">식 트리는 코드를 정의하는 데이터 구조이며,</span><span class="sxs-lookup"><span data-stu-id="9f617-105">An Expression Tree is a data structure that defines code.</span></span> <span data-ttu-id="9f617-106">컴파일러에서 코드를 분석하고 컴파일된 출력을 생성하는 데 사용하는 것과 동일한 구조를 기반으로 합니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-106">They are based on the same structures that a compiler uses to analyze code and generate the compiled output.</span></span> <span data-ttu-id="9f617-107">이 자습서를 읽다 보면 식 트리와 Roslyn API에서 [Analyzers and CodeFixes](https://github.com/dotnet/roslyn-analyzers)(분석기 및 CodeFix)를 빌드하는 데 사용된 형식 사이에 상당히 많은 유사성이 있음을 확인할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-107">As you read through this tutorial, you will notice quite a bit of similarity between Expression Trees and the types used in the Roslyn APIs to build [Analyzers and CodeFixes](https://github.com/dotnet/roslyn-analyzers).</span></span>
<span data-ttu-id="9f617-108">분석기 및 CodeFix는 코드에 대해 정적 분석을 수행하고 개발자에게 잠재적 해결 방법을 제안할 수 있는 NuGet 패키지입니다. 개념이 유사하며 최종 결과는 의미 있는 방식으로 소스 코드를 검사할 수 있는 데이터 구조입니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-108">(Analyzers and CodeFixes are NuGet packages that perform static analysis on code and can suggest potential fixes for a developer.) <span data-ttu-id="9f617-110">간단한 예제를 살펴보겠습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-110">Let's look at a simple example.</span></span>
<span data-ttu-id="9f617-111">코드 줄은 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-111">Here's a line of code:</span></span>
```csharp

```
<span data-ttu-id="9f617-112">이 코드를 식 트리로 분석하려는 경우 트리에 여러 개의 노드가 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-112">If you were to analyze this as an expression tree, the tree contains several nodes.</span></span>
<span data-ttu-id="9f617-113">가장 바깥쪽 노드는 할당을 사용하는 변수 선언문(`var sum = 1 + 2;`)입니다. 이 가장 바깥쪽 노드에는 변수 선언, 대입 연산자 및 등호의 오른쪽을 나타내는 식과 같은 여러 개의 자식 노드가 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-113">The outermost node is a variable declaration statement with assignment (`var sum = 1 + 2;`) That outermost node contains several child nodes: a variable declaration, an assignment operator, and an expression representing the right hand side of the equals sign.</span></span> <span data-ttu-id="9f617-114">이 식은 더하기 연산을 나타내는 식, 더하기의 왼쪽과 오른쪽 피연산자로 다시 구분됩니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-114">That expression is further subdivided into expressions that represent the addition operation, and left and right operands of the addition.</span></span>

<span data-ttu-id="9f617-115">등호의 오른쪽을 구성하는 식을 좀더 자세히 살펴보겠습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-115">Let's drill down a bit more into the expressions that make up the right side of the equals sign.</span></span>
<span data-ttu-id="9f617-116">식은 `1 + 2`이며,</span><span class="sxs-lookup"><span data-stu-id="9f617-116">The expression is `1 + 2`.</span></span> <span data-ttu-id="9f617-117">이진 식입니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-117">That's a binary expression.</span></span> <span data-ttu-id="9f617-118">보다 구체적으로 이진 더하기 식입니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-118">More specifically, it's a binary addition expression.</span></span> <span data-ttu-id="9f617-119">이진 더하기 식에는 더하기 식의 왼쪽과 오른쪽 노드를 나타내는 두 개의 자식이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-119">A binary addition expression has two children, representing the left and right nodes of the addition expression.</span></span> <span data-ttu-id="9f617-120">여기에서 두 노드는 상수 식입니다. 왼쪽 피연산자는 `1` 값이고, 오른쪽 피연산자는 `2` 값입니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-120">Here, both nodes are constant expressions: The left operand is the value `1`, and the right operand is the value `2`.</span></span>

<span data-ttu-id="9f617-121">시각적으로 전체 문은 트리입니다. 루트 노드에서 시작하여 트리의 각 노드로 이동하면서 문을 구성하는 코드를 확인할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-121">Visually, the entire statement is a tree: You could start at the root node, and travel to each node in the tree to see the code that makes up the statement:</span></span>

- <span data-ttu-id="9f617-122">할당을 사용하는 변수 선언문(`var sum = 1 + 2;`)</span><span class="sxs-lookup"><span data-stu-id="9f617-122">Variable declaration statement with assignment (`var sum = 1 + 2;`)</span></span>
    * <span data-ttu-id="9f617-123">암시적 변수 형식 선언(`var sum`)</span><span class="sxs-lookup"><span data-stu-id="9f617-123">Implicit variable type declaration (`var sum`)</span></span>
        - <span data-ttu-id="9f617-124">암시적 var 키워드(`var`)</span><span class="sxs-lookup"><span data-stu-id="9f617-124">Implicit var keyword (`var`)</span></span>
        - <span data-ttu-id="9f617-125">변수 이름 선언(`sum`)</span><span class="sxs-lookup"><span data-stu-id="9f617-125">Variable name declaration (`sum`)</span></span>
    * <span data-ttu-id="9f617-126">대입 연산자(`=`)</span><span class="sxs-lookup"><span data-stu-id="9f617-126">Assignment operator (`=`)</span></span>
    * <span data-ttu-id="9f617-127">이진 더하기 식(`1 + 2`)</span><span class="sxs-lookup"><span data-stu-id="9f617-127">Binary addition expression (`1 + 2`)</span></span>
        - <span data-ttu-id="9f617-128">왼쪽 피연산자(`1`)</span><span class="sxs-lookup"><span data-stu-id="9f617-128">Left operand (`1`)</span></span>
        - <span data-ttu-id="9f617-129">더하기 연산자(`+`)</span><span class="sxs-lookup"><span data-stu-id="9f617-129">Addition operator (`+`)</span></span>
        - <span data-ttu-id="9f617-130">오른쪽 피연산자(`2`)</span><span class="sxs-lookup"><span data-stu-id="9f617-130">Right operand (`2`)</span></span>

<span data-ttu-id="9f617-131">다음 코드는 복잡해 보이지만 매우 강력합니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-131">This may look complicated, but it is very powerful.</span></span> <span data-ttu-id="9f617-132">동일한 프로세스에 따라 훨씬 더 복잡한 식을 분해할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-132">Following the same process, you can decompose much more complicated expressions.</span></span> <span data-ttu-id="9f617-133">다음 식을 살펴보세요.</span><span class="sxs-lookup"><span data-stu-id="9f617-133">Consider this expression:</span></span>
```csharp
var finalAnswer = this.SecretSauceFunction(
    currentState.createInterimResult(), currentState.createSecondValue(1, 2),
    decisionServer.considerFinalOptions("hello")) +
    MoreSecretSauce('A', DateTime.Now, true);
```

<span data-ttu-id="9f617-134">위의 식은 할당을 사용하는 변수 선언이기도 합니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-134">The expression above is also a variable declaration with an assignment.</span></span>
<span data-ttu-id="9f617-135">이 경우 할당의 오른쪽이 훨씬 더 복잡한 트리입니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-135">In this instance, the right hand side of the assignment is a much more complicated tree.</span></span>
<span data-ttu-id="9f617-136">이 식은 분해하지 않고 서로 다른 노드 부분을 살펴보겠습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-136">I'm not going to decompose this expression, but consider what the different nodes might be.</span></span> <span data-ttu-id="9f617-137">현재 개체를 수신기로 사용하는 메서드 호출, 명시적 `this` 수신기가 있는 호출과 그렇지 않은 호출이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-137">There are method calls using the current object as a receiver, one that has an explicit `this` receiver, one that does not.</span></span> <span data-ttu-id="9f617-138">다른 수신기 개체를 사용하는 메서드 호출이 있고 다양한 형식의 상수 인수가 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-138">There are method calls using other receiver objects, there are constant arguments of different types.</span></span> <span data-ttu-id="9f617-139">마지막으로 이진 더하기 연산자가 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-139">And finally, there is a binary addition operator.</span></span> <span data-ttu-id="9f617-140">`SecretSauceFunction()` 또는 `MoreSecretSauce()`의 반환 형식에 따라 해당 이진 더하기 연산자는 재정의된 더하기 연산자에 대한 메서드 호출이 되어 클래스에 대해 정의된 이진 더하기 연산자에 대한 정적 메서드 호출로 확인될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-140">Depending on the return type of `SecretSauceFunction()` or `MoreSecretSauce()`, that binary addition operator may be a method call to an overridden addition operator, resolving to a static method call to the binary addition operator defined for a class.</span></span>

<span data-ttu-id="9f617-141">이러한 복잡성에도 불구하고 위의 식은 첫 번째 샘플만큼 쉽게 탐색할 수 있는 트리 구조를 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-141">Despite this perceived complexity, the expression above creates a tree structure that can be navigated as easily as the first sample.</span></span> <span data-ttu-id="9f617-142">식에서 자식 노드를 계속 트래버스하여 리프 노드를 찾을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-142">You can keep traversing child nodes to find leaf nodes in the expression.</span></span> <span data-ttu-id="9f617-143">부모 노드에는 자식에 대한 참조가 있으며 각 노드에는 노드의 종류를 설명하는 속성이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-143">Parent nodes will have references to their children, and each node has a property that describes what kind of node it is.</span></span>

<span data-ttu-id="9f617-144">식 트리의 구조는 거의 일치합니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-144">The structure of an expression tree is very consistent.</span></span> <span data-ttu-id="9f617-145">기본 사항을 익혔으면 식 트리로 표현될 때 가장 복잡한 코드도 이해할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-145">Once you've learned the basics, you can understand even the most complex code when it is represented as an expression tree.</span></span> <span data-ttu-id="9f617-146">간결한 데이터 구조는 C# 컴파일러에서 가장 복잡한 C# 프로그램을 분석하고 복잡한 해당 소스 코드에서 적절한 출력을 만드는 방법을 설명합니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-146">The elegance in the data structure explains how the C# compiler can analyze the most complex C# programs and create proper output from that complicated source code.</span></span>

<span data-ttu-id="9f617-147">식 트리의 구조에 익숙해지고 나면 신속하게 습득한 지식을 통해 훨씬 더 많은 고급 시나리오를 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-147">Once you become familiar with the structure of expression trees, you will find that knowledge you've gained quickly enables you to work with many more and more advanced scenarios.</span></span> <span data-ttu-id="9f617-148">식 트리에는 정말 강력한 기능이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-148">There is incredible power to expression trees.</span></span>

<span data-ttu-id="9f617-149">알고리즘을 변환하여 다른 환경에서 실행하는 것 외에 식 트리를 사용하면 코드를 실행하기 전에 검사하는 알고리즘을 더 쉽게 작성할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-149">In addition to translating algorithms to execute in other environments, expression trees can be used to make it easier to write algorithms that inspect code before executing it.</span></span> <span data-ttu-id="9f617-150">인수가 식인 메서드를 작성한 다음 코드를 실행하기 전에 해당 식을 검사할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-150">You can write a method whose arguments are expressions and then examine those expressions before executing the code.</span></span> <span data-ttu-id="9f617-151">식 트리는 코드의 전체 표현이며, 모든 하위 식의 값을 확인할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-151">The Expression Tree is a full representation of the code: you can see values of any sub-expression.</span></span>
<span data-ttu-id="9f617-152">메서드 및 속성 이름을 확인할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-152">You can see method and property names.</span></span> <span data-ttu-id="9f617-153">모든 상수 식의 값을 확인할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-153">You can see the value of any constant expressions.</span></span>
<span data-ttu-id="9f617-154">또한 식 트리를 실행 가능한 대리자로 변환하고 코드를 실행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9f617-154">You can also convert an expression tree into an executable delegate, and execute the code.</span></span>
