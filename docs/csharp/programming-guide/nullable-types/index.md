## <a name="nullable-types-overview"></a><span data-ttu-id="0b294-110">null 허용 형식 개요</span><span class="sxs-lookup"><span data-stu-id="0b294-110">Nullable Types Overview</span></span>  
 <span data-ttu-id="0b294-111">null 허용 형식은 다음 특성을 갖습니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-111">Nullable types have the following characteristics:</span></span>  
  
-   <span data-ttu-id="0b294-112">null 허용 형식은 `null` 값이 할당될 수 있는 값-형식 변수를 나타냅니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-112">Nullable types represent value-type variables that can be assigned the value of `null`.</span></span> <span data-ttu-id="0b294-113">참조 형식에 따라 null 허용 형식을 만들 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-113">You cannot create a nullable type based on a reference type.</span></span> <span data-ttu-id="0b294-114">(참조 형식은 `null` 값을 이미 지원합니다.)</span><span class="sxs-lookup"><span data-stu-id="0b294-114">(Reference types already support the `null` value.)</span></span>  
  
-   <span data-ttu-id="0b294-115">구문 `T?`는 <xref:System.Nullable%601>의 축약형입니다. 여기서 `T`는 값 형식입니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-115">The syntax `T?` is shorthand for <xref:System.Nullable%601>, where `T` is a value type.</span></span> <span data-ttu-id="0b294-116">두 가지 형태는 동일하게 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-116">The two forms are interchangeable.</span></span>  
  
-   <span data-ttu-id="0b294-117">일반 값 형식의 경우처럼 null 허용 형식에 값을 할당합니다(예: `int? x = 10;` 또는 `double? d = 4.108;`).</span><span class="sxs-lookup"><span data-stu-id="0b294-117">Assign a value to a nullable type just as you would for an ordinary value type, for example `int? x = 10;` or `double? d = 4.108;`.</span></span> <span data-ttu-id="0b294-118">null 허용 형식에 값 `null`을 할당할 수도 있습니다. `int? x = null;`</span><span class="sxs-lookup"><span data-stu-id="0b294-118">A nullable type can also be assigned the value `null`: `int? x = null;`.</span></span>  
  
-   <span data-ttu-id="0b294-119"><xref:System.Nullable%601.GetValueOrDefault%2A?displayProperty=nameWithType> 메서드를 사용하여 할당된 값 또는 내부 형식의 기본값(값이 `null`인 경우)을 반환합니다(예: `int j = x.GetValueOrDefault();`).</span><span class="sxs-lookup"><span data-stu-id="0b294-119">Use the <xref:System.Nullable%601.GetValueOrDefault%2A?displayProperty=nameWithType> method to return either the assigned value, or the default value for the underlying type if the value is `null`, for example `int j = x.GetValueOrDefault();`</span></span>  
  
-   <span data-ttu-id="0b294-120">다음 예제와 같이 <xref:System.Nullable%601.HasValue%2A> 및 <xref:System.Nullable%601.Value%2A> 읽기 전용 속성을 사용하여 null인지 테스트하고 값을 검색합니다. `if(x.HasValue) j = x.Value;`</span><span class="sxs-lookup"><span data-stu-id="0b294-120">Use the <xref:System.Nullable%601.HasValue%2A> and <xref:System.Nullable%601.Value%2A> read-only properties to test for null and retrieve the value, as shown in the following example: `if(x.HasValue) j = x.Value;`</span></span>  
  
    -   <span data-ttu-id="0b294-121">`HasValue` 속성은 변수에 값이 있으면 `true`를, `null`이면 `false`를 반환합니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-121">The `HasValue` property returns `true` if the variable contains a value, or `false` if it is `null`.</span></span>  
  
    -   <span data-ttu-id="0b294-122">`Value` 속성은 할당된 값이 있으면 해당 값을 반환합니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-122">The `Value` property returns a value if one is assigned.</span></span> <span data-ttu-id="0b294-123">그렇지 않으면 <xref:System.InvalidOperationException?displayProperty=nameWithType>이 throw됩니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-123">Otherwise, a <xref:System.InvalidOperationException?displayProperty=nameWithType> is thrown.</span></span>  
  
    -   <span data-ttu-id="0b294-124">`HasValue`의 기본값은 `false`입니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-124">The default value for `HasValue` is `false`.</span></span> <span data-ttu-id="0b294-125">`Value` 속성에는 기본값이 없습니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-125">The `Value` property has no default value.</span></span>  
  
    -   <span data-ttu-id="0b294-126">다음 예제와 같이 null 허용 형식에 `==` 및 `!=` 연산자를 사용할 수도 있습니다. `if (x != null) y = x;`</span><span class="sxs-lookup"><span data-stu-id="0b294-126">You can also use the `==` and `!=` operators with a nullable type, as shown in the following example: `if (x != null) y = x;`</span></span>  
  
-   <span data-ttu-id="0b294-127">`??` 연산자를 사용하여 현재 값이 `null`인 null 허용 형식이 null을 허용하지 않는 형식에 할당되면 적용될 기본값을 할당합니다(예: `int? x = null; int y = x ?? -1;`).</span><span class="sxs-lookup"><span data-stu-id="0b294-127">Use the `??` operator to assign a default value that will be applied when a nullable type whose current value is `null` is assigned to a non-nullable type, for example `int? x = null; int y = x ?? -1;`</span></span>  
  
-   <span data-ttu-id="0b294-128">중첩된 null 허용 형식은 허용되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="0b294-128">Nested nullable types are not allowed.</span></span> <span data-ttu-id="0b294-129">다음 줄은 컴파일되지 않습니다. `Nullable<Nullable<int>> n;`</span><span class="sxs-lookup"><span data-stu-id="0b294-129">The following line will not compile: `Nullable<Nullable<int>> n;`</span></span>
