> [!NOTE]

<span data-ttu-id="f5ee2-107">.NET Core 1.0의 경우 라이브러리가 NuGet 패키지로 배포되어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-107">For .NET Core 1.0, libraries are expected to be distributed as NuGet packages.</span></span>  <span data-ttu-id="f5ee2-108">이는 실제로 모든 .NET 표준 라이브러리가 배포되고 사용되는 방법이며,</span><span class="sxs-lookup"><span data-stu-id="f5ee2-108">This is in fact how all of the .NET Standard libraries are distributed and consumed.</span></span>  <span data-ttu-id="f5ee2-109">`dotnet pack` 명령을 사용하여 가장 쉽게 수행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-109">This is most easily done with the `dotnet pack` command.</span></span>

<span data-ttu-id="f5ee2-110">NuGet을 통해 배포하려는 놀라운 새 라이브러리를 작성했다고 가정해 보세요.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-110">Imagine that you just wrote an awesome new library that you would like to distribute over NuGet.</span></span>  <span data-ttu-id="f5ee2-111">플랫폼 간 도구를 사용하여 NuGet 패키지를 만들면 이 작업을 정확하게 수행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-111">You can create a NuGet package with cross platform tools to do exactly that!</span></span>  <span data-ttu-id="f5ee2-112">다음 예제에서는 `netstandard1.0`을 대상으로 하는 **SuperAwesomeLibrary**라는 라이브러리를 가정합니다.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-112">The following example assumes a library called **SuperAwesomeLibrary** which targets `netstandard1.0`.</span></span>

<span data-ttu-id="f5ee2-113">전이적 종속성 즉, 다른 프로젝트에 종속되어 있는 프로젝트가 있는 경우 NuGet 패키지를 만들기 전에 `dotnet restore` 명령을 사용하여 전체 솔루션에 대한 패키지를 복원해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-113">If you have transitive dependencies; that is, a project which depends on another project, you'll need to make sure to restore packages for your entire solution with the `dotnet restore` command before creating a NuGet package.</span></span>  <span data-ttu-id="f5ee2-114">이렇게 복원하지 않으면 `dotnet pack` 명령이 제대로 작동하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-114">Failing to do so will result in the `dotnet pack` command to not work properly.</span></span>

<span data-ttu-id="f5ee2-115">패키지가 복원되었는지 확인한 후 라이브러리가 있는 디렉터리로 이동할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-115">After ensuring packages are restored, you can navigate to the directory where a library lives:</span></span>

<span data-ttu-id="f5ee2-120">이제 `/bin` 폴더에 릴리스 이진 파일과 함께 NuGet 패키지를 포함하는 `release` 폴더가 있습니다.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-120">Your `/bin` folder will now have a `release` folder containing your NuGet package with release binaries:</span></span>

```
SuperAwesomeLibrary.1.0.0.symbols.nupkg
```

<span data-ttu-id="f5ee2-121">또한 NuGet 패키지를 게시하는 데 필요한 파일이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="f5ee2-121">And now you have the necessary files to publish a NuGet package!</span></span>
