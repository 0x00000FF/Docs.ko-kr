### <a name="why-create-a-framework-dependent-deployment"></a><span data-ttu-id="28d35-119">프레임워크 종속 배포를 만드는 이유</span><span class="sxs-lookup"><span data-stu-id="28d35-119">Why create a framework-dependent deployment?</span></span>

## <a name="self-contained-deployments-scd"></a><span data-ttu-id="28d35-131">자체 포함 배포(SCD)</span><span class="sxs-lookup"><span data-stu-id="28d35-131">Self-contained deployments (SCD)</span></span>

<span data-ttu-id="28d35-132">자체 포함 배포에서는 앱과 필요한 타사 종속성 외에도 앱을 빌드하는 데 사용한 .NET Core 버전도 배포합니다.</span><span class="sxs-lookup"><span data-stu-id="28d35-132">For a self-contained deployment, you deploy your app and any required third-party dependencies along with the version of .NET Core that you used to build the app.</span></span> <span data-ttu-id="28d35-133">SCD를 만들 때 다양한 플랫폼의 [.NET Core에 대한 기본 종속성](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md)을 포함하지 않으므로 앱을 실행하려면 이러한 종속성이 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="28d35-133">Creating an SCD doesn't include the [native dependencies of .NET Core](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md) on various platforms, so these must be present before the app runs.</span></span> <span data-ttu-id="28d35-134">런타임 시 버전 바인딩에 대한 자세한 내용은 [.NET Core의 버전 바인딩](../versions/selection.md)에 대한 문서를 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="28d35-134">For more information on version binding at runtime, see the article on [version binding in .NET Core](../versions/selection.md).</span></span>

NuGet.org에서 자동으로 다운로드됩니다. 패치 버전 롤포워드를 옵트아웃하는 방법을 비롯하여 자세한 내용은 [자체 포함 배포 런타임 롤포워드](runtime-patch-selection.md)를 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="28d35-137">(The latest serviced runtime includes security patches and other bug fixes.) <span data-ttu-id="28d35-140">자체 포함 배포를 배포하면 다음과 같은 두 가지 주요 장점이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="28d35-140">Deploying a Self-contained deployment has two major advantages:</span></span>
