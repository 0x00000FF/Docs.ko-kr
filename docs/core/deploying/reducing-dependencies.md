# <a name="reducing-package-dependencies-with-projectjson"></a><span data-ttu-id="fd547-103">project.json으로 패키지 종속성 감소</span><span class="sxs-lookup"><span data-stu-id="fd547-103">Reducing Package Dependencies with project.json</span></span>

<span data-ttu-id="fd547-104">이 문서에서는 `project.json` 라이브러리를 작성할 때 패키지 종속성을 줄이는 방법에 대해 알아야 할 내용을 다룹니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-104">This article covers what you need to know about reducing your package dependencies when authoring `project.json` libraries.</span></span> <span data-ttu-id="fd547-105">이 문서를 마칠 때는 필요한 종속성만을 사용하도록 라이브러리를 작성하는 방법을 이해하게 될 것입니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-105">By the end of this article, you will learn how to compose your library such that it only uses the dependencies it needs.</span></span>

## <a name="why-its-important"></a><span data-ttu-id="fd547-106">중요한 이유</span><span class="sxs-lookup"><span data-stu-id="fd547-106">Why it's Important</span></span>

<span data-ttu-id="fd547-107">.NET Core는 NuGet 패키지로 구성된 제품입니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-107">.NET Core is a product made up of NuGet packages.</span></span>  <span data-ttu-id="fd547-108">필수 패키지는 [.NET 표준 라이브러리 메타패키지](https://www.nuget.org/packages/NETStandard.Library)입니다. 이는 다른 패키지들로 구성된 NuGet 패키지로,</span><span class="sxs-lookup"><span data-stu-id="fd547-108">An essential package is the [.NETStandard.Library metapackage](https://www.nuget.org/packages/NETStandard.Library), which is a NuGet package composed of other packages.</span></span> <span data-ttu-id="fd547-109">.NET Framework, .NET Core 및 Xamarin/Mono 같은 여러 .NET 구현에서 작업할 수 있도록 보장하는 패키지 집합을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-109">It provides you with the set of packages that are guaranteed to work on multiple .NET implementations, such as .NET Framework, .NET Core, and Xamarin/Mono.</span></span>

<span data-ttu-id="fd547-110">그러나 라이브러리에서 포함된 모든 패키지를 사용하지는 않을 것입니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-110">However, there's a good chance that your library won't use every single package it contains.</span></span>  <span data-ttu-id="fd547-111">라이브러리를 작성하여 NuGet을 통해 배포할 때, 실제로 사용하는 패키지만으로 종속성을 "잘라내는" 것이 가장 바람직합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-111">When authoring a library and distributing it over NuGet, it's a best practice to "trim" your dependencies down to only the packages you actually use.</span></span>  <span data-ttu-id="fd547-112">이렇게 하면 NuGet 패키지에서 전체적인 공간을 적게 차지합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-112">This results in a smaller overall footprint for NuGet packages.</span></span>

## <a name="how-to-do-it"></a><span data-ttu-id="fd547-113">방법</span><span class="sxs-lookup"><span data-stu-id="fd547-113">How to do it</span></span>

<span data-ttu-id="fd547-114">현재로서는 패키지 참조를 잘라내는 공식 `dotnet` 명령이 없습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-114">Currently, there is no official `dotnet` command that trims package references.</span></span>  <span data-ttu-id="fd547-115">이 작업을 수동으로 수행해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-115">Instead, you'll have to do it manually.</span></span>  <span data-ttu-id="fd547-116">일반적인 프로세스는 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-116">The general process looks like the following:</span></span>

1. <span data-ttu-id="fd547-117">`NETStandard.Library`의 `1.6.0` 섹션에서 `dependencies` 버전 `project.json`을 참조합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-117">Reference `NETStandard.Library` version `1.6.0` in a `dependencies` section of your `project.json`.</span></span>
2. <span data-ttu-id="fd547-118">명령줄에서 `dotnet restore`([참고 참조](#dotnet-restore-note))로 패키지를 복원합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-118">Restore packages with `dotnet restore` ([see note](#dotnet-restore-note)) from the command line.</span></span>
3. <span data-ttu-id="fd547-119">`project.lock.json` 파일을 검사하고 `NETStandard.Library` 섹션을 찾습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-119">Inspect the `project.lock.json` file and find the `NETStandard.Library` section.</span></span>  <span data-ttu-id="fd547-120">파일의 시작 부분에 있습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-120">It's near the beginning of the file.</span></span>
4. <span data-ttu-id="fd547-121">`dependencies` 아래에 나열된 모든 패키지를 복사합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-121">Copy all of the listed packages under `dependencies`.</span></span>
5. <span data-ttu-id="fd547-122">`.NETStandard.Library` 참조를 제거하고 복사된 패키지로 교체합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-122">Remove the `.NETStandard.Library` reference and replace it with the copied packages.</span></span>
6. <span data-ttu-id="fd547-123">필요 없는 패키지에 대한 참조를 제거합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-123">Remove references to packages you don't need.</span></span>

<span data-ttu-id="fd547-124">다음 방법 중 하나를 사용하여 필요 없는 패키지를 알아낼 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-124">You can find out which packages you don't need by one of the following ways:</span></span>

1. <span data-ttu-id="fd547-125">시행착오.</span><span class="sxs-lookup"><span data-stu-id="fd547-125">Trial and error.</span></span> <span data-ttu-id="fd547-126">패키지를 제거하고 복원하면서 여전히 컴파일되는지를 알아보고 이 프로세스를 반복합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-126">This involves removing a package, restoring, seeing if your library still compiles, and repeating this process.</span></span>
2. <span data-ttu-id="fd547-127">참조를 피킹하여 실제로 어떤 코드가 사용되는지를 알아보기 위해 [ILSpy](https://github.com/icsharpcode/ILSpy#ilspy-------) 또는 [.NET Reflector](https://www.red-gate.com/products/dotnet-development/reflector) 같은 도구를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-127">Using a tool such as [ILSpy](https://github.com/icsharpcode/ILSpy#ilspy-------) or [.NET Reflector](https://www.red-gate.com/products/dotnet-development/reflector) to peek at references to see what your code is actually using.</span></span> <span data-ttu-id="fd547-128">그런 다음 사용 중인 형식에 해당하지 않는 패키지를 제거할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-128">You can then remove packages that don't correspond to types you're using.</span></span>

## <a name="example"></a><span data-ttu-id="fd547-129">예제</span><span class="sxs-lookup"><span data-stu-id="fd547-129">Example</span></span>

<span data-ttu-id="fd547-130">일반 컬렉션 형식에 추가 기능을 제공한 라이브러리를 작성했다고 가정해 보겠습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-130">Imagine that you wrote a library that provided additional functionality to generic collection types.</span></span> <span data-ttu-id="fd547-131">그런 라이브러리는 `System.Collections` 같은 패키지에 종속되어야 하지만, `System.Net.Http` 같은 패키지에는 전혀 종속되지 않을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-131">Such a library would need to depend on packages such as `System.Collections`, but may not at all depend on packages such as `System.Net.Http`.</span></span> <span data-ttu-id="fd547-132">따라서 이 라이브러리에 필요한 것으로만 패키지 종속성을 잘라낼 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-132">As such, it would be good to trim package dependencies down to only what this library required!</span></span>

<span data-ttu-id="fd547-133">이 라이브러리를 잘라내려면 `project.json` 파일로 시작해서 `NETStandard.Library` 버전 `1.6.0`에 대한 참조를 추가합니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-133">To trim this library, you start with the `project.json` file and add a reference to `NETStandard.Library` version `1.6.0`.</span></span>

<span data-ttu-id="fd547-134">그런 다음 `dotnet restore`([참고 참조](#dotnet-restore-note))로 패키지를 복원하고, `project.lock.json` 파일을 검사하고, `NETStandard.Library`에 대해 복원된 모든 패키지를 찾습니다.</span><span class="sxs-lookup"><span data-stu-id="fd547-134">Next, you restore packages with `dotnet restore` ([see note](#dotnet-restore-note)), inspect the `project.lock.json` file, and find all the packages restored for `NETStandard.Library`.</span></span>
