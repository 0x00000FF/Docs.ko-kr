# 생성자 디자인
생성자의 두 가지가: 생성자 및 인스턴스 생성자를 입력 합니다.  
  
 형식 생성자는 정적 및 형식을 사용 하기 전에 CLR에서 실행 됩니다. 인스턴스 생성자는 형식의 인스턴스를 만들 때 실행 합니다.  
  
 형식 생성자 매개 변수를 사용할 수 없습니다. 인스턴스 생성자는 다음 작업을 할 수 있습니다. 매개 변수를 사용 하지 마십시오 인스턴스 생성자에 기본 생성자 라고 합니다.  
  
 생성자는 형식의 인스턴스를 만드는 가장 자연 스러운 방법입니다. 대부분의 개발자가 검색 하 고 \(예: 팩터리 메서드\) 인스턴스를 만드는 여러 가지를 고려 하기 전에 생성자를 사용 하려고 합니다.  
  
 **✓ 고려** 단순, 이상적으로 기본, 생성자를 제공 합니다.  
  
 단순 생성자에 매개 변수의 매우 작은 숫자 및 모든 매개 변수는 기본 형식 또는 열거형입니다. 이러한 간단한 생성자 프레임 워크의 유용성을 늘립니다.  
  
 **✓ 고려** 아니면 부자연 스러운 느낌 생성자 디자인 지침에 따라 원하는 작업의 의미 체계의 새 인스턴스를 생성에 직접 매핑되지 않는 경우 생성자를 대신 정적 팩터리 메서드를 사용 합니다.  
  
 **✓ 수행** 주 속성을 설정 하기 위한 생성자 매개 변수 바로 가기로 사용 합니다.  
  
 의미 간의 차이가 없어야 빈 생성자 일부 속성 집합을 통해 다음을 사용 하 고 생성자를 사용 하 여 여러 인수를 사용 합니다.  
  
 **✓ 수행** 생성자 매개 변수는 단순히 속성을 설정 하는 데 사용 되는 경우 생성자 매개 변수 및 속성에 대 한 같은 이름을 사용 합니다.  
  
 이러한 매개 변수 및 속성 간의 유일한 차이점 대\/소문자 구분 해야 합니다.  
  
 **✓ 수행** 생성자에서 최소한의 작업입니다.  
  
 생성자 안 캡처 이외의 작업량 생성자 매개 변수입니다. 필요할 때까지 다른 처리 비용은 지연 해야 합니다.  
  
 **✓ 수행** 해당 하는 경우 인스턴스 생성자에서 예외를 throw 합니다.  
  
 **✓ 수행** 이러한 생성자가 필요한 경우 클래스에 공용 기본 생성자를 명시적으로 선언 합니다.  
  
 형식에서 생성자를 명시적으로 선언 하지, 다양 한 언어 \(예: C\#\) 공용 기본 생성자를 자동으로 추가 됩니다. \(추상 클래스에는 protected 생성자를 가져옵니다.\)  
  
 클래스에 매개 변수가 있는 생성자를 추가 컴파일러에서 기본 생성자를 추가 하지 못하도록 합니다. 이 실수로 주요 변경 사항이 많은 경우.  
  
 **X 방지** 구조체에 기본 생성자를 명시적으로 정의 합니다.  
  
 이렇게 하면 배열 만들기 보다 빠르게 없기 때문에 기본 생성자를 정의 하지 않은 경우이 배열에 있는 모든 슬롯에서 실행 될 수 있습니다. Note C\#을 포함 하 여 많은 컴파일러에서는이 매개 변수가 없는 생성자가 하는 구조체를 허용 하지 않음.  
  
 **X 방지** 개체의 생성자 내에서 가상 멤버를 호출 합니다.  
  
 가상 멤버를 호출 하면 가장 많이 파생 된 재정의를 호출할 수는 가장 많이 파생 된 형식의 생성자가 완전히 아직 실행 되지 하는 경우에 됩니다.  
  
### 형식 생성자 지침  
 **✓ 수행** 정적 생성자를 비공개로 설정 합니다.  
  
 형식을 초기화 하는 정적 생성자는 클래스 생성자를 사용 됩니다. CLR 형식의 첫 번째 인스턴스를 만들거나 해당 형식에 정적 멤버 호출 되기 전에 static 생성자를 호출 합니다. 사용자에 정적 생성자를 호출할 때 제어할 수 없습니다. 정적 생성자가 private이 아니면 CLR 이외의 코드에서 호출할 수 있습니다. 생성자에서 수행 하는 작업에 따라이 예기치 않은 동작이 발생할 수 있습니다. C\# 컴파일러는 정적 생성자를 전용 되도록 합니다.  
  
 **X 하지 않으려면** 정적 생성자에서 예외를 throw 합니다.  
  
 형식 생성자에서 예외가 발생 하는 경우 형식에에서 없는 경우 사용할 수 있는 현재 응용 프로그램 도메인  
  
 **✓ 고려** 런타임 형식에서 명시적으로 정의 된 정적 생성자가 없는의 성능을 최적화할 수 있기 때문에 정적 생성자를 사용 하 여 명시적으로 하는 대신 정적 필드 인라인을 초기화 합니다.
