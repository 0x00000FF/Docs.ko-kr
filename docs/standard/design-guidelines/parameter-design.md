 <span data-ttu-id="73eb5-159">이 메서드를 호출 하 여 배열 요소를 인수 목록에 직접 전달 하 여 사용자 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="73eb5-159">Doing this allows the user to call the method by passing the array elements directly in the argument list.</span></span>  
  
 `String.Format("File {0} not found in {1}",filename,directory);`  
  
 <span data-ttu-id="73eb5-160">Params 키워드 매개 변수 목록의 마지막 매개 변수에 추가할 수 있는지 확인 합니다.</span><span class="sxs-lookup"><span data-stu-id="73eb5-160">Note that the params keyword can be added only to the last parameter in the parameter list.</span></span>  
  
 <span data-ttu-id="73eb5-161">**✓ 고려** 는 적은 수의 요소 배열을 전달 하는 최종 사용자가 예상 되는 경우에 params 키워드 배열 매개 변수를 추가 합니다.</span><span class="sxs-lookup"><span data-stu-id="73eb5-161">**✓ CONSIDER** adding the params keyword to array parameters if you expect the end users to pass arrays with a small number of elements.</span></span> <span data-ttu-id="73eb5-162">예상 하지만 다양 한 요소 전달할 공통 시나리오, 사용자가 이러한 요소를 인라인으로 누르고, 통과 하지 못할는 및에 params 키워드 이므로 필요 하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="73eb5-162">If it’s expected that lots of elements will be passed in common scenarios, users will probably not pass these elements inline anyway, and so the params keyword is not necessary.</span></span>

 <span data-ttu-id="73eb5-174">특별 한 경우 전체 코드 경로 배열을 만듭니다 뿐 아니라 한 보다 일반적인 메서드를 호출 하려는 경우에 다음이 수행만 해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="73eb5-174">You should only do this if you are going to special-case the entire code path, not just create an array and call the more general method.</span></span>
