# 열거형 디자인
열거형은 값 형식의 특수 한 종류입니다. 두 가지 방법으로는 열거형: 단순 열거형 및 플래그 열거형입니다.  
  
 단순 열거형 중 작은 닫힌된 집합을 나타냅니다. 단순 열거형의 일반적인 예로 색 집합입니다.  
  
 플래그 열거형은 enum 값에 비트 작업을 지원 하도록 설계 되었습니다. 플래그 열거형의 일반적인 예로 옵션의 목록이 있습니다.  
  
 **✓ 수행** 열거형을 사용 하 여 강력한 형식의 매개 변수, 속성 및 값 집합을 나타내는 값을 반환 합니다.  
  
 **✓ 수행** 정적 상수 대신 열거형을 사용 합니다.  
  
 **X 하지 않으려면** 집합 \(예: 운영 체제 버전, 친구, 등의 이름입니다.\)에 대 한 열거형을 사용 합니다.  
  
 **X 하지 않으려면** 나중에 사용 하려는 예약 된 열거형 값을 제공 합니다.  
  
 이후 단계에서 기존 열거형에 값을 항상 간단히 추가할 수 있습니다. 참조 [열거형에 값 추가](#add_value) 에 대 한 자세한 내용은 열거형에 값을 추가 합니다. 예약된 값 오염 실제 값 집합이 고 사용자 오류가 발생 하는 경향이 했습니다.  
  
 **X 방지** 열거형 값을 하나만로 공개적으로 노출 합니다.  
  
 C Api의 다음 버전의 확장을 보장 하는 데 일반적으로 메서드 서명에 예약 된 매개 변수를 추가 하는 것입니다. 이러한 예약 된 매개 변수는 단일 기본 값을 가진 열거형으로 나타낼 수 있습니다. 이 관리 되는 Api에서 하면 안 됩니다. 메서드 오버 로딩 추가할 수 있습니다. 나중에 매개 변수를 해제 합니다.  
  
 **X 하지 않으려면** 열거형에서 센티널 값을 포함 합니다.  
  
 프레임 워크 개발자에 게 도움이 되기도 하지만 센티널 값은 프레임 워크의 사용자에 게 혼동 합니다. Enum으로 표시 되는 집합에서 값 중 하나는 것 보다는 열거형의 상태를 추적 하는 데 사용 됩니다.  
  
 **✓ 수행** 단순 열거형에는 0 값을 제공 합니다.  
  
 "None"입니다. 같은 값을 호출 하는 것이 좋습니다. 이러한 값이 특정 열거형에 대 한 적합 하지 않은 경우 열거형에 대 한 가장 일반적인 기본값 0의 내부 값을 할당 해야 합니다.  
  
 **✓ 고려** 를 사용 하 여 <xref:System.Int32> \(대부분의 프로그래밍 언어의 기본값\) 열거형의 내부 형식으로는 다음과 같은 경우에:  
  
-   열거형 플래그 열거형 이며 32 개 보다 많은 플래그 또는 앞으로 더 많아질 하기를 원합니다.  
  
-   기본 형식이 다를 수 해야 <xref:System.Int32> 열거형 다른 크기를 예상 하는 관리 되지 않는 코드와 쉽게 상호 운용성에 대 한 합니다.  
  
-   내부 형식이 더 작은 공간에 크게 절감으로 반환 됩니다. 제어 흐름에 대 한 인수로 서 주로 사용할 열거형에 예정 이라면 크기 거의 차이가 있습니다. 크기 절약을 그다지 경우:  
  
    -   매우 자주 인스턴스화되는 구조 또는 클래스의 필드로 사용할 enum을 예상 합니다.  
  
    -   원하는 사용자가 큰 배열 또는 열거형 인스턴스의 컬렉션을 만들 수 있습니다.  
  
    -   Serialize 되는 열거형의 인스턴스 수가 많은 예상 합니다.  
  
 메모리 사용량에 대 한 주의 관리 되는 개체는 항상 `DWORD`\-정렬 되므로 효과적으로 여러 열거형 또는 기타 작은 구조 때문에 총 인스턴스 크기는 항상 최대 반올림할 차이 확인 하기 위해 사용 하 여 더 작은 열거형 팩 인스턴스에 `DWORD`합니다.  
  
 **✓ 수행** 단 수 명사 또는 명사구로 간단한 열거형 및 플래그 열거형을 복수 명사 또는 명사구로 이름을 지정 합니다.  
  
 **X 하지 않으려면** 확장 <xref:System.Enum?displayProperty=fullName> 직접.  
  
 <xref:System.Enum?displayProperty=fullName> 특수 한 유형 CLR에서 만드는 데 사용자 정의 열거형. 대부분의 프로그래밍 언어는이 기능에 액세스할 수 있는 프로그래밍 요소를 제공 합니다. 예를 들어 C\#에서 `enum` 키워드 열거를 정의 하는 데 사용 됩니다.  
  
<a name="design"></a>   
### 플래그 열거형 디자인  
 **✓ 수행** 적용 된 <xref:System.FlagsAttribute?displayProperty=fullName> 플래그 열거형에 있습니다. 이 특성을 단순 열거형에 적용 되지 않습니다.  
  
 **✓ 수행** 수 자유롭게 결합 비트 OR 연산을 사용 하 여 플래그 열거형 값에 2의 제곱을 사용 합니다.  
  
 **✓ 고려** 플래그의 조합을 사용 되는 일반적으로 특수 한 열거 값을 제공 합니다.  
  
 비트 연산 고급 개념 이며 간단한 작업에 대 한 필요가 없어야 합니다.<xref:System.IO.FileAccess> 이러한 특수 한 값의 예입니다.  
  
 **X 방지** 만들기 플래그 열거형 값의 조합이 올바르지 않습니다.  
  
 **X 방지** 값 "플래그를 모두 선택 취소"를 나타내는 이름은 다음 지침에서 설명 했 듯이 적절 하지 않는 한 열거형 값이 0 인 플래그를 사용 하 여 합니다.  
  
 **✓ 수행** 0 플래그 열거형 값의 이름을 `None`합니다. 플래그 열거형에 대 한 값 해야 한다고 해 서 반드시 "플래그를 모두 선택 취소 합니다."  
  
<a name="add_value"></a>   
### 열거형에 값 추가  
 것은 매우 일반적를 이미 배송 한 후 열거형에 값을 추가 해야 합니다. 잠재적인 응용 프로그램 호환성 문제가 있습니다 새로 추가 된 값을 기존 API에서 반환 되 면 잘못 작성 된 응용 프로그램의 새 값을 올바르게 처리 하지 않을 수 있습니다.  
  
 **✓ 고려** 작은 호환성 위험 불구 하 고 열거형 값을 추가 합니다.  
  
 열거형에 대 한 추가 인해 발생 하는 응용 프로그램 호환성 문제에 대 한 실제 데이터를 설정한 경우에 새 부모와 이전 값을 반환 하는 새로운 API를 추가 하는 것이 좋습니다. 하 고 이전 값만 반환 계속 해야 하는 기존 API를 사용할 수 없게 합니다. 이렇게 하면 기존 응용 프로그램이 계속 호환 됩니다.
