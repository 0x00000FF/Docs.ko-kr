# 중첩 형식
중첩된 형식은 바깥쪽 형식의 이라고 하는 다른 형식의 범위 내에 정의 된 유형이 있습니다. 중첩된 형식은 바깥쪽 형식의 모든 멤버에 액세스할 수 있음 예를 들어 전용 필드를 바깥쪽 형식의 모든 상위 항목에 정의 된 필드를 보호 하 고 바깥쪽 형식의 정의에 대 한 액세스를 있습니다.  
  
 일반적으로 중첩된 형식이 필요한 경우에 사용 해야 합니다. 여기에는 여러 가지 이유가 있습니다. 일부 개발자 들은 완전 하 게 개념에 익숙하지 않은 것입니다. 이러한 개발자 중첩 형식의 변수를 선언 구문 사용 하 여 문제가 예를 들어 할 수 있습니다. 중첩된 형식은 바깥쪽 형식도 매우 밀접 하 게 결합 하 고 이와 같이 범용 형식에 적합 하지 않습니다.  
  
 중첩된 형식은 자신의 바깥쪽 형식의 구현 세부 정보를 모델링 하는 데 적합 합니다. 최종 사용자 중첩 형식의 변수를 선언할 필요가 거의 결코에 있어야 합니다 중첩된 형식을 명시적으로 인스턴스화합니다. 예를 들어, 컬렉션의 열거자 중첩된 형식은 해당 컬렉션의 수입니다. 열거자를 바깥쪽 형식으로 일반적으로 인스턴스화됩니다 있고 다양 한 언어 지원 하므로 foreach 문을 열거자 변수 거의 최종 사용자가 선언 될 수 있습니다.  
  
 **✓ 수행** 중첩 형식과 외부 형식 간의 관계는 멤버 액세스 가능성 의미 체계는 바람직한 되도록 하는 경우 중첩 된 형식을 사용 합니다.  
  
 **X 하지 않으려면** 논리적 그룹으로 public 중첩된 형식을 사용 하 여 네임 스페이스를 사용 하 여이 생성 합니다.  
  
 **X 방지** 중첩된 형식 공개적으로 노출 됩니다. 이 유일한 예외는 중첩 형식의 변수를 서브클래싱 또는 기타 고급 사용자 지정 시나리오와 같이 드문 시나리오 에서만에서 선언 해야 하는 경우.  
  
 **X 하지 않으려면** 형식이 포함 하는 형식 외부에서 참조 될 가능성이 높은 경우 중첩 된 형식을 사용 합니다.  
  
 예를 들어, 클래스에 정의 된 메서드에 전달 되는 enum 클래스에 중첩 된 형식으로 정의 되어야 합니다.  
  
 **X 하지 않으려면** 클라이언트 코드에서 인스턴스화되는 경우 중첩 된 형식을 사용 합니다.  형식에 public 생성자가 있는 경우 중첩 하지 않이 되어야 합니다.  
  
 형식을 인스턴스화할 수, 있으면 하는 것 처럼 보이지만 형식에는 자체적으로 프레임 워크에서 \(수 만들를 사용 하 여, 외부 형식을 사용할 필요 없이 삭제\), 따라서 중첩 해야 합니다. 내부 형식 관계 없는 외부 형식 외부에서 광범위 하 게 재사용 되지 않아야 외부 형식에 어떠한 합니다.  
  
 **X 하지 않으려면** 인터페이스의 구성원으로 중첩 된 형식을 정의 합니다. 대부분의 언어에서는 이러한 구문을 지원 하지 않습니다.
