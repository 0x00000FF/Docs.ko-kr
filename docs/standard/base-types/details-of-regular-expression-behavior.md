 <span data-ttu-id="e1441-113">DFA 엔진과 달리 기존의 NFA 엔진이 패턴 일치를 수행할 경우 해당 처리 순서는 정규식 패턴에 의해 결정됩니다.</span><span class="sxs-lookup"><span data-stu-id="e1441-113">Unlike DFA engines, when traditional NFA 문자열의 수만큼 찾습니다.</span><span class="sxs-lookup"><span data-stu-id="e1441-114">As it processes a particular language element, the engine uses greedy matching; that is, it matches as much of the input after successfully matching a subexpression.</span></span> <span data-ttu-id="e1441-116">일치에 결국 실패하면 엔진이 추가 일치를 시도할 수 있도록 저장된 상태로 돌아갈 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="e1441-116">If a match eventually fails, the engine can return to a saved state so it can try additional matches.</span></span> <span data-ttu-id="e1441-117">정규식에서 이후 언어 요소가 일치할 수 있도록 성공적인 class="sxs-lookup"><span data-stu-id="e1441-118">NFA engines use backtracking to test all possible expansions of a regular expression in a specific order and accept the first match.</span></span> <span data-ttu-id="e1441-119">기존의 NFA 엔진이 성공적으로 일치를 수행하기 위해 정규식의 특정 확장을 구성하기 때문에 하위 식 일치 및 일치 역참조를 캡처할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="e1441-119">Because a traditional NFA data-ttu-id="e1441-120">그러나 기존의 NFA가 역추적하기 때문에 다른 경로를 통해 그 상태가 된 경우 여러 번 동일한 상태가 될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="e1441-120">However, because a traditional NFA backtracks,  <span data-ttu-id="e1441-125">기존의 NFA 엔진은 DFA 또는 POSIX NFA 엔진보다 일치하는 문자열을 효율적으로 제어하기 때문에 프로그래머로부터 선호되었습니다.</span><span class="sxs-lookup"><span data-stu-id="e1441-125">Traditional NFA engines are favored by programmers because they offer greater control over string matching than either DFA or POSIX NFA engines.</span></span> <span data-ttu-id="e1441-126">최악의 경우 실행 속도가 느려질 수 있지만 모호성을 줄이고 역추적을 제한하는 패턴을 사용하여 선형 또는 다항 시간에서 일치 항목을 찾도록 조정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="e1441-126">Although, in the worst case, they can run slowly, you can steer them to find matches in linear or polynomial time by using patterns that reduce ambiguities and limit backtracking.</span></span> <span data-ttu-id="e1441-127">즉, NFA 엔진은 강력함과 유연성을 제공하는 대신 성능이 떨어지지만 대부분의 경우에 정규식을 잘 작성하여 역추적이 성능을 기하급수적으로 저하시키지 않도록 방지한다면 사용 가능한 적절한 성능을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="e1441-127">In other words, although NFA engines trade performance for power and flexibility, in most cases they offer good to acceptable performance if a regular expression is well-written and avoids cases in which backtracking degrades performance exponentially.</span></span>
