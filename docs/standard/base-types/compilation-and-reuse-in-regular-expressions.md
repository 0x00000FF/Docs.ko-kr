 <span data-ttu-id="1c355-108">경우는 <xref:System.Text.RegularExpressions.Regex> 개체로 구성 되며는 <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> 옵션을 정규식을 고급 정규식 내부 지침 대신 명시적 MSIL 코드를 컴파일합니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-108">If a <xref:System.Text.RegularExpressions.Regex> object is constructed with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option, it compiles the regular expression to explicit MSIL code instead of high-level regular expression internal instructions.</span></span> <span data-ttu-id="1c355-109">이렇게 하면 .NET의 JIT(Just-In-Time) 컴파일러가 성능 향상을 위해 식을 네이티브 기계어 코드로 변환할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-109">This allows .NET's just-in-time (JIT) compiler to convert the expression to native machine code for higher performance.</span></span>

 <span data-ttu-id="1c355-113">다른 정규식 사용 하 여 컴파일할 수를 제한 해야는 <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> 너무 많은 리소스를 사용 하지 않도록 하려면 옵션입니다.</span><span <span data-ttu-id="1c355-116">대신 미리 컴파일된 정규식을 사용 하는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-116">An alternative is to use precompiled regular expressions.</span></span> <span ## <a name="the-regular-expressions-cache"></a><span data-ttu-id="1c355-119">정규식 캐시</span><span class="sxs-lookup"><span data-stu-id="1c355-119">The Regular Expressions Cache</span></span>  
 <span data-ttu-id="1c355-120">성능 향상을 위해 정규식 엔진은 응용 프로그램 수준의 컴파일된 정규식 캐시를 유지 관리합니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-120">To improve performance, the regular expression engine maintains an application-wide cache of compiled regular expressions.</span></span> <span data-ttu-id="1c355-121">캐시는 정적 메서드 호출에만 사용되는 정규식 패턴을 저장합니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-121">The cache stores regular expression patterns that are used only in static method calls.</span></span> <span data-ttu-id="1c355-122">인스턴스 메서드에 제공된 정규식 패턴은 캐시되지 않습니다. 따라서 사용할 때마다 식을 고급 바이트 코드로 다시 구문 분석할 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-122">(Regular expression patterns supplied to instance methods are not cached.)  <span data-ttu-id="1c355-123">캐시 된 정규식의 최대 수의 값에 의해 결정 됩니다는 `static` (`Shared` Visual basic에서) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> 속성입니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-123">The maximum number of cached regular expressions is determined by the value of the `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="1c355-124">기본적으로 정규식 엔진은 최대 15개의 컴파일된 정규식을 캐시합니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-124">By default, the regular expression engine caches up to 15 compiled regular expressions.</span></span> <span data-ttu-id="1c355-125">컴파일된 정규식 개수가 캐시 크기를 초과하면 가장 오래 전에 사용한 정규식이 삭제되고 새 정규식이 캐시됩니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-125">If the number of compiled regular expressions exceeds the cache size, the least recently used regular expression is discarded and the new regular expression is cached.</span></span>  
  
 <span data-ttu-id="1c355-126">응용 프로그램은 다음 두 가지 방법 중 하나로 미리 컴파일된 정규식을 활용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-126">Your application can take advantage of precompiled regular expressions in one of the following two ways:</span></span>  
  
-   <span data-ttu-id="1c355-127">정적 메서드를 사용 하 여는 <xref:System.Text.RegularExpressions.Regex> 정규식을 정의 하는 개체입니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-127">By using a static method of the <xref:System.Text.RegularExpressions.Regex> object to define the regular expression.</span></span> <span data-ttu-id="1c355-128">다른 정적 메서드 호출에서 이미 정의된 정규식 패턴을 사용하는 경우 정규식 엔진이 캐시에서 검색합니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-128">If you are using a regular expression pattern that has already been defined in another static method call, the regular expression engine will retrieve it from the cache.</span></span> <span data-ttu-id="1c355-129">이미 정의된 정규식 패턴을 사용하지 않는 경우 엔진이 정규식을 컴파일하고 캐시에 추가합니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-129">If not, the engine will compile the regular expression and add it to the cache.</span></span>  
  
-   <span data-ttu-id="1c355-130">기존 다시 사용 하 여 <xref:System.Text.RegularExpressions.Regex> 정규식 패턴은 필요한 경우 개체입니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-130">By reusing an existing <xref:System.Text.RegularExpressions.Regex> object as long as its regular expression pattern is needed.</span></span>  
  
 <span data-ttu-id="1c355-131">개체 인스턴스화 및 만들고 신속 하 게 삭제 하면 다양 한 정규식 컴파일 오버 헤드로 인해 <xref:System.Text.RegularExpressions.Regex> 개체는 비용이 많이 드는 프로세스입니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-131">Because of the overhead of object instantiation and regular expression compilation, creating and rapidly destroying numerous <xref:System.Text.RegularExpressions.Regex> objects is a very expensive process.</span></span> <span data-ttu-id="1c355-132">많은 수의 정규식을 사용 하는 응용 프로그램의 경우 정적에 대 한 호출을 사용 하 여 성능을 최적화할 수 `Regex` 메서드와 늘려서 정규식 캐시의 크기입니다.</span><span class="sxs-lookup"><span data-stu-id="1c355-132">For applications that use a large number of different regular expressions, you can optimize performance by using calls to static `Regex` methods and possibly by increasing the size of the regular expression cache.</span></span>
