## <a name="compiled-regular-expressions"></a><span data-ttu-id="52e24-105">컴파일된 정규식</span><span class="sxs-lookup"><span data-stu-id="52e24-105">Compiled Regular Expressions</span></span>  
 <span data-ttu-id="52e24-106">기본적으로 정규식 엔진은 정규식을 내부 명령 시퀀스(Microsoft 중간 언어, 즉 MSIL과 다른 고급 코드)로 컴파일합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-106">By default, the regular expression engine compiles a regular expression to a sequence of internal instructions (these are high-level codes that are different from Microsoft intermediate language, or MSIL).</span></span> <span data-ttu-id="52e24-107">엔진은 정규식을 실행할 때 내부 코드를 해석합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-107">When the engine executes a regular expression, it interprets the internal codes.</span></span>  
  
 <span data-ttu-id="52e24-108"><xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> 옵션으로 <xref:System.Text.RegularExpressions.Regex> 개체를 생성하는 경우 정규식이 고급 정규식 내부 명령 대신 명시적 MSIL 코드로 컴파일됩니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-108">If a <xref:System.Text.RegularExpressions.Regex> object is constructed with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option, it compiles the regular expression to explicit MSIL code instead of high-level regular expression internal instructions.</span></span> <span data-ttu-id="52e24-109">이렇게 하면 .NET의 JIT(Just-In-Time) 컴파일러가 성능 향상을 위해 식을 네이티브 기계어 코드로 변환할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-109">This allows .NET's just-in-time (JIT) compiler to convert the expression to native machine code for higher performance.</span></span>  
  
<span data-ttu-id="52e24-110">그러나 생성된 MSIL을 언로드할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-110">However, generated MSIL cannot be unloaded.</span></span> <span data-ttu-id="52e24-111">코드를 언로드하는 유일한 방법은 전체 응용 프로그램 도메인을 언로드하는 것(즉, 모든 응용 프로그램 코드 언로드)입니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-111">The only way to unload code is to unload an entire application domain (that is, to unload all of your application's code.).</span></span> <span data-ttu-id="52e24-112">실제로 <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> 옵션으로 정규식을 컴파일하고 나면, 정규식을 만든 <xref:System.Text.RegularExpressions.Regex> 개체가 가비지 수집에 사용하도록 해제된 경우에도 컴파일된 식에 사용된 리소스를 해제하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-112">Effectively, once a regular expression is compiled with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option,   never releases the resources used by the compiled expression, even if the regular expression was created by a <xref:System.Text.RegularExpressions.Regex> object that is itself released to garbage collection.</span></span>  
  
 <span data-ttu-id="52e24-113">너무 많은 리소스가 사용되지 않도록 <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> 옵션으로 컴파일하는 정규식 수를 제한해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-113">You must be careful to limit the number of different regular expressions you compile with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option to avoid consuming too many resources.</span></span> <span data-ttu-id="52e24-114">응용 프로그램이 많거나 무제한 개수의 <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType>로 컴파일해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-115">However, if a small number of regular expressions are used repeatedly, they should be compiled with <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> for better performance.</span></span> <span data-ttu-id="52e24-116">한 가지 대안은 미리 컴파일된 정규식을 사용하는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-116">An alternative is to use precompiled regular expressions.</span></span> <span data-ttu-id="52e24-117"><xref:System.Text.RegularExpressions.Regex.CompileToAssembly%2A> 메서드를 사용하여 모든 식을 재사용 가능한 DLL로 컴파일할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-117">You can 속도에 따른 이점은 계속 활용하면서 런타임 시 컴파일할 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-118">This avoids the need to compile at runtime while still benefiting from the speed of compiled ## <a name="the-regular-expressions-cache"></a><span data-ttu-id="52e24-119">정규식 캐시</span><span class="sxs-lookup"><span data-stu-id="52e24-119">The Regular Expressions Cache</span></span>  
 <span data-ttu-id="52e24-120">성능 향상을 위해 정규식 엔진은 응용 프로그램 수준의 컴파일된 정규식 캐시를 유지 관리합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-120">To improve performance, the regular expression engine maintains an application-wide cache of compiled regular expressions.</span></span> <span data-ttu-id="52e24-121">캐시는 정적 메서드 호출에만 사용되는 정규식 패턴을 저장합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-121">The cache stores regular expression patterns that are used only in static method calls.</span></span> <span data-ttu-id="52e24-122">인스턴스 메서드에 제공된 정규식 패턴은 캐시되지 않습니다. 따라서 사용할 때마다 식을 고급 바이트 코드로 다시 구문 분석할 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-122">(Regular expression patterns supplied to instance methods are not cached.) This avoids the need to reparse an expression into high-level byte code each time it is used.</span></span>  
  
 <span data-ttu-id="52e24-123">캐시된 정규식의 최대 개수는 `static`(Visual Basic의 경우 `Shared`) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> 속성 값에 의해 결정됩니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-123">The maximum number of cached regular expressions is determined by the value of the `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="52e24-124">기본적으로 정규식 엔진은 최대 15개의 컴파일된 정규식을 캐시합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-124">By default, the regular expression engine caches up to 15 compiled regular expressions.</span></span> <span data-ttu-id="52e24-125">컴파일된 정규식 개수가 캐시 크기를 초과하면 가장 오래 전에 사용한 정규식이 삭제되고 새 정규식이 캐시됩니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-125">If the number of compiled regular expressions exceeds the cache size, the least recently used regular expression is discarded and the new regular expression is cached.</span></span>  
  
 <span data-ttu-id="52e24-126">응용 프로그램은 다음 두 가지 방법 중 하나로 미리 컴파일된 정규식을 활용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-126">Your application can take advantage of precompiled regular expressions in one of the following two ways:</span></span>  
  
-   <span data-ttu-id="52e24-127"><xref:System.Text.RegularExpressions.Regex> 개체의 정적 메서드를 사용하여 정규식을 정의합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-127">By using a static method of the <xref:System.Text.RegularExpressions.Regex> object to define the regular expression.</span></span> <span data-ttu-id="52e24-128">다른 정적 메서드 호출에서 이미 정의된 정규식 패턴을 사용하는 경우 정규식 엔진이 캐시에서 검색합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-128">If you are using a regular expression pattern that has already been defined in another static method call, the regular expression engine will retrieve it from the cache.</span></span> <span data-ttu-id="52e24-129">이미 정의된 정규식 패턴을 사용하지 않는 경우 엔진이 정규식을 컴파일하고 캐시에 추가합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-129">If not, the engine will compile the regular expression and add it to the cache.</span></span>  
  
-   <span data-ttu-id="52e24-130">해당 정규식 패턴이 필요할 때까지 기존 <xref:System.Text.RegularExpressions.Regex> 개체를 다시 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-130">By reusing an existing <xref:System.Text.RegularExpressions.Regex> object as long as its regular expression pattern is needed.</span></span>  
  
 <span data-ttu-id="52e24-131">개체 인스턴스화 및 정규식 컴파일의 오버헤드 때문에 수많은 <xref:System.Text.RegularExpressions.Regex> 개체를 만들고 금세 삭제할 경우 매우 큰 비용이 듭니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-131">Because of the overhead of object instantiation and regular expression compilation, creating and rapidly destroying numerous <xref:System.Text.RegularExpressions.Regex> objects is a very expensive process.</span></span> <span data-ttu-id="52e24-132">다수의 정규식을 사용하는 응용 프로그램의 경우 정적 `Regex` 메서드 호출을 사용하고 정규식 캐시의 크기를 늘려 성능을 최적화할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="52e24-132">For applications that use a large number of different regular expressions, you can optimize performance by using calls to static `Regex` methods and possibly by increasing the size of the regular expression cache.</span></span>
