<span data-ttu-id="1136d-103">엔터티 참조가 확장 되어을 나타내는 텍스트로 **XmlEntityReference** 노드가 만들어지지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-103">When the entity reference is expanded and replaced by the text it represents, the **XmlEntityReference** node is not created.</span></span> <span data-ttu-id="1136d-104">대신 엔터티 선언이 구문 분석 되 고 대신에 복사 되 고 선언의 내용에서 만든 노드는 **XmlEntityReference**합니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-104">Instead, the entity declaration is parsed, and nodes created from the content in the declaration are copied in the place of the **XmlEntityReference**.</span></span> <span data-ttu-id="1136d-105">따라서는 `&publisher;` 예제에서는 `&publisher;` 저장 되지 않고, 대신는 **XmlText** 노드가 생성 됩니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-105">Therefore, in the `&publisher;` example, the `&publisher;` is not saved, but instead, an **XmlText** node is created.</span></span>

<span data-ttu-id="1136d-107">확장되는 엔터티 참조의 트리 구조</span><span class="sxs-lookup"><span data-stu-id="1136d-107">Tree structure for entity references that are expanded</span></span>

 <span data-ttu-id="1136d-110">유지 하기 위해 **XmlEntityReference** 엔터티 참조의 자식 노드 및 노드 연결을 설정는 **EntityHandling** 플래그를 **ExpandCharEntities**합니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-110">To preserve **XmlEntityReference** nodes and child nodes of the entity reference attached to it, set the **EntityHandling** flag to **ExpandCharEntities**.</span></span> <span data-ttu-id="1136d-111">그렇지 않으면 둡니다는 **EntityHandling** 플래그를 하는 기본 **ExpandEntities**합니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-111">Otherwise, leave the **EntityHandling** flag at the default, which is to **ExpandEntities**.</span></span> <span data-ttu-id="1136d-112">이 경우 DOM에서 entityreference 노드가 유지되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-112">In this case, you will not see entity reference nodes in the DOM.</span></span> <span data-ttu-id="1136d-113">노드는 엔터티 선언의 자식 노드 복사본인 노드로 바뀝니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-113">The nodes are replaced by the nodes that are copies of the child nodes of the entity declaration.</span></span>  
  
 <span data-ttu-id="1136d-114">엔터티 참조를 유지하지 않는 경우 발생할 수 있는 부작용 중 하나는 문서를 저장하고 다른 응용 프로그램에 전달할 때 수신 응용 프로그램에서 노드가 엔터티 참조로 만들어졌다는 것을 알 수 없게 된다는 점입니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-114">One side effect of not preserving entity references is that when the document is saved and passed on to another application, the receiving application does not know that the nodes were generated by an entity reference.</span></span> <span data-ttu-id="1136d-115">그러나 엔터티 참조를 유지하는 경우 수신 응용 프로그램에서 엔터티 참조를 인식하고 자식 노드를 읽습니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-115">However, when entity references are preserved, a receiving application sees an entity reference and reads the child nodes.</span></span> <span data-ttu-id="1136d-116">자식 노드가 엔터티 선언에 있던 정보를 나타내는 것은 분명합니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-116">It is apparent that the child nodes represent the information that was in the entity declaration.</span></span> <span data-ttu-id="1136d-117">예를 들어, 엔터티 참조가 유지되는 경우 DOM은 이론적으로 다음과 같은 구조를 가집니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-117">For example, the DOM theoretically has the following structure if entity references are preserved.</span></span>  
  
 <span data-ttu-id="1136d-118">XmlElement: publisher</span><span class="sxs-lookup"><span data-stu-id="1136d-118">XmlElement: publisher</span></span>  
  
 <span data-ttu-id="1136d-119">XmlEntityReference: `&publisher;`</span><span class="sxs-lookup"><span data-stu-id="1136d-119">XmlEntityReference: `&publisher;`</span></span>  
  
 <span data-ttu-id="1136d-120">XmlText: Microsoft Press</span><span class="sxs-lookup"><span data-stu-id="1136d-120">XmlText: Microsoft Press</span></span>  
  
 <span data-ttu-id="1136d-121">엔터티 참조가 DOM에서 확장되는 것이 기본 메서드이며 이 경우 다음 트리 형식의 구조를 갖습니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-121">If entity references are expanded in the DOM, which is the default method, the structure has this type of tree:</span></span>  
  
 <span data-ttu-id="1136d-122">XmlElement: publisher</span><span class="sxs-lookup"><span data-stu-id="1136d-122">XmlElement: publisher</span></span>  
  
 <span data-ttu-id="1136d-123">XmlText: Microsoft Press</span><span class="sxs-lookup"><span data-stu-id="1136d-123">XmlText: Microsoft Press</span></span>  
  
 <span data-ttu-id="1136d-124">엔터티 참조 노드는 사라지고 있고 수신 응용 프로그램 이라는 것을 알 수 없는 **XmlText** "Microsoft Press"를 포함 하는 노드가 엔터티 선언에서 생성 합니다.</span><span class="sxs-lookup"><span data-stu-id="1136d-124">Notice that the entity reference node is gone, and the receiving application cannot tell that the **XmlText** node containing "Microsoft Press" was created from an entity declaration.</span></span>
