-   <span data-ttu-id="ca5d1-104">컨테이너는 프로덕션 환경의 종속성 결핍으로 인한 배포 문제에 대한 해결책이므로 컨테이너 기반 솔루션은 비용 절감이라는 중요한 이점을 제공하여 DevOps 및 프로덕션 운영을 획기적으로 개선합니다.</span><span class="sxs-lookup"><span data-stu-id="ca5d1-104">Container-based solutions provide important benefits of cost savings because containers are a solution to deployment problems caused by the lack of dependencies in production environments, therefore, improving DevOps and production operations significantly.</span></span>

-   <span data-ttu-id="ca5d1-105">Microsoft를 포함한 Windows 및 Linux 에코시스템에서 가장 중요한 공급업체가 지원하는 Docker는 컨테이너 업계의 사실상 표준이 되고 있습니다.</span><span class="sxs-lookup"><span data-stu-id="ca5d1-105">Docker is becoming the de facto standard in the container industry, supported by the most significant vendors in the Linux and Windows ecosystems, including Microsoft.</span></span> <span data-ttu-id="ca5d1-106">향후에는 아마도 클라우드 또는 온-프레미스의 어떤 데이터 센터에서나 Docker를 아주 흔히 보게 될 것입니다.</span><span class="sxs-lookup"><span data-stu-id="ca5d1-106">In the future, Docker will be ubiquitous in any datacenter in the cloud or on-premises.</span></span>

-   <span data-ttu-id="ca5d1-107">Docker 컨테이너는 서버 기반 응용 프로그램 또는 서비스 배포의 표준 단위가 되고 있습니다.</span><span class="sxs-lookup"><span data-stu-id="ca5d1-107">A Docker container is becoming the standard unit of deployment for any server-based application or service.</span></span>

-   <span data-ttu-id="ca5d1-108">Azure Container Service(Mesosphere Datacenter Operating System(DC/OS), Docker Swarm, Kubernetes) 및 Azure Service Fabric에 제공되는 것과 같은 Docker 오케스트레이터는 상당한 복잡성과 확장성이 필요한 마이크로 서비스 기반 또는 마이크로 컨테이너 응용 프로그램에 핵심적이고 필수적입니다.</span><span class="sxs-lookup"><span data-stu-id="ca5d1-108">Docker orchestrators like the ones provided in Azure Container Service (Mesosphere Datacenter Operating System (DC/OS), Docker Swarm, Kubernetes) and Azure Service Fabric are fundamental and indispensable for any microservices-based or multicontainer application with significant complexity and scalability needs.</span></span>

-   <span data-ttu-id="ca5d1-109">프로덕션 Docker 환경에 연결하는 지속적인 통합/지속적인 배포를 지원하는 포괄적인 DevOps 환경은 민첩성을 제공하며, 궁극적으로 응용 프로그램의 출시 시기를 앞당깁니다.</span><span class="sxs-lookup"><span data-stu-id="ca5d1-109">An end-to-end DevOps environment supporting Continuous Integration/Continuous Deployment connecting to the production Docker environments provides agility and ultimately improves the time to market of your applications.</span></span>

<span data-ttu-id="ca5d1-110">Visual Studio Team Services는 Azure에 기반을 둔 단순한 Docker 환경 또는 고급 마이크로 서비스 및 컨테이너 오케스트레이터를 비롯한 지속적인 배포 파이프라인의 Docker 환경에 지정된 DevOps 환경을 매우 간소화합니다.</span><span class="sxs-lookup"><span data-stu-id="ca5d1-110">Visual Studio Team Services greatly simplifies your DevOps environment designated to Docker environments from your Continuous Deployment pipelines, including simple Docker environments or more advanced microservice and container orchestrators based on Azure.</span></span>
