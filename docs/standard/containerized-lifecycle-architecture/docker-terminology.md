# <a name="docker-terminology"></a><span data-ttu-id="3f63a-103">Docker 용어</span><span class="sxs-lookup"><span data-stu-id="3f63a-103">Docker terminology</span></span>

<span data-ttu-id="3f63a-104">용어 및 정의를 잘 알고 있어야와 Docker 자세히 살펴보기 전에 먼저이 섹션에 나열 (추가로 정의 대 한 참조는 광범위 한 [용어집](https://docs.docker.com/glossary/) Docker에서 제공한 <https://docs.docker.com/glossary/>:</span><span class="sxs-lookup"><span data-stu-id="3f63a-104">This section lists terms and definitions with which you should become familiar with before delving deeper into Docker (for further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker at <https://docs.docker.com/glossary/>:</span></span>

-   <span data-ttu-id="3f63a-105">**컨테이너 이미지** 모든 종속성 및 컨테이너를 만드는 데 필요한 정보를 사용 하 여 패키지 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-105">**Container image** A package with all of the dependencies and information needed to create a container.</span></span> <span data-ttu-id="3f63a-106">이미지는 종속성 (예: 프레임 워크), 배포 및 컨테이너 런타임에서 사용할 구성을 모두 포함 됩니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-106">An image includes all of the dependencies (such as frameworks) plus deployment and configuration to be used by a container runtime.</span></span> <span data-ttu-id="3f63a-107">일반적으로 이미지에서에서 파생 여러 기본 이미지 계층으로 쌓이는 지 하나는 컨테이너의 파일 시스템을 구성 하기 위해 다른 위에 표시 됩니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-107">Usually, an image derives from multiple base images that are layers stacked one atop the other to form the container's file system.</span></span> <span data-ttu-id="3f63a-108">만든 후에 이미지는 변경할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-108">An image is immutable after it has been created.</span></span>

-   <span data-ttu-id="3f63a-109">**컨테이너** Docker 이미지의 인스턴스입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-109">**Container** An instance of a Docker image.</span></span> <span data-ttu-id="3f63a-110">컨테이너는 단일 응용 프로그램, 프로세스 또는 서비스에 대 한 런타임을 나타냅니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-110">A container represents a runtime for a single application, process, or service.</span></span> <span data-ttu-id="3f63a-111">Docker 이미지를, 런타임 환경 및 명령의 표준 집합의 내용을 구성 됩니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-111">It consists of the contents of a Docker image, a runtime environment, and a standard set of instructions.</span></span> <span data-ttu-id="3f63a-112">서비스의 크기를 조정하는 경우 동일한 이미지에서 컨테이너의 여러 인스턴스를 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-112">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="3f63a-113">또는 일괄 작업 각 인스턴스에 다른 매개 변수를 전달 하는 동일한 이미지에서 여러 컨테이너를 만들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-113">Or, a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

-   <span data-ttu-id="3f63a-114">**태그** 서로 다른 이미지 또는 (버전 번호 또는 대상 환경)에 따라 동일한 이미지의 버전을 식별할 수 있도록 이미지에 적용할 수 있는 레이블 또는 표시 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-114">**Tag** A mark or label that you can apply to images so that different images or versions of the same image (depending on the version number or the destination environment) can be identified.</span></span>

-   <span data-ttu-id="3f63a-115">**Dockerfile** Docker 이미지를 작성 하는 방법에 대 한 지침을 포함 하는 텍스트 파일입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-115">**Dockerfile** A text file that contains instructions for how to build a Docker image.</span></span>

-   <span data-ttu-id="3f63a-116">**빌드** 정보와 이미지 빌드되는 폴더에 있는 추가 파일 뿐만 아니라 해당 Dockerfile에서 제공 하는 컨텍스트를 기준으로 작업의 컨테이너 이미지를 작성 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-116">**Build** The action of building a container image based on the information and context provided by its Dockerfile as well as additional files in the folder where the image is built.</span></span> <span data-ttu-id="3f63a-117">Docker docker 빌드 명령을 사용 하 여 이미지를 작성할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-117">You can build images by using the Docker docker build command.</span></span>

-   <span data-ttu-id="3f63a-118">**리포지토리 (리 포 라고도 함)** 이미지 버전을 나타내는 태그를 붙이지 관련된 Docker 이미지의 컬렉션입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-118">**Repository (also known as repo)** A collection of related Docker images labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="3f63a-119">일부 저장소 포함 런타임 (밝은)만 포함 된 이미지 (많은) Sdk에 포함 된 이미지와 같은 특정 이미지의 여러 변형을 추가할 등입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-119">Some repositories contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), and so on.</span></span> <span data-ttu-id="3f63a-120">이러한 변형은 태그로 표시할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-120">Those variants can be marked with tags.</span></span> <span data-ttu-id="3f63a-121">단일 저장소 플랫폼 변형 Linux 이미지 등 Windows 이미지를 포함할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-121">A single repository can contain platform variants, such as a Linux image and a Windows image.</span></span>

-   <span data-ttu-id="3f63a-122">**레지스트리** 저장소에 대 한 액세스를 제공 하는 서비스입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-122">**Registry** A service that provides access to repositories.</span></span> <span data-ttu-id="3f63a-123">대부분의 공용 이미지에 대한 기본 레지스트리는 [Docker 허브](https://hub.docker.com/)(조직인 Docker에서 소유함)입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="3f63a-124">레지스트리는 일반적으로 여러 팀의 리포지토리를 포함합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="3f63a-125">회사에 저장 하 고 이미지를 작성 한 관리 사설 레지스트리 있는 경우가 많습니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-125">Companies often have private registries to store and manage images that they've created.</span></span> <span data-ttu-id="3f63a-126">*Azure 컨테이너 레지스트리* 은 또 다른 예제입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-126">*Azure Container Registry* is another example.</span></span>

-   <span data-ttu-id="3f63a-127">**Docker 허브** 이미지를 업로드 하 고 작업을 하려면 공개 레지스트리 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-127">**Docker Hub** A public registry to upload images and work with them.</span></span> <span data-ttu-id="3f63a-128">Docker 허브는 Docker 이미지 호스팅, 공개 또는 개인 레지스트리, 빌드 트리거 및 웹후크, GitHub 및 Bitbucket과 통합을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

-   <span data-ttu-id="3f63a-129">**Azure 컨테이너 레지스트리** Docker 이미지와 Azure에서 구성 요소를 사용 하기 위한 공용 리소스입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-129">**Azure Container Registry** A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="3f63a-130">여기서는 Azure에서 배포에 가깝고, 액세스에 대한 제어를 제공하는 레지스트리를 제공하여 Azure Active Directory 그룹 및 권한을 사용할 수 있도록 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

-   <span data-ttu-id="3f63a-131">**Docker 신뢰 레지스트리 DTR ()** A Docker 레지스트리 (Docker)에서 설치할 수 있는 온-프레미스 서비스를 회사의 데이터 센터 및 네트워크에 상주 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-131">**Docker Trusted Registry (DTR)** A Docker registry service (from Docker) that you can install on-premises so that it resides within the organization's datacenter and network.</span></span> <span data-ttu-id="3f63a-132">엔터프라이즈 내에서 관리되어야 하는 개인 이미지에 유용합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="3f63a-133">Docker Trusted Registry는 Docker 데이터 센터 제품의 일부로 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="3f63a-134">자세한 내용은 [ https://docs.docker.com/docker-trusted-registry/overview/ ](https://docs.docker.com/docker-trusted-registry/overview/)합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-134">For more information, go to [https://docs.docker.com/docker-trusted-registry/overview/](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

-   <span data-ttu-id="3f63a-135">**Docker Community Edition (CE)** 창과 macOS 빌드, 실행 및 컨테이너를 로컬로 테스트를 위한 개발 도구입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-135">**Docker Community Edition (CE)** Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="3f63a-136">Windows용 Docker CE는 Linux 및 Windows 컨테이너에 개발 환경을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="3f63a-137">Windows의 Linux Docker 호스트를 기반으로 한 [Hyper-v](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) VM입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) VM.</span></span> <span data-ttu-id="3f63a-138">Windows 컨테이너에 대한 호스트는 Windows를 직접 기반으로 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="3f63a-139">Apple 하이퍼바이저 프레임 워크를 기반으로 하는 Mac 용 docker CE 및 [xhyve 하이퍼바이저](https://github.com/mist64/xhyve), Mac OS X. CE Windows 용 Docker와 Mac에 대 한 Linux Docker 호스트 VM을 제공 하는 Oracle VirtualBox을 기반으로 하는 Docker 도구 상자를 대체 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host VM on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

-   <span data-ttu-id="3f63a-140">**Docker Enterprise Edition (EE)** 는 엔터프라이즈급 버전의 Linux 및 Windows 개발을 위한 Docker 도구입니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-140">**Docker Enterprise Edition (EE)** An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

-   <span data-ttu-id="3f63a-141">**작성** 명령줄 도구 및 YAML 파일 형식으로 메타 데이터를 정의 하 고 multicontainer 응용 프로그램을 실행 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-141">**Compose** A command-line tool and YAML file format with metadata for defining and running multicontainer applications.</span></span> <span data-ttu-id="3f63a-142">환경에 따라 값을 재정의할 수 있는 하나 이상의 .yml 파일을 사용하여 여러 이미지를 기반으로 하는 단일 응용 프로그램을 정의합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="3f63a-143">정의 만든 후 단일 명령을 사용 하 여 전체 multicontainer 응용 프로그램을 배포할 수 있습니다 (docker-를 작성) 이미지당 컨테이너 Docker 호스트에 만들어지는 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-143">After you have created the definitions, you can deploy the entire multicontainer application by using a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

-   <span data-ttu-id="3f63a-144">**클러스터** 컬렉션 큐브인 것 처럼 단일 가상 Docker 호스트 응용 프로그램 서비스의 여러 인스턴스를 확장할 수 있도록 노출 하는 Docker 호스트 클러스터 내의 여러 호스트에 분산 합니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-144">**Cluster** A collection of Docker hosts exposed as if they were a single virtual Docker host so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="3f63a-145">Docker는 Docker Swarm, Mesosphere DC/OS, Kubernetes, 및 Azure Service Fabric을 사용 하 여 Docker 클러스터를 만들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3f63a-145">You can create Docker clusters by using Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="3f63a-146">(클러스터를 관리하는 데 Docker Swarm을 사용하는 경우 일반적으로 클러스터 대신 클러스터를 *swarm*으로 참조합니다.)</span><span class="sxs-lookup"><span data-stu-id="3f63a-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>
