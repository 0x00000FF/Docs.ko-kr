# Using Threads and Threading
이 단원의 항목에서는 관리되는 스레드 만들기 및 관리, 관리되는 스레드에 데이터를 전달하고 결과를 다시 가져오는 방법 및 스레드를 삭제하고 <xref:System.Threading.ThreadAbortException>을 처리하는 방법을 설명합니다.  
  
## 단원 내용  
 [Creating Threads and Passing Data at Start Time](../../../docs/standard/threading/creating-threads-and-passing-data-at-start-time.md)  
 새 스레드에 데이터를 전달하는 방법과 데이터를 다시 가져오는 방법을 비롯하여 관리되는 스레드 만들기를 설명합니다.  
  
 [Pausing and Resuming Threads](../../../docs/standard/threading/pausing-and-resuming-threads.md)  
 관리되는 스레드를 일시 중지하고 다시 시작하는 문제를 설명합니다.  
  
 [Destroying Threads](../../../docs/standard/threading/destroying-threads.md)  
 관리되는 스레드를 없애는 문제와 <xref:System.Threading.ThreadAbortException>을 처리하는 방법을 설명합니다.  
  
 [Scheduling Threads](../../../docs/standard/threading/scheduling-threads.md)  
 스레드의 우선 순위와 이 우선 순위가 스레드 스케줄링에 주는 영향을 설명합니다.  
  
## 참조  
 <xref:System.Threading.Thread>  
 관리되는 스레드가 비관리 코드에서 파생되었는지 관리되는 응용 프로그램에서 만들어졌는지에 상관없이 관리되는 스레드를 나타내는 <xref:System.Threading.Thread> 클래스의 참조 설명서를 제공합니다.  
  
 <xref:System.Threading.ThreadStart>  
 매개 변수가 없는 스레드 프로시저를 나타내는 <xref:System.Threading.ThreadStart> 대리자에 대한 참조 설명서를 제공합니다.  
  
 <xref:System.Threading.ParameterizedThreadStart>  
 강력한 형식화를 수행하지 않고도 스레드 프로시저에 데이터를 쉽게 전달할 수 있는 방법을 제공합니다.
