# <a name="using-threads-and-threading"></a>스레드 및 스레딩 사용
이 섹션의 항목 생성 및 관리의 관리 되는 스레드, 관리 되는 스레드 데이터를 전달 하 고 결과 다시 가져오는 방법 및 스레드를 삭제 하 고 처리 하는 방법에 설명 된 <xref:System.Threading.ThreadAbortException>합니다.  
  
## <a name="in-this-section"></a>단원 내용  
 [스레드 만들기 및 시작할 때 데이터 전달](../../../docs/standard/threading/creating-threads-and-passing-data-at-start-time.md)  
 에 대해 설명 하 고 데이터를 가져오는 방법을 다시 데이터 새 스레드를 전달 하는 방법을 비롯 하 여 관리 되는 스레드를 만드는 방법을 보여 줍니다.  
  
 [스레드 일시 중지 및 다시 시작](../../../docs/standard/threading/pausing-and-resuming-threads.md)  
 일시 중지 하 고 관리 되는 스레드를 다시 시작이 발생 하는 결과 설명 합니다.  
  
 [스레드 제거](../../../docs/standard/threading/destroying-threads.md)  
 없애는 관리 되는 스레드 및 처리 하는 방법에 설명 된 <xref:System.Threading.ThreadAbortException>합니다.  
  
 [스레드 스케줄링](../../../docs/standard/threading/scheduling-threads.md)  
 스레드 우선 순위 및 스레드 일정에 어떻게 영향을 설명 합니다.  
  
## <a name="reference"></a>참조  
 <xref:System.Threading.Thread>  
 에 대 한 참조 설명서를 제공는 <xref:System.Threading.Thread> 비관리 코드에서 가져왔는지 또는 관리 되는 응용 프로그램에서 만들어진 관리 되는 스레드를 나타내는 클래스입니다.  
  
 <xref:System.Threading.ThreadStart>  
 에 대 한 참조 설명서를 제공는 <xref:System.Threading.ThreadStart> 매개 변수가 없는 스레드 프로시저를 나타내는 대리자입니다.  
  
 <xref:System.Threading.ParameterizedThreadStart>  
 수행 하지 않고도 강력한 형식 지정 스레드 프로시저에 데이터를 전달 하는 쉬운 방법을 제공 합니다.
