## <a name="when-to-use-multiple-threads"></a><span data-ttu-id="1447d-118">다중 스레드 사용 시기</span><span class="sxs-lookup"><span data-stu-id="1447d-118">When to use multiple threads</span></span>

<span data-ttu-id="1447d-119">다중 스레딩을 사용하여 애플리케이션의 응답성을 높이고, 다중 프로세서 또는 다중 코어 시스템을 활용하여 애플리케이션의 처리량을 늘릴 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="1447d-119">You use multiple threads to increase the responsiveness of your application and to take advantage of a multiprocessor or multi-core system to increase the application's throughput.</span></span>

<span data-ttu-id="1447d-123">프로그램에서 병렬로 수행될 수 있는 작업을 수행하는 경우 별도 스레드에서 해당 작업을 수행하고 다중 프로세서 또는 다중 코어 시스템에서 프로그램을 실행하여 총 실행 시간을 줄일 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="1447d-123">If your program performs operations that can be done in parallel, the total execution time can be decreased by performing those operations in separate threads and running the program on a multiprocessor or multi-core system.</span></span> <span data-ttu-id="1447d-124">이러한 시스템에서 다중 스레드를 사용하면 응답성이 함께 증가하며 처리량도 증가할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="1447d-124">On such a system, use of multithreading might increase throughput along with the increased responsiveness.</span></span>
