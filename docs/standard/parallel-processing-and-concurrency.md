# Parallel Processing and Concurrency in the .NET Framework
.NET Framework는 응용 프로그램이 사용자에게 반응성을 유지하면서 사용자 컴퓨터의 성능을 극대화하기 위해 여러 실행 스레드를 사용하는 여러 가지 방법을 제공합니다.  
  
## 단원 내용  
 [Threading](../../docs/standard/threading/index.md)  
 .NET Framework에서 제공되는 기본 동시성 및 동기화 메커니즘을 설명합니다.  
  
 [Asynchronous Programming Patterns](../../docs/standard/asynchronous-programming-patterns/index.md)  
 .NET Framework에서 지원되는 세 가지 비동기 프로그래밍 패턴을 간단히 소개를 합니다.  
  
-   [Asynchronous Programming Model \(APM\)](../../docs/standard/asynchronous-programming-patterns/asynchronous-programming-model-apm.md)\(레거시\)  
  
-   [Event\-based Asynchronous Pattern \(EAP\)](../../docs/standard/asynchronous-programming-patterns/event-based-asynchronous-pattern-eap.md)\(레거시\)  
  
-   [Task\-based Asynchronous Pattern \(TAP\)](../../docs/standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md)은 새로운 개발 시 권장됩니다.  
  
 [Parallel Programming](../../docs/standard/parallel-programming/index.md)  
 개발자는 스레드 또는 스레드 풀을 직접 건드릴 필요 없이 효율적이고 세부적이고 확장 가능한 병렬 코드를 자연스러운 언어로 작성할 수 있도록 병렬 개발을 단순화하는 작업 기반 프로그래밍 모델을 설명합니다.
