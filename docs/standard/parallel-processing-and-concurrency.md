# <a name="parallel-processing-and-concurrency-in-the-net-framework"></a>.NET Framework의 병렬 처리 및 동시성
.NET Framework는 다중 스레드 방식의 실행을 사용하여 사용자에 대한 응용 프로그램 응답성을 유지하면서 사용자 컴퓨터의 성능을 극대화할 수 있는 여러 가지 방법을 제공합니다.  
  
## <a name="in-this-section"></a>단원 내용  
 [스레딩](../../docs/standard/threading/index.md)  
 .NET Framework에서 제공하는 기본 동시성 및 동기화 메커니즘에 대해 설명합니다.  
  
 [비동기 프로그래밍 패턴](../../docs/standard/asynchronous-programming-patterns/index.md)  
 .NET Framework에서 지원되는 세 가지 비동기 프로그래밍 패턴에 대해 간략하게 설명합니다.  
  
-   [APM(비동기 프로그래밍 모델)](../../docs/standard/asynchronous-programming-patterns/asynchronous-programming-model-apm.md)(레거시)  
  
-   [EAP(이벤트 기반 비동기 패턴)](../../docs/standard/asynchronous-programming-patterns/event-based-asynchronous-pattern-eap.md)(레거시)  
  
-   [TAP(작업 기반 비동기 패턴)](../../docs/standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md)(새로운 개발에 권장)  
  
 [병렬 프로그래밍](../../docs/standard/parallel-programming/index.md)  
 병렬 개발을 단순화하여 스레드 또는 스레드 풀을 직접 건드릴 필요 없이 효율적이고 세부적이며 확장 가능한 병렬 코드를 자연스러운 언어로 작성할 수 있는 작업 기반 프로그래밍 모델에 대해 설명합니다.
