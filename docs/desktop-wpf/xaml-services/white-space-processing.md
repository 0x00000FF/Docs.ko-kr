<span data-ttu-id="8ef29-160">또한 흐름 문서 모델에서 줄 바선을 나타내는 특정 인라인 요소는 공백 중요한 컬렉션에서도 의도적으로 여분의 공간을 도입하지 않아야 합니다.</span><span class="sxs-lookup"><span data-stu-id="8ef29-160">Also, certain inline elements that connote a line break in a flow document model should deliberately not introduce an extra space even in a white-space significant collection.</span></span> <span data-ttu-id="8ef29-161">예를 들어 <xref:System.Windows.Documents.LineBreak> 요소는 HTML에서 \<BR/> 태그와 동일한 목적을 가지며 태그의 가독성을 위해 일반적으로 a는 <xref:System.Windows.Documents.LineBreak> 작성된 줄 피드에 의해 후속 텍스트와 분리됩니다.</span><span class="sxs-lookup"><span data-stu-id="8ef29-161">For example, the <xref:System.Windows.Documents.LineBreak> element has the same purpose as the \<BR/> tag in HTML, and for readability in markup, typically a <xref:System.Windows.Documents.LineBreak> is separated from any subsequent text by an authored linefeed.</span></span> <span data-ttu-id="8ef29-162">후속 줄의 선행 공백이 될 수 있으므로 줄 바꿈을 정규화하면 안 됩니다.</span><span class="sxs-lookup"><span data-stu-id="8ef29-162">That linefeed should not be normalized to become a leading space in the subsequent line.</span></span> <span data-ttu-id="8ef29-163">해당 동작을 활성화하기 위해 <xref:System.Windows.Documents.LineBreak> 요소에 대한 <xref:System.Windows.Markup.TrimSurroundingWhitespaceAttribute>클래스 정의는 를 [!INCLUDE[TLA2#tla_xaml](../../../includes/tla2sharptla-xaml-md.md)] 적용한 다음 프로세서에서 해석하여 주변의 <xref:System.Windows.Documents.LineBreak> 공백이 항상 잘립니다.</span><span class="sxs-lookup"><span data-stu-id="8ef29-163">To enable that behavior, the class definition for the <xref:System.Windows.Documents.LineBreak> element applies the <xref:System.Windows.Markup.TrimSurroundingWhitespaceAttribute>, which is then interpreted by the [!INCLUDE[TLA2#tla_xaml](../../../includes/tla2sharptla-xaml-md.md)] processor to mean that white space surrounding <xref:System.Windows.Documents.LineBreak> is always trimmed.</span></span>
