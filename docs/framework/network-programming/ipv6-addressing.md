-   <span data-ttu-id="35a3d-113">**압축된 형식**.</span><span class="sxs-lookup"><span data-stu-id="35a3d-113">**Compressed form**.</span></span> <span data-ttu-id="35a3d-114">주소 길이 때문에 일반적으로 0으로 구성된 긴 문자열이 있는 주소를 포함합니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-114">Due to the address length, it is common to have addresses containing a long string of zeros.</span></span> <span data-ttu-id="35a3d-115">이러한 주소 작성을 간소화하려면 0 블록의 단일 연속 시퀀스를 이중 콜론 기호(::)로 표현하는 압축된 형식을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-115">To simplify writing these addresses, use the compressed form, in which a single contiguous sequence of 0 blocks are represented by a double-colon symbol (::).</span></span> <span data-ttu-id="35a3d-116">이 기호는 주소에서 한 번만 표시될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-116">This symbol can appear only once in an address.</span></span> <span data-ttu-id="35a3d-117">예를 들어 압축된 형식의 멀티캐스트 주소 `FFED:0:0:0:0:BA98:3210:4562`는 `FFED::BA98:3210:4562`입니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-117">For example, the multicast address `FFED:0:0:0:0:BA98:3210:4562` in compressed form is `FFED::BA98:3210:4562`.</span></span> <span data-ttu-id="35a3d-118">압축된 형식의 유니캐스트 주소 `3FFE:FFFF:0:0:8:800:20C4:0`은 `3FFE:FFFF::8:800:20C4:0`입니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-118">The unicast address `3FFE:FFFF:0:0:8:800:20C4:0` in compressed form is `3FFE:FFFF::8:800:20C4:0`.</span></span> <span data-ttu-id="35a3d-119">압축된 형식의 루프백 주소 `0:0:0:0:0:0:0:1`은 `::`1입니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-119">The loopback address `0:0:0:0:0:0:0:1` in compressed form is `::`1.</span></span> <span data-ttu-id="35a3d-120">압축된 형식의 지정되지 않은 주소 `0:0:0:0:0:0:0:0`은 `::`입니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-120">The unspecified address `0:0:0:0:0:0:0:0` in compressed form is `::`.</span></span>  
  
-   <span data-ttu-id="35a3d-121">**혼합된 형식**.</span><span class="sxs-lookup"><span data-stu-id="35a3d-121">**Mixed form**.</span></span> <span data-ttu-id="35a3d-122">이 형식은 IPv4 주소와 IPv6 주소를 결합합니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-122">This form combines IPv4 and IPv6 addresses.</span></span> <span data-ttu-id="35a3d-123">이 경우 주소 형식은 n:n:n:n:n:n:d.d.d.d입니다. 여기서 각 n은 6개의 IPv6 높은 순서 16비트 주소 요소의 16진수 값을 나타내고 각 d는 IPv4 주소의 10진수 값을 나타냅니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-123">In this case, the address format is n:n:n:n:n:n:d.d.d.d, where each n represents the hexadecimal values of the six IPv6 high-order 16-bit address elements, and each d represents the decimal value of an IPv4 address.</span></span>  
  
## <a name="address-types"></a><span data-ttu-id="35a3d-124">주소 형식</span><span class="sxs-lookup"><span data-stu-id="35a3d-124">Address Types</span></span>

 <span data-ttu-id="35a3d-130">64비트 접두사가 포함된 주소의 예는 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-130">The following is an example of an address with a 64-bit prefix.</span></span>  
  
 <span data-ttu-id="35a3d-131">`3FFE:FFFF:0:CD30:0:0:0:0/64`.</span><span class="sxs-lookup"><span data-stu-id="35a3d-131">`3FFE:FFFF:0:CD30:0:0:0:0/64`.</span></span>  
  
 <span data-ttu-id="35a3d-132">이 예의 접두사는 `3FFE:FFFF:0:CD30`입니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-132">The prefix in this example is `3FFE:FFFF:0:CD30`.</span></span> <span data-ttu-id="35a3d-133">주소는 압축된 형식으로 기록될 수도 있습니다(예: `3FFE:FFFF:0:CD30::/64`).</span><span class="sxs-lookup"><span data-stu-id="35a3d-133">The address can also be written in a compressed form, as `3FFE:FFFF:0:CD30::/64`.</span></span>  
  
 <span data-ttu-id="35a3d-134">IPv6은 다음 주소 형식을 정의합니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-134">IPv6 defines the following address types:</span></span>  
  
-   <span data-ttu-id="35a3d-135">**유니캐스트 주소**.</span><span class="sxs-lookup"><span data-stu-id="35a3d-135">**Unicast address**.</span></span> <span data-ttu-id="35a3d-136">단일 인터페이스의 식별자입니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-136">An identifier for a single interface.</span></span> <span data-ttu-id="35a3d-137">이 주소에 전송된 패킷은 식별된 인터페이스에 전달됩니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-137">A packet sent to this address is delivered to the identified interface.</span></span> <span data-ttu-id="35a3d-138">유니캐스트 주소는 높은 순서 8진수 값으로 멀티캐스트 주소와 구별됩니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-138">The unicast addresses are distinguished from the multicast addresses by the value of the high-order octet.</span></span> <span data-ttu-id="35a3d-139">멀티캐스트 주소의 높은 순서 8진수에는 16진수 값 FF가 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-139">The multicast addresses' high-order octet has the hexadecimal value of FF.</span></span> <span data-ttu-id="35a3d-140">이 8진수 식별자의 다른 값은 유니캐스트 주소를 식별합니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-140">Any other value for this octet identifies a unicast address.</span></span> <span data-ttu-id="35a3d-141">다양한 형식의 유니캐스트 주소는 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-141">The following are different types of unicast addresses:</span></span>  
  
    -   <span data-ttu-id="35a3d-142">**링크-로컬 주소**.</span><span class="sxs-lookup"><span data-stu-id="35a3d-142">**Link-local addresses**.</span></span> <span data-ttu-id="35a3d-143">이러한 주소는 단일 링크에서 사용되고 다음 형식을 사용합니다. FE80::*InterfaceID*.</span><span class="sxs-lookup"><span data-stu-id="35a3d-143">These addresses are used on a single link and have the following format: FE80::*InterfaceID*.</span></span> <span data-ttu-id="35a3d-144">링크-로컬 주소는 자동 주소 구성, 네트워크 환경 검색을 위해 또는 라우터가 없는 경우 링크에서 노드 간에 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-144">Link-local addresses are used between nodes on a link for auto-address configuration, neighbor discovery, or when no routers are present.</span></span> <span data-ttu-id="35a3d-145">링크-로컬 주소는 주로 시작 시 사용되고 시스템이 더 큰 범위의 주소를 아직 획득하지 못한 경우 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="35a3d-145">A link-local address is used primarily at startup and when the system has not yet acquired addresses of larger scope.</span></span>
