 <span data-ttu-id="9699f-146">`Sort(IComparer)` 메서드 오버 로드는 <xref:System.Collections.IComparer> 인터페이스를 매개 변수로 구현 하는 클래스의 인스턴스를 사용 합니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-146">The `Sort(IComparer)` method overload takes an instance of a class that implements the <xref:System.Collections.IComparer> interface as a parameter.</span></span> <span data-ttu-id="9699f-147">이 오버 로드는 사용자 지정 정렬을 제공 하려는 경우에 유용 합니다. 예를 들어 열에 있는 값에 자연 정렬 순서가 없거나 자연 정렬 순서가 부적절 한 경우입니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-147">This overload is useful when you want to provide custom sorting; for example, when the values in a column do not have a natural sort order or when the natural sort order is inappropriate.</span></span> <span data-ttu-id="9699f-148">이 경우 자동 정렬을 사용할 수 없지만 열 머리글을 클릭 하 여 사용자가 정렬 하도록 할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-148">In this case, you cannot use automatic sorting, but you might still want your users to sort by clicking the column headers.</span></span> <span data-ttu-id="9699f-149">선택 영역에 열 머리글을 사용 하지 않는 경우 <xref:System.Windows.Forms.DataGridView.ColumnHeaderMouseClick> 이벤트에 대 한 처리기에서이 오버 로드를 호출할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-149">You can call this overload in a handler for the <xref:System.Windows.Forms.DataGridView.ColumnHeaderMouseClick> event if you do not use column headers for selection.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="9699f-150">`Sort(IComparer)` 메서드 오버 로드는 <xref:System.Windows.Forms.DataGridView> 컨트롤이 외부 데이터 원본에 바인딩되지 않고 <xref:System.Windows.Forms.DataGridView.VirtualMode%2A> 속성 값이 `false`되는 경우에만 작동 합니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-150">The `Sort(IComparer)` method overload works only when the <xref:System.Windows.Forms.DataGridView> control is not bound to an external data source and the <xref:System.Windows.Forms.DataGridView.VirtualMode%2A> property value is `false`.</span></span> <span data-ttu-id="9699f-151">외부 데이터 원본에 바인딩된 열의 정렬을 사용자 지정 하려면 데이터 원본에서 제공 하는 정렬 작업을 사용 해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-151">To customize sorting for columns bound to an external data source, you must use the sorting operations provided by the data source.</span></span> <span data-ttu-id="9699f-152">가상 모드에서는 바인딩되지 않은 열에 대 한 고유한 정렬 작업을 제공 해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-152">In virtual mode, you must provide your own sorting operations for unbound columns.</span></span>  
  
 <span data-ttu-id="9699f-153">`Sort(IComparer)` 메서드 오버 로드를 사용 하려면 <xref:System.Collections.IComparer> 인터페이스를 구현 하는 고유한 클래스를 만들어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-153">To use the `Sort(IComparer)` method overload, you must create your own class that implements the <xref:System.Collections.IComparer> interface.</span></span> <span data-ttu-id="9699f-154">이 인터페이스를 사용 하려면 클래스에서 <xref:System.Collections.IComparer.Compare%2A?displayProperty=nameWithType> 메서드를 구현 해야 합니다 .이 메서드는 <xref:System.Windows.Forms.DataGridView> `Sort(IComparer)` 메서드 오버 로드가 호출 될 때 개체 <xref:System.Windows.Forms.DataGridViewRow> 개체를 입력으로 전달 합니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-154">This interface requires your class to implement the <xref:System.Collections.IComparer.Compare%2A?displayProperty=nameWithType> method, to which the <xref:System.Windows.Forms.DataGridView> passes <xref:System.Windows.Forms.DataGridViewRow> objects as input when the `Sort(IComparer)` method overload is called.</span></span> <span data-ttu-id="9699f-155">이를 통해 모든 열의 값을 기준으로 올바른 행 순서를 계산할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-155">With this, you can calculate the correct row ordering based on the values in any column.</span></span>  
  
 <span data-ttu-id="9699f-156">`Sort(IComparer)` 메서드 오버 로드는 <xref:System.Windows.Forms.DataGridView.SortedColumn%2A> 및 <xref:System.Windows.Forms.DataGridView.SortOrder%2A> 속성을 설정 하지 않으므로 항상 <xref:System.Windows.Forms.DataGridViewColumnHeaderCell.SortGlyphDirection%2A?displayProperty=nameWithType> 속성을 설정 하 여 정렬 문자 모양을 표시 해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-156">The `Sort(IComparer)` method overload does not set the <xref:System.Windows.Forms.DataGridView.SortedColumn%2A> and <xref:System.Windows.Forms.DataGridView.SortOrder%2A> properties, so you must always set the <xref:System.Windows.Forms.DataGridViewColumnHeaderCell.SortGlyphDirection%2A?displayProperty=nameWithType> property to display the sorting glyph.</span></span>  
  
 <span data-ttu-id="9699f-157">`Sort(IComparer)` 메서드 오버 로드에 대 한 대 안으로 <xref:System.Windows.Forms.DataGridView.SortCompare> 이벤트에 대 한 처리기를 구현 하 여 사용자 지정 정렬을 제공할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-157">As an alternative to the `Sort(IComparer)` method overload, you can provide custom sorting by implementing a handler for the <xref:System.Windows.Forms.DataGridView.SortCompare> event.</span></span> <span data-ttu-id="9699f-158">이 이벤트는 사용자가 자동 정렬을 위해 구성 된 열의 머리글을 클릭 하거나 <xref:System.Windows.Forms.DataGridView.Sort%2A> 메서드의 `Sort(DataGridViewColumn,ListSortDirection)` 오버 로드를 호출할 때 발생 합니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-158">This event occurs when users click the headers of columns configured for automatic sorting or when you call the `Sort(DataGridViewColumn,ListSortDirection)` overload of the <xref:System.Windows.Forms.DataGridView.Sort%2A> method.</span></span> <span data-ttu-id="9699f-159">이 이벤트는 컨트롤의 각 행 쌍에 대해 발생 하므로 올바른 순서를 계산할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9699f-159">The event occurs for each pair of rows in the control, enabling you to calculate their correct order.</span></span>
