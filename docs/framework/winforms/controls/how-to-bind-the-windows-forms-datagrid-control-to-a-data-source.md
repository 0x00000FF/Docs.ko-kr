### <a name="to-data-bind-the-datagrid-control-programmatically"></a><span data-ttu-id="e6d0a-108">데이터 바인딩 DataGrid 컨트롤 프로그래밍 방식으로 하려면</span><span class="sxs-lookup"><span data-stu-id="e6d0a-108">To data-bind the DataGrid control programmatically</span></span>  
  
1.  <span data-ttu-id="e6d0a-109">데이터 집합을 채우는 코드를 작성 합니다.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-109">Write code to fill the dataset.</span></span>  
  
     <span data-ttu-id="e6d0a-110">이면 데이터 원본이 데이터 집합 또는 데이터 집합 테이블을 기반으로 데이터 뷰를 폼에 데이터 집합을 채우는 코드를 추가 합니다.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-110">If the data source is a dataset or a data view based on a dataset table, add code to the form to fill the dataset.</span></span>  
  
     <span data-ttu-id="e6d0a-111">사용할 올바른 코드는 데이터 집합 데이터를 가져오는 위치에 따라 달라 집니다.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-111">The exact code you use depends on where the dataset is getting data.</span></span> <span data-ttu-id="e6d0a-112">일반적으로 호출 하는 데이터베이스에서 직접 데이터 집합을 채우는 경우는 `Fill` 메서드 라는 데이터 집합을 채우는 다음 예제와 같이 데이터 어댑터의 `DsCategories1`:</span><span class="sxs-lookup"><span data-stu-id="e6d0a-112">If the dataset is being populated directly from a database, you typically call the `Fill` method of a data adapter, as in the following example, which populates a dataset called `DsCategories1`:</span></span>  
  
    ```vb  
    sqlDataAdapter1.Fill(DsCategories1)  
    ```  
  
    ```csharp  
    sqlDataAdapter1.Fill(DsCategories1);  
    ```  
  
    ```cpp  
    sqlDataAdapter1->Fill(dsCategories1);  
    ```  
  
     <span data-ttu-id="e6d0a-113">데이터 집합, XML 웹 서비스에서 채워진 경우 일반적으로 코드에는 서비스의 인스턴스를 만들 하 고 데이터 집합을 반환 하는 방법 중 하나를 호출 합니다.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-113">If the dataset is being filled from an XML Web service, you typically create an instance of the service in your code and then call one of its methods to return a dataset.</span></span> <span data-ttu-id="e6d0a-114">그런 다음 로컬 데이터 집합에 XML 웹 서비스에서 데이터 집합을 병합합니다.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-114">You then merge the dataset from the XML Web service into your local dataset.</span></span> <span data-ttu-id="e6d0a-115">다음 예제에서는 호출 하는 XML 웹 서비스의 인스턴스를 만드는 방법을 보여 줍니다. `CategoriesService`, 호출의 `GetCategories` 메서드를 호출 하는 로컬 데이터 집합으로 결과 데이터 집합 병합 `DsCategories1`:</span><span class="sxs-lookup"><span data-stu-id="e6d0a-115">The following example shows how you can create an instance of an XML Web service called `CategoriesService`, call its `GetCategories` method, and merge the resulting dataset into a local dataset called `DsCategories1`:</span></span>  
  
    ```vb

    ws.Credentials = System.Net.CredentialCache.DefaultCredentials  
    DsCategories1.Merge(ws.GetCategories())  
    ```  
  
    ```csharp  
    MyProject.localhost.CategoriesService ws = new MyProject.localhost.CategoriesService();  
    ws.Credentials = System.Net.CredentialCache.DefaultCredentials;  
    DsCategories1.Merge(ws.GetCategories());  
    ```  
  
    ```cpp  
    MyProject::localhost::CategoriesService^ ws =   
       new MyProject::localhost::CategoriesService();  
    ws->Credentials = System::Net::CredentialCache::DefaultCredentials;  
    dsCategories1->Merge(ws->GetCategories());  
    ```  
  
2.  <span data-ttu-id="e6d0a-116">호출 된 <xref:System.Windows.Forms.DataGrid> 컨트롤의 <xref:System.Windows.Forms.DataGrid.SetDataBinding%2A> 메서드를 데이터 원본 및 데이터 멤버에 전달 합니다.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-116">Call the <xref:System.Windows.Forms.DataGrid> control's <xref:System.Windows.Forms.DataGrid.SetDataBinding%2A> method, passing it the data source and a data member.</span></span> <span data-ttu-id="e6d0a-117">데이터 멤버를 명시적으로 전달 하도록 필요 하지 않은 경우 빈 문자열을 전달 합니다.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-117">If you do not need to explicitly pass a data member, pass an empty string.</span></span>  
  
    > [!NOTE]
    >  <span data-ttu-id="e6d0a-118">컨트롤의 눈금에 처음으로 바인딩하는 경우 설정할 수 있습니다 <xref:System.Windows.Forms.DataGrid.DataSource%2A> 및 <xref:System.Windows.Forms.DataGrid.DataMember%2A> 속성입니다.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-118">If you are binding the grid for the first time, you can set the control's <xref:System.Windows.Forms.DataGrid.DataSource%2A> and <xref:System.Windows.Forms.DataGrid.DataMember%2A> properties.</span></span> <span data-ttu-id="e6d0a-119">그러나 설정 된 후 이러한 속성을 재설정할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-119">However, you cannot reset these properties once they have been set.</span></span> <span data-ttu-id="e6d0a-120">따라서 것이 좋습니다 항상 사용 하 여 <xref:System.Windows.Forms.DataGrid.SetDataBinding%2A> 메서드.</span><span class="sxs-lookup"><span data-stu-id="e6d0a-120">Therefore, it is recommended that you always use the <xref:System.Windows.Forms.DataGrid.SetDataBinding%2A> method.</span></span>  
  
     <span data-ttu-id="e6d0a-121">다음 예제에서는 프로그래밍 방식으로 바인딩하는 방법을 라는 데이터 집합의 Customers 테이블에 `DsCustomers1`:</span><span class="sxs-lookup"><span data-stu-id="e6d0a-121">The following example shows how you can programmatically bind to the Customers table in a dataset called `DsCustomers1`:</span></span>  
  
    ```vb  
    DataGrid1.SetDataBinding(DsCustomers1, "Customers")  
    ```  
  
    ```csharp  
    DataGrid1.SetDataBinding(DsCustomers1, "Customers");  
    ```  
  
    ```cpp  
    dataGrid1->SetDataBinding(dsCustomers1, "Customers");  
    ```  
  
     <span data-ttu-id="e6d0a-122">데이터 집합에 Customers 테이블을 사용 하는 경우 바인딩할 수도 있습니다. 표에서 이러한 방식으로:</span><span class="sxs-lookup"><span data-stu-id="e6d0a-122">If the Customers table is the only table in the dataset, you could alternatively bind the grid this way:</span></span>  
  
    ```vb  
    DataGrid1.SetDataBinding(DsCustomers1, "")  
    ```  
  
    ```csharp  
    DataGrid1.SetDataBinding(DsCustomers1, "");  
    ```  
  
    ```cpp  
    dataGrid1->SetDataBinding(dsCustomers1, "");
    ```  
