# .NET Framework 배포
.NET Framework 설명서의 이 섹션에서는 .NET Framework에 응용 프로그램을 설치할 개발자와 네트워크에서 .NET Framework 배포할 관리자를 위해 정보를 제공합니다.  그리고 정품 인증을 설명하고 배포와 .NET Framework 설치의 진행 상황을 모니터링 하는 방법과 관련된 문제를 다시 시작합니다.  
  
## 단원 내용  
 [개발자를 위한 배포 가이드](../../../docs/framework/deployment/deployment-guide-for-developers.md)  
 개발자가 .NET Framework를 응용 프로그램과 함께 사용자 컴퓨터에 설치할 수 있는 방법에 대해 설명합니다.  
  
 [관리자를 위한 배포 가이드](../../../docs/framework/deployment/guide-for-administrators.md)  
 시스템 관리자가 SCCM\(System Center Configuration Manager\)을 사용하여 .NET Framework 및 해당 시스템 종속성을 네트워크 전체에 배포할 수 있는 방법에 대해 설명합니다.  
  
 [.NET Framework 4.5를 설치하는 동안 시스템 다시 시작 줄이기](../../../docs/framework/deployment/reducing-system-restarts.md)  
 가능할 때마다 다시 부팅되는 것을 방지하는 다시 시작 관리자를 설명하고 .NET Framework를 설치하는 응용 프로그램에서 이용할 수 있는 방법을 설명합니다.  
  
 [방법: .NET Framework 4.5 설치 관리자에서 진행률 가져오기](../../../docs/framework/deployment/how-to-get-progress-from-the-dotnet-installer.md)  
 설치 진행률을 자체적으로 보여주면서 .NET Framework 설치 프로세스를 자동으로 시작하고 추적하는 방법을 설명합니다.  
  
 [.NET Framework 초기화 오류: 사용자 환경 관리](../../../docs/framework/deployment/initialization-errors-managing-the-user-experience.md)  
 .NET Framework 응용 프로그램이 유효하지 않거나 사용자 컴퓨터에 설치되지 않은 CLR 버전을 필요로 할 경우 발생하는 오류에 대해 설명하고, 이 오류를 해결하는 방법과 오류 메시지를 제어하는 방법을 설명합니다.  
  
 [방법: CLR 활성화 문제 디버깅](../../../docs/framework/deployment/how-to-debug-clr-activation-issues.md)  
 올바른 버전의 CLR\(공용 언어 런타임\)을 사용하여 실행한 응용 프로그램에서 문제가 발생한 경우 CLR 활성화 로그를 보고 디버깅하는 방법을 설명합니다.
