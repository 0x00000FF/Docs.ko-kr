# <a name="application-compatibility-in-the-net-framework"></a>.NET Framework의 응용 프로그램 호환성
다음 문서에서는 최신 버전의 .NET Framework에서의 응용 프로그램 호환성 문제에 대해 설명합니다.  
  
-   버전 4와 4.5 간: [4.5의 응용 프로그램 호환성](../../../docs/framework/migration-guide/application-compatibility-in-the-net-framework-4-5.md)  
  
-   버전 4.5와 4.5.1 간: [4.5.1의 응용 프로그램 호환성](../../../docs/framework/migration-guide/application-compatibility-in-the-net-framework-4-5-1.md)  
  
-   버전 4.5.1과 4.5.2 간: [4.5.2의 응용 프로그램 호환성](../../../docs/framework/migration-guide/application-compatibility-in-the-net-framework-4-5-2.md)  
  
-   버전 4.5.2와 4.6 간: [4.6의 응용 프로그램 호환성](../../../docs/framework/migration-guide/application-compatibility-in-the-net-framework-4-6.md)  
  
-   버전 4.6과 [!INCLUDE[net_v461](../../../includes/net-v461-md.md)] 간: [4.6.1의 응용 프로그램 호환성](../../../docs/framework/migration-guide/application-compatibility-in-the-net-framework-4-6-1.md)  
  
-   버전 4.6.1과 [!INCLUDE[net_v462](../../../includes/net-v462-md.md)] 간: [4.6.2의 응용 프로그램 호환성](../../../docs/framework/migration-guide/application-compatibility-in-the-net-framework-4-6-2.md)  

- 버전 4.6.2와 .NET Framework 4.7 간: [4.7의 응용 프로그램 호환성](../../../docs/framework/migration-guide/application-compatibility-in-the-net-framework-4-7.md)  

일반적으로 .NET Framework 버전 간의 고수준 역호환성이 유지되도록 하였으므로 특정 버전 대상의 앱은 후속 버전에서 수정하지 않고 실행됩니다. 대부분의 응용 프로그램 호환성 문제는 미미한 영향을 주며 앱에서 프로그래밍을 수정할 필요가 없습니다. 수정이 필요한 변경 내용은 다음 링크의 표에서 영향 열을 참조하세요.
