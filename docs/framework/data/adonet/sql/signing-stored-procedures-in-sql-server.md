 <span data-ttu-id="0e7b1-103">디지털 설명은 서명자의 개인 키로 암호화된 데이터 다이제스트입니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-103">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="0e7b1-104">개인 키를 사용하면 디지털 서명이 소유자별로 고유하게 유지됩니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-104">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="0e7b1-105">저장된 프로시저, (인라인 테이블 반환 함수)를 제외한 함수, 트리거 및 어셈블리를 서명할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-105">You can sign stored procedures, functions (except for inline table-valued functions), triggers, and assemblies.</span></span>  
  
 <span data-ttu-id="0e7b1-106">인증서나 비대칭 키를 사용하여 저장 프로시저에 서명할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-106">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="0e7b1-107">이 기능은 권한을 소유권 체인을 통해 상속할 수 없거나 동적 SQL과 같이 소유권 체인이 끊어진 시나리오를 위해 디자인되었습니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-107">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="0e7b1-108">그런 다음 인증서에 액세스 해야 하는 저장된 프로시저 개체에 사용자 권한 부여 인증서에 매핑된 사용자를 만들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-108">You can then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  

 <span data-ttu-id="0e7b1-109">동일한 인증서에 매핑된 로그인을 만들려면 및 그런 다음 해당 로그인에 필요한 서버 수준 사용 권한을 부여 하거나 수도 로그인 하나 이상의 고정된 서버 역할을 추가 합니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-109">You can also create a login mapped to the same certificate, and then grant any necessary server-level permissions to that login, or add the login to one or more of the fixed server roles.</span></span> <span data-ttu-id="0e7b1-110">사용을 방지 하도록 설계 된이 `TRUSTWORTHY` 데이터베이스는 높은 권한 수준이 필요한 시나리오에 대 한 설정입니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-110">This is designed to avoid enabling the `TRUSTWORTHY` database setting for scenarios in which higher level permissions are needed.</span></span>

## <a name="creating-certificates"></a><span data-ttu-id="0e7b1-114">인증서 만들기</span><span class="sxs-lookup"><span data-stu-id="0e7b1-114">Creating Certificates</span></span>  
 <span data-ttu-id="0e7b1-115">인증서 또는 비대칭 키의 저장된 프로시저 코드에서 execute와 함께 암호화 된 해시로 구성 된 데이터 다이제스트를 사용 하 여 저장된 프로시저에 서명할 때-개인 키를 사용 하 여 만든는 사용자입니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-115">When you sign a stored procedure with a certificate or asymmetric key, a data digest consisting of the encrypted hash of the stored procedure code, along with the execute-as user, is created using the private key.</span></span> <span data-ttu-id="0e7b1-116">런타임에 데이터 다이제스트는 공개 키로 암호 해독되어 저장 프로시저의 해시 값과 비교됩니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-116">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="0e7b1-117">Execute 변경-으로 디지털 서명은 더 이상 일치 하도록 사용자에 해시 값을 무효화 합니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-117">Changing the execute-as user invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="0e7b1-118">저장된 프로시저를 수정 삭제 서명을 완전히 방지 하는 저장된 프로시저 코드 변경에서 개인 키에 대 한 액세스 권한이 있는 사람입니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-118">Modifying the stored procedure drops the signature entirely, which prevents someone who does not have access to the private key from changing the stored procedure code.</span></span> <span data-ttu-id="0e7b1-119">두 경우 모두 다시 서명 해야 프로시저 코드 또는 execute를 변경할 때마다-사용자로 합니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-119">In either case, you must re-sign the procedure each time you change the code or the execute-as user.</span></span>  
  
 <span data-ttu-id="0e7b1-120">두 가지 필수 단계는 모듈에 서명할 때는:</span><span class="sxs-lookup"><span data-stu-id="0e7b1-120">There are two required steps involved in signing a module:</span></span>  
  
1.  <span data-ttu-id="0e7b1-121">Transact-SQL `CREATE CERTIFICATE [certificateName]` 문을 사용하여 인증서를 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-121">Create a certificate using the Transact-SQL `CREATE CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="0e7b1-122">이 문에는 시작 및 종료 날짜와 암호를 설정하기 위한 몇 가지 옵션이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-122">This statement has several options for setting a start and end date and a password.</span></span> <span data-ttu-id="0e7b1-123">기본 만료 날짜는 1 년입니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-123">The default expiration date is one year.</span></span>

<span data-ttu-id="0e7b1-125">모듈 서명 된 후 하나 이상의 보안 주체는 인증서와 연결 해야 하는 추가 사용 권한을 보유 하기 위해 만들 수 해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-125">Once the module has been signed, one or more principals needs to be created in order to hold the additional permissions that should be associated with the certificate.</span></span>  

<span data-ttu-id="0e7b1-126">모듈 추가 데이터베이스 수준 권한이 필요로 하는 경우:</span><span class="sxs-lookup"><span data-stu-id="0e7b1-126">If the module needs additional database-level permissions:</span></span>

1.  <span data-ttu-id="0e7b1-129">인증서 사용자에 게 필요한 데이터베이스 수준 사용 권한을 부여 합니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-129">Grant the certificate user the required database-level permissions.</span></span>  
  
<span data-ttu-id="0e7b1-130">모듈 추가 서버 수준 권한이 필요 하는 경우:</span><span class="sxs-lookup"><span data-stu-id="0e7b1-130">If the module needs additional server-level permissions:</span></span>  
  
1.  <span data-ttu-id="0e7b1-131">인증서를 복사 합니다 `master` 데이터베이스입니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-131">Copy the certificate to the `master` database.</span></span>  
 
1.  <span data-ttu-id="0e7b1-132">TRANSACT-SQL을 사용 하 여 해당 인증서와 연결 된 로그인을 만들려면 `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` 문입니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-132">Create a login associated with that certificate using the Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` statement.</span></span>  
  
1.  <span data-ttu-id="0e7b1-133">인증서 로그인 필요한 서버 수준 사용 권한을 부여 합니다.</span><span class="sxs-lookup"><span data-stu-id="0e7b1-133">Grant the certificate login the required server-level permissions.</span></span>  
  
> [!NOTE]
