 <span data-ttu-id="f8237-106">클라이언트 또는 프레젠테이션 계층에서 중간 계층의 원격 인터페이스에 있는 메서드를 호출하면 중간 계층의 DAL에서는 <xref:System.Data.Linq.DataContext> 메서드에 매핑된 쿼리나 저장 프로시저를 실행합니다.</span><span class="sxs-lookup"><span data-stu-id="f8237-106">The client or presentation layer calls methods on the middle-tier's remote interface, and the DAL on that tier will execute queries or stored procedures that are mapped to <xref:System.Data.Linq.DataContext> methods.</span></span> <span data-ttu-id="f8237-107">중간 계층에서는 일반적으로 프록시 개체 또는 엔터티의 XML 표현으로 클라이언트에 데이터를 반환합니다.</span><span class="sxs-lookup"><span data-stu-id="f8237-107">The middle tier returns the data to clients typically as XML representations of entities or proxy objects.</span></span>

 <span data-ttu-id="f8237-113">ASP.NET 응용 프로그램에서는 이러한 복잡한 작업을 <xref:System.Web.UI.WebControls.LinqDataSource>에서 대부분 처리합니다.</span><span class="sxs-lookup"><span data-stu-id="f8237-113">In ASP.NET applications, the <xref:System.Web.UI.WebControls.LinqDataSource> manages most of this complexity.</span></span> <span data-ttu-id="f8237-114">자세한 내용은 [NIB: LinqDataSource 웹 서버 컨트롤 개요](https://msdn.microsoft.com/library/104cfc3f-7385-47d3-8a51-830dfa791136)합니다.</span><span class="sxs-lookup"><span data-stu-id="f8237-114">For more information, see [NIB: LinqDataSource Web Server Control Overview](https://msdn.microsoft.com/library/104cfc3f-7385-47d3-8a51-830dfa791136).</span></span>
