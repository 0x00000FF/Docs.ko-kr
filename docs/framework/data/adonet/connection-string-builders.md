## <a name="connection-string-injection-attacks"></a><span data-ttu-id="63554-110">연결 문자열 삽입 공격</span><span class="sxs-lookup"><span data-stu-id="63554-110">Connection String Injection Attacks</span></span>  
 <span data-ttu-id="63554-111">연결 문자열 삽입 공격은 사용자 입력에 대해 동적 문자열 연결을 사용하여 연결 문자열을 작성할 때 발생할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="63554-111">A connection string injection attack can occur when dynamic string concatenation is used to build connection strings that are based on user input.</span></span> <span data-ttu-id="63554-112">문자열의 유효성이 확인되지 않고 악의적인 텍스트 또는 문자를 이스케이프할 수 없는 경우 공격자가 서버에서 중요한 데이터 또는 기타 리소스에 액세스할 가능성이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="63554-112">If the string is not validated and malicious text or characters not escaped, an attacker can potentially access sensitive data or other resources on the server.</span></span> <span data-ttu-id="63554-113">예를 들어 공격자가 세미콜론을 입력한 다음 값을 추가하여 공격을 시작할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="63554-113">For example, an attacker could mount an attack by supplying a semicolon and appending an additional value.</span></span> <span data-ttu-id="63554-114">연결 문자열은 "최신 항목 사용" 알고리즘을 사용하여 구문 분석되므로 악의적인 입력이 올바른 값을 대체할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="63554-114">The connection string is parsed by using a "last one wins" algorithm, and the hostile input is substituted for a legitimate value.</span></span>  
  
 <span data-ttu-id="63554-115">연결 문자열 작성기 클래스는 모호한 작업을 제거하고 구문 오류 및 보안 취약점으로부터 보호하도록 디자인되었습니다.</span><span class="sxs-lookup"><span data-stu-id="63554-115">The connection string builder classes are designed to eliminate guesswork and protect against syntax errors and security vulnerabilities.</span></span> <span data-ttu-id="63554-116">즉, 각 데이터 공급자에서 허용하는 알려진 키/값 쌍에 해당하는 메서드와 속성을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="63554-116">They provide methods and properties corresponding to the known key/value pairs permitted by each data provider.</span></span> <span data-ttu-id="63554-117">각 클래스는 고정된 동의어 컬렉션을 유지하며 동의어를 그에 해당하는 잘 알려진 키 이름으로 변환할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="63554-117">Each class maintains a fixed collection of synonyms and can translate from a synonym to the corresponding well-known key name.</span></span> <span data-ttu-id="63554-118">또한, 유효한 키/값 쌍에 대한 검사를 수행하며 유효하지 않은 쌍이 있는 경우 예외를 throw합니다.</span><span class="sxs-lookup"><span data-stu-id="63554-118">Checks are performed for valid key/value pairs and an invalid pair throws an exception.</span></span> <span data-ttu-id="63554-119">뿐만 아니라 삽입된 값을 안전한 방식으로 처리합니다.</span><span class="sxs-lookup"><span data-stu-id="63554-119">In addition, injected values are handled in a safe manner.</span></span>  
  
 <span data-ttu-id="63554-120">다음 예제에서는 <xref:System.Data.SqlClient.SqlConnectionStringBuilder>에서 `Initial Catalog` 설정에 삽입된 추가 값을 처리하는 방법을 보여 줍니다.</span><span class="sxs-lookup"><span data-stu-id="63554-120">The following example demonstrates how the <xref:System.Data.SqlClient.SqlConnectionStringBuilder> handles an inserted extra value for the `Initial Catalog` setting.</span></span>

