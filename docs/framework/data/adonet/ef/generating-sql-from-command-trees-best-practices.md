## <a name="flatten-joins-in-a-sql-select-statement"></a><span data-ttu-id="08e13-119">SQL SELECT 문에서 조인 평면화</span><span class="sxs-lookup"><span data-stu-id="08e13-119">Flatten Joins in a SQL SELECT Statement</span></span>

<span data-ttu-id="08e13-120">여러 노드를 단일 SQL SELECT 문에 집계하는 예로 여러 조인 식을 단일 SQL SELECT 문에 집계하는 경우를 들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="08e13-120">One case of aggregating multiple nodes into a single SQL SELECT statement is aggregating multiple join expressions into a single SQL SELECT statement.</span></span> <span data-ttu-id="08e13-121">DbJoinExpression은 두 입력 간의 단일 조인을 나타냅니다.</span><span class="sxs-lookup"><span data-stu-id="08e13-121">DbJoinExpression represents a single join between two inputs.</span></span> <span data-ttu-id="08e13-122">그러나 단일 SQL SELECT 문의 일부로 둘 이상의 조인을 지정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="08e13-122">However, as part of a single SQL SELECT statement, more than one join can be specified.</span></span> <span data-ttu-id="08e13-123">이 경우 지정된 순서대로 조인이 수행됩니다.</span><span class="sxs-lookup"><span data-stu-id="08e13-123">In that case the joins are performed in the order specified.</span></span>

<span data-ttu-id="08e13-124">왼쪽 편 조인(다른 조인의 왼쪽 자식으로 나타나는 조인)은 단일 SQL SELECT 문으로 보다 쉽게 평면화할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="08e13-124">Left spine joins, (joins that appear as a left child of another join) can be more easily flattened into a single SQL SELECT statement.</span></span> <span data-ttu-id="08e13-125">예를 들어, 다음과 같은 쿼리 명령 트리를 살펴보겠습니다.</span><span class="sxs-lookup"><span data-stu-id="08e13-125">For example, consider the following query command tree:</span></span>

<span data-ttu-id="08e13-126">이 트리는 다음으로 올바르게 변환될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="08e13-126">This can be correctly translated into:</span></span>

<span data-ttu-id="08e13-127">그러나 왼쪽 편 조인이 아닌 조인은 쉽게 평면화할 수 없으므로 평면화하려고 하면 안 됩니다.</span><span class="sxs-lookup"><span data-stu-id="08e13-127">However, non-left spine joins cannot easily be flattened, and you should not try to flatten them.</span></span> <span data-ttu-id="08e13-128">다음 쿼리 명령 트리의 조인을 예로 들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="08e13-128">For example, the joins in the following query command tree:</span></span>

   d = Extent("TableC")
   ON c.y = d.x),
   ON a.z = b.c.y
)

   LEFT OUTER JOIN TableC as d
   ON c.y = d.x) as b
