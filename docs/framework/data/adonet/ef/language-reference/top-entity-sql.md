## <a name="remarks"></a><span data-ttu-id="d592d-109">설명</span><span class="sxs-lookup"><span data-stu-id="d592d-109">Remarks</span></span>  
 <span data-ttu-id="d592d-110">TOP 식은 단일 숫자 리터럴이거나 단일 매개 변수여야 합니다.</span><span class="sxs-lookup"><span data-stu-id="d592d-110">The TOP expression must be either a single numeric literal or a single parameter.</span></span> <span data-ttu-id="d592d-111">상수 리터럴이 사용되는 경우, 리터럴 형식은 Edm.Int64(바이트. int16, int32, int64이거나 Edm.Int64로 확장 가능한 형식으로 매핑되는 모든 공급자 형식)로의 암시적 확장이 가능해야 하며 그 값이 0이거나 0보다 커야 합니다.</span><span class="sxs-lookup"><span data-stu-id="d592d-111">If a constant literal is used, the literal type must be implicitly promotable to Edm.Int64 (byte, int16, int32 or int64 or any provider type that maps to a type that is promotable to Edm.Int64) and its value must be greater than or equal to zero.</span></span> <span data-ttu-id="d592d-112">그렇지 않으면 예외가 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="d592d-112">Otherwise an exception will be raised.</span></span> <span data-ttu-id="d592d-113">매개 변수가 식으로 사용되는 경우에는 매개 변수 형식도 역시 암시적으로 Edm.Int64로 확장할 수 있어야 합니다. 하지만 매개 변수 값은 런타임에 바인딩되므로 컴파일 과정에서 실제 매개 변수 값에 대한 확인은 이루어지지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="d592d-113">If a parameter is used as an expression, the parameter type must also be implicitly promotable to Edm.Int64, but there will be no validation of the actual parameter value during compilation because the parameter values are late bounded.</span></span>
