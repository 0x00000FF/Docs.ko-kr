2.  <span data-ttu-id="6c562-198">**프로젝트 디자이너**를 닫습니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-198">Close the **Project Designer**.</span></span> <span data-ttu-id="6c562-199">설정을 수정하지 않더라도 **서비스** 페이지에서는 App.config 파일이 자동으로 업데이트되기 때문입니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-199">This is necessary because the **Services** page will automatically update your App.config file even if you do not modify any settings.</span></span> <span data-ttu-id="6c562-200">이 절차에서 설명하는 대로 App.config 파일을 수동으로 수정한 후 **서비스** 페이지로 돌아가면 수정 내용이 다시 설정됩니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-200">If you manually modify your App.config file as described in this procedure and then return to the **Services** page, your modifications will be reset.</span></span>  
  
3.  <span data-ttu-id="6c562-201">**솔루션 탐색기**에서 App.config를 두 번 클릭합니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-201">In **Solution Explorer**, double-click App.config.</span></span>  
  
     <span data-ttu-id="6c562-202">응용 프로그램 구성 파일이 텍스트 편집기에서 열립니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-202">The application configuration file opens in the text editor.</span></span>  
  
4.  <span data-ttu-id="6c562-203">`<providers>` 또는 `<membership>` 요소 내에서 `<roleManager>` 요소를 찾습니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-203">Find the `<providers>` element within the `<membership>` or `<roleManager>` element.</span></span> <span data-ttu-id="6c562-204">이러한 요소는 `<system.web>` 요소의 자식입니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-204">These elements are children of the `<system.web>` element.</span></span> <span data-ttu-id="6c562-205">`<membership>` 요소는 인증 공급자를 지정하는 데 사용되고 `<roleManager>` 요소는 역할 공급자를 지정하는 데 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-205">The `<membership>` element is used to specify authentication providers, and the `<roleManager>` element is used to specify role providers.</span></span>  
  
5.  <span data-ttu-id="6c562-206">`<add>` 요소를 `<providers>` 요소의 자식으로 추가합니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-206">Add an `<add>` element as a child of the `<providers>` element.</span></span> <span data-ttu-id="6c562-207">다음 예제에 나와 있는 대로 `name` 및 `type` 특성을 지정해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-207">You must specify `name` and `type` attributes as shown in the following example.</span></span> <span data-ttu-id="6c562-208">`type` 특성 값은 정규화된 어셈블리 형식 이름이어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-208">The `type` attribute value must be an assembly-qualified type name.</span></span> <span data-ttu-id="6c562-209">자세한 내용은 <xref:System.Type.AssemblyQualifiedName%2A?displayProperty=nameWithType> 및 [어셈블리 이름](../../../docs/framework/app-domains/assembly-names.md)을 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="6c562-209">For more information, see <xref:System.Type.AssemblyQualifiedName%2A?displayProperty=nameWithType> and [Assembly Names](../../../docs/framework/app-domains/assembly-names.md).</span></span>  
  
    ```xml  
    <add name="MyCustomRoleProvider" type="MyNamespace.MyRoleProvider, MyAssembly" />  
    ```  
  
6.  <span data-ttu-id="6c562-210">`defaultProvider` 또는 `<membership>` 요소의 `<roleManager>` 특성을 수정하여 이전 단계에서 추가한 `<add>` 요소에서 이름 값을 지정합니다.</span><span class="sxs-lookup"><span data-stu-id="6c562-210">Modify the `defaultProvider` attribute of the `<membership>` or `<roleManager>` element to specify the name value from the `<add>` element that you added in the previous step.</span></span>  
  
    ```xml  
    <roleManager enabled="true" defaultProvider="MyCustomRoleProvider">  
    ```
