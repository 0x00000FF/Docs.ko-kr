 [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)]<span data-ttu-id="0ece5-109">값을 사용 하는 서비스는 <xref:System.ServiceModel.Channels.LocalServiceSecuritySettings.MaxClockSkew%2A> 속성에는 [SecurityBindingElement](../../../../docs/framework/wcf/diagnostics/wmi/securitybindingelement.md) 클록 클라이언트와 서비스 간의 시간차를 허용 하도록 합니다.</span><span class="sxs-lookup"><span data-stu-id="0ece5-109"> services use the value of the <xref:System.ServiceModel.Channels.LocalServiceSecuritySettings.MaxClockSkew%2A> property on the [SecurityBindingElement](../../../../docs/framework/wcf/diagnostics/wmi/securitybindingelement.md) to allow for clock skew between the client and service.</span></span> <span data-ttu-id="0ece5-110">페더레이션에서 `MaxClockSkew` 설정은 클라이언트가 발급된 토큰을 가져온 클라이언트 및 보안 토큰 서비스 사이에 모두 클럭 오차를 적용합니다.</span><span class="sxs-lookup"><span data-stu-id="0ece5-110">In federation, the `MaxClockSkew` setting applies to clock skews between both the client and the security token service from where the client obtained the issued token.</span></span> <span data-ttu-id="0ece5-111">따라서 보안 토큰 서비스는 발급된 토큰의 유효 시간 및 만료 시간을 설정할 때 클럭 오차를 허용할 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="0ece5-111">Therefore, security token services need not make clock-skew allowances when setting the issued token's effective and expiration times.</span></span>
