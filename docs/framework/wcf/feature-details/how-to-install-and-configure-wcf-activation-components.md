### <a name="to-configure-the-was-to-support-tcp-activation"></a><span data-ttu-id="5aa23-115">TCP 활성화를 지원하도록 WAS를 구성하려면</span><span class="sxs-lookup"><span data-stu-id="5aa23-115">To configure the WAS to support TCP activation</span></span>  
  
1.  <span data-ttu-id="5aa23-116">net.tcp 활성화를 지원하려면 먼저 기본 웹 사이트를 net.tcp 포트에 바인딩해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-116">To support net.tcp activation, the default Web site must first be bound to a net.tcp port.</span></span> <span data-ttu-id="5aa23-117">이 작업은 [!INCLUDE[iisver](../../../../includes/iisver-md.md)] 관리 도구 집합과 함께 설치되는 Appcmd.exe를 사용하여 수행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-117">You can do this by using Appcmd.exe, which is installed with the [!INCLUDE[iisver](../../../../includes/iisver-md.md)] management toolset.</span></span> <span data-ttu-id="5aa23-118">관리자 수준 명령 프롬프트 창에서 다음 명령을 실행합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-118">In an administrator-level Command Prompt window, run the following command.</span></span>  
  
    ```  
    %windir%\system32\inetsrv\appcmd.exe set site "Default Web Site" -+bindings.[protocol='net.tcp',bindingInformation='808:*']  
    ```  
  
    > [!NOTE]
    >  <span data-ttu-id="5aa23-119">이 명령은 줄 바꿈 없이 한 줄로 입력해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-119">This command is a single line of text.</span></span> <span data-ttu-id="5aa23-120">이 명령은 임의의 호스트 이름을 사용하여 TCP 포트 808에서 수신 대기하는 기본 웹 사이트에 net.tcp 사이트 바인딩을 추가합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-120">This command adds a net.tcp site binding to the default Web site listening on TCP port 808 with any host name.</span></span>  
  
2.  <span data-ttu-id="5aa23-121">사이트 내의 모든 응용 프로그램이 공통된 net.tcp 바인딩을 공유하지만 각 응용 프로그램에서 개별적으로 net.tcp 지원을 사용하도록 설정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-121">Although all applications within a site share a common net.tcp binding, each application can enable net.tcp support individually.</span></span> <span data-ttu-id="5aa23-122">응용 프로그램에 대해 net.tcp를 사용하도록 설정하려면 관리자 수준 명령 프롬프트에서 다음 명령을 실행합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-122">To enable net.tcp for the application, run the following command from an administrator-level command prompt.</span></span>

    ```  
    "Default Web Site/<WCF Application>" /enabledProtocols:http,net.tcp  
    ```  
  
    > [!NOTE]
    >  <span data-ttu-id="5aa23-123">이 명령은 줄 바꿈 없이 한 줄로 입력해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-123">This command is a single line of text.</span></span> <span data-ttu-id="5aa23-124">이 명령을 사용 하 여 /\<*WCF 응용 프로그램*> 응용 프로그램에 두 http://localhost를 사용 하 여 액세스할 수*/\<WCF 응용 프로그램 >* 및 net.tcp:// localhost /*\<WCF 응용 프로그램 >*합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-124">This command enables the /\<*WCF Application*> application to be accessed using both http://localhost*/\<WCF Application>* and net.tcp://localhost/*\<WCF Application>*.</span></span>  
  
     <span data-ttu-id="5aa23-125">이 샘플에 대해 추가한 net.tcp 사이트 바인딩을 제거합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-125">Remove the net.tcp site binding you added for this sample.</span></span>  
  
     <span data-ttu-id="5aa23-126">편의를 위해 다음 두 단계는 샘플 디렉터리에 있는 RemoveNetTcpSiteBinding.cmd라는 배치 파일에서 구현됩니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-126">As a convenience, the following two steps are implemented in a batch file called RemoveNetTcpSiteBinding.cmd located in the sample directory.</span></span>

        > [!NOTE]
        >  <span data-ttu-id="5aa23-128">이 명령은 줄 바꿈 없이 한 줄로 입력해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-128">This command is a single line of text.</span></span>  
  
    2.  <span data-ttu-id="5aa23-129">고급 명령 프롬프트 창에서 다음 명령을 실행하여 net.tcp 사이트 바인딩을 제거합니다.</span><span class="sxs-lookup"><span data-stu-id="5aa23-129">Remove the net.tcp site binding by running the following command in an elevated Command Prompt window:</span></span>
