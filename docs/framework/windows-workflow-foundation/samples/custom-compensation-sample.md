## <a name="sample-details"></a><span data-ttu-id="86678-105">샘플 세부 정보</span><span class="sxs-lookup"><span data-stu-id="86678-105">Sample Details</span></span>  
 <span data-ttu-id="86678-106">시뮬레이션하는 단계는 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="86678-106">The steps simulated are:</span></span>  
  
1.  <span data-ttu-id="86678-107">사용자가 지정된 날짜에 대한 트럭 렌탈 가격 견적을 요청합니다.</span><span class="sxs-lookup"><span data-stu-id="86678-107">The user requests truck rental quotes for a given date.</span></span>  
  
2.  <span data-ttu-id="86678-108">세 개의 트럭 회사에 연락하여 세 개의 견적 가격을 받습니다.</span><span class="sxs-lookup"><span data-stu-id="86678-108">Three truck companies are contacted and the three quotes are provided.</span></span>  
  
3.  <span data-ttu-id="86678-109">사용자는 하나의 트럭 견적 가격을 선택하고 신용 카드로 주문합니다.</span><span class="sxs-lookup"><span data-stu-id="86678-109">The user selects one truck quote and proceeds to order by credit card.</span></span>  
  
4.  <span data-ttu-id="86678-110">응용 프로그램은 나머지 두 개의 트럭 가격 견적을 취소합니다.</span><span class="sxs-lookup"><span data-stu-id="86678-110">The application cancels the other two truck quotes.</span></span>  
  
5.  <span data-ttu-id="86678-111">응용 프로그램은 예약 날짜 10일 전까지 취소할 경우 프리미엄 계좌가 아닌 일반 계좌의 경우 환불되지 않는 서비스 요금을 청구합니다.</span><span class="sxs-lookup"><span data-stu-id="86678-111">The application charges a service fee that is non-refundable for non-premium accounts if cancelation happens 10 days or less prior to the reservation date.</span></span>  
  
6.  <span data-ttu-id="86678-112">응용 프로그램은 트럭 렌탈 요금을 청구합니다.</span><span class="sxs-lookup"><span data-stu-id="86678-112">The application charges the truck rental fee.</span></span>  
  
7.  <span data-ttu-id="86678-113">응용 프로그램은 예약 날짜와 고객이 예약을 취소하기로 결정한 날짜 중 빠른 날짜까지 기다립니다.</span><span class="sxs-lookup"><span data-stu-id="86678-113">The application waits until the reservation date or until the customer decided to cancel the reservation, whichever comes first.</span></span>  
  
8.  <span data-ttu-id="86678-114">고객이 예약을 취소하는 경우 <xref:System.Activities.Statements.CompensableActivity.CompensationHandler%2A> 사용자 지정 보정 논리는 다음 논리에 따라 실행됩니다.</span><span class="sxs-lookup"><span data-stu-id="86678-114">If the customer cancels the reservation, the <xref:System.Activities.Statements.CompensableActivity.CompensationHandler%2A> custom compensation logic runs according to the following logic:</span></span>  
  
    1.  <span data-ttu-id="86678-115">고객의 계좌가 프리미엄 계좌가 아니고 현재 시점부터 예약 날짜까지 남은 날짜가 10일 미만인 경우 서비스 요금이 청구되며, 이외의 경우에는 응용 프로그램이 서비스 요금을 환불합니다.</span><span class="sxs-lookup"><span data-stu-id="86678-115">If the customer has a non-premium account and it is less than 10 days prior to the reservation date, then the service fee is still charged; otherwise, the application refunds the service fee.</span></span>  
  
    2.  <span data-ttu-id="86678-116">나머지 보정 가능한 활동(트럭 주문 + 트럭 주문 요금)은 실행 순서의 반대로 보정하는 기본 보정 논리에 따라 실행됩니다.</span><span class="sxs-lookup"><span data-stu-id="86678-116">The rest of the compensable activities (truck order + truck order fee) are run according to the default compensation logic, which is to compensate in reverse order of execution.</span></span>
