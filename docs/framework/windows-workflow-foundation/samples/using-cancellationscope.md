 <span data-ttu-id="05d4a-108">취소는 활동 내에서 또는 활동의 부모에서 트리거할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05d4a-108">Cancellation can be triggered either from within an activity or from the activity's parent.</span></span>  <span data-ttu-id="05d4a-109">자식 활동은 <xref:System.Activities.Statements.Sequence>, <xref:System.Activities.Statements.Parallel>, <xref:System.Activities.Statements.Flowchart> 또는 사용자 지정 복합 활동과 같은 부모 활동에 의해 예약됩니다.</span><span class="sxs-lookup"><span data-stu-id="05d4a-109">Child activities are scheduled by their parent activity (such as a <xref:System.Activities.Statements.Sequence>, <xref:System.Activities.Statements.Parallel>, <xref:System.Activities.Statements.Flowchart>, or a custom composite activity).</span></span>  <span data-ttu-id="05d4a-110">이유에 관계없이 부모 활동은 자식 활동을 취소할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05d4a-110">The parent activity can cancel child activities for any reason.</span></span>  <span data-ttu-id="05d4a-111">예를 들어 세 개의 자식 분기가 있는 <xref:System.Activities.Statements.Parallel> 활동은 분기를 완료하고 <xref:System.Activities.Statements.Parallel.CompletionCondition%2A>식이 `true`로 평가될 때마다 나머지 자식 분기를 취소합니다.</span><span class="sxs-lookup"><span data-stu-id="05d4a-111">For example, a <xref:System.Activities.Statements.Parallel> activity with three child branches will cancel the remaining child branches whenever it completes a branch and the <xref:System.Activities.Statements.Parallel.CompletionCondition%2A> expression evaluates to `true`.</span></span> <span data-ttu-id="05d4a-112"><xref:System.Activities.WorkflowApplication.Cancel%2A>을 호출하여 호스트 응용 프로그램에서 워크플로를 외부적으로 취소할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05d4a-112">The workflow can also be canceled externally by the host application by calling <xref:System.Activities.WorkflowApplication.Cancel%2A>.</span></span>

 <span data-ttu-id="05d4a-114">이 샘플에서는 활동 자체 내에서 활동을 취소하는 방법을 보여 줍니다.</span><span class="sxs-lookup"><span data-stu-id="05d4a-114">This sample demonstrates cancelling an activity from within the activity itself.</span></span>

since the <xref:System.Activities.Statements.Parallel.CompletionCondition%2A> is checked after any branch completes, this will cause the remaining branch to be canceled after the first branch completes.</span></span> <span data-ttu-id="05d4a-119">클라이언트 응용 프로그램은 두 여행사에 비행기표 구입을 문의한 후 둘 중 한 여행사에서 비행기표를 구입했음을 확인하면 다른 여행사에서의 주문을 취소합니다.</span><span class="sxs-lookup"><span data-stu-id="05d4a-119">The client application asks both agencies to buy the ticket, and when the first one confirms that the ticket has been bought, the application cancels the order at the other agency.</span></span>
