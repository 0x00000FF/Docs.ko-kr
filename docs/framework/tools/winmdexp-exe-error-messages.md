# Winmdexp.exe 오류 메시지
**[!INCLUDE[wrt](../../../includes/wrt-md.md)] Component** template in [!INCLUDE[vs_dev11_long](../../../includes/vs-dev11-long-md.md)] 를 사용 하는 경우 빌드 프로세스는 [Winmdexp.exe\(Windows Runtime 메타데이터 내보내기 도구\)](../../../docs/framework/tools/winmdexp-exe-windows-runtime-metadata-export-tool.md)를 호출합니다. 그래서 Winmdexp.exe 에러 메시지가 **Error List**에 표시됩니다.  Winmdexp.exe은 `/target:winmdobj` 옵션으로 컴파일 되는 모듈에서 작동합니다.  컴파일된 모듈을 입력으로 하므로 컴파일에 성공 하지 않으면 오류 메시지가 나타나지 않습니다.  
  
 오류 메시지는 보고 된 오류 조건을 해결 하기 위해 필요한 모든 정보를 포함 하도록 설계 되었습니다.  그러나 몇 가지 문제가 메시지에 들어가는 것 보다 더 많은 정보가 필요 합니다.  추가 정보는 Windows 개발자 센터의 [Diagnosing Windows Runtime component error conditions](http://go.microsoft.com/fwlink/p/?LinkId=251127) 에서 찾을 수 있습니다.  
  
 문서에서 에러가 설명되지 않은 경우, 메세지가 문제를 해결 하는 데 충분한 정보를 포함 하지 않는 다고 생각하면, 해당 문서의 피드백 링크를 사용하여 에러 메세지를 보내십시오.  대안으로서, [Microsoft Connect website](http://go.microsoft.com/fwlink/p/?LinkId=251130)에 있는 버그를 정리할 수 있습니다.  자세한 내용은 또한 [Microsoft Forums](http://go.microsoft.com/fwlink/p/?LinkId=251129)에서도 찾을 수 있습니다.
