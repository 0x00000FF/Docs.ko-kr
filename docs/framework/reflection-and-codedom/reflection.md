-   <span data-ttu-id="3c315-124"><xref:System.Reflection.CustomAttributeData>를 사용하여 응용 프로그램 도메인의 리플렉션 전용 컨텍스트에서 작업할 때 사용자 지정 특성에 대한 정보를 검색합니다.</span><span class="sxs-lookup"><span data-stu-id="3c315-124">Use <xref:System.Reflection.CustomAttributeData> to discover information about custom attributes when you are working in the reflection-only context of an application domain.</span></span> <span data-ttu-id="3c315-125"><xref:System.Reflection.CustomAttributeData>를 사용하면 특성 인스턴스를 만들지 않고 특성을 검사할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3c315-125"><xref:System.Reflection.CustomAttributeData> allows you to examine attributes without creating instances of them.</span></span>

 <span data-ttu-id="3c315-133">리플렉션은 형식, 멤버, 매개 변수, 기타 코드 엔터티를 나타내는 <xref:System.Type> 및 <xref:System.Reflection.MethodInfo>와 같은 클래스를 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="3c315-133">Reflection provides classes, such as <xref:System.Type> and <xref:System.Reflection.MethodInfo>, to represent types, members, parameters, and other code entities.</span></span> <span data-ttu-id="3c315-134">하지만 리플렉션을 사용할 때 이들 클래스를 직접 사용하지 않습니다. 이들 클래스는 대부분 추상 클래스입니다(Visual Basic의 `MustInherit`).</span><span class="sxs-lookup"><span data-stu-id="3c315-134">However, when you use reflection you don't work directly with these classes, most of which are abstract (`MustInherit` in Visual Basic).</span></span> <span data-ttu-id="3c315-135">대신에 CLR(공용 언어 런타임)에서 제공된 형식을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="3c315-135">Instead, you work with types provided by the common language runtime (CLR).</span></span>
