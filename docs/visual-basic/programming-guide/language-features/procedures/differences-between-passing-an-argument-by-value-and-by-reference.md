 <span data-ttu-id="0a3c1-111">인수를 전달 하면 *참조별* 지정 하 여 합니다 [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) 프로시저 정의에서 해당 매개 변수에 키워드.</span><span class="sxs-lookup"><span data-stu-id="0a3c1-111">You pass an argument *by reference* by specifying the [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="0a3c1-112">이 전달 메커니즘을 사용 하면 Visual Basic에서는 호출 코드에서 기본 프로그래밍 요소에 대 한 직접 참조가 프로시저에 제공 합니다.</span><span class="sxs-lookup"><span data-stu-id="0a3c1-112">When you use this passing mechanism, Visual Basic gives the procedure a direct reference to the underlying programming element in the calling code.</span></span>

 <span data-ttu-id="0a3c1-114">전달 메커니즘의 선택은 기본 요소 형식의 분류와 동일 합니다.</span><span class="sxs-lookup"><span data-stu-id="0a3c1-114">The choice of passing mechanism is not the same as the classification of the underlying element type.</span></span> <span data-ttu-id="0a3c1-115">값별 또는 참조별 전달 프로시저 코드를 Visual Basic 제공 하는 새로운 참조입니다.</span><span class="sxs-lookup"><span data-stu-id="0a3c1-115">Passing by value or by reference refers to what Visual Basic supplies to the procedure code.</span></span> <span data-ttu-id="0a3c1-116">값 형식 또는 참조 형식 프로그래밍 요소는 메모리에 저장 하는 방법을 가리킵니다.</span><span class="sxs-lookup"><span data-stu-id="0a3c1-116">A value type or reference type refers to how a programming element is stored in memory.</span></span>  
  
 <span data-ttu-id="0a3c1-117">그러나 전달 메커니즘 및 요소 형식이 사용 하는 서로 관련 되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0a3c1-117">However, the passing mechanism and element type are interrelated.</span></span> <span data-ttu-id="0a3c1-118">참조 형식의 값은 메모리의 다른 곳에서 데이터에 대 한 포인터입니다.</span><span class="sxs-lookup"><span data-stu-id="0a3c1-118">The value of a reference type is a pointer to the data elsewhere in memory.</span></span> <span data-ttu-id="0a3c1-119">따라서 참조 형식을 값을 전달 하면 프로시저 코드에 기본 요소의 데이터에 대 한 포인터도 내부 요소에 액세스할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="0a3c1-119">This means that when you pass a reference type by value, the procedure code has a pointer to the underlying element's data, even though it cannot access the underlying element itself.</span></span> <span data-ttu-id="0a3c1-120">예를 들어, 요소 배열 변수 이면 프로시저 코드 자체는 변수에 액세스할 수 없지만 배열 멤버에 액세스할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0a3c1-120">For example, if the element is an array variable, the procedure code does not have access to the variable itself, but it can access the array members.</span></span>
