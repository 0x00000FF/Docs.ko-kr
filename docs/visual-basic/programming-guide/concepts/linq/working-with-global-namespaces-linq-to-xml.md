## <a name="examples-of-global-namespaces"></a><span data-ttu-id="af08d-115">전역 네임스페이스의 예제</span><span class="sxs-lookup"><span data-stu-id="af08d-115">Examples of Global Namespaces</span></span>  
 <span data-ttu-id="af08d-116">다음 예제에서는 `Imports` 문을 사용하여 기본 전역 네임스페이스를 선언한 다음 XML 리터럴을 사용하여 해당 네임스페이스에서 <xref:System.Xml.Linq.XElement> 개체를 초기화합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-116">The following example declares a default global namespace by using the `Imports` statement, and then uses an XML literal to initialize an <xref:System.Xml.Linq.XElement> object in that namespace:</span></span>  
  
```vb

    Sub Main()  
        Dim root As XElement = <Root/>  
        Console.WriteLine(root)  
    End Sub  
End Module  
```  
  
 <span data-ttu-id="af08d-117">이 예제는 다음과 같은 출력을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-117">This example produces the following output:</span></span>  
  
```xml  
<Root xmlns="http://www.adventure-works.com" />  
```  
  
 <span data-ttu-id="af08d-118">다음 예제에서는 접두사가 포함된 전역 네임스페이스를 선언한 다음 XML 리터럴을 사용하여 요소를 초기화합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-118">The following example declares a global namespace with a prefix, and then uses an XML literal to initialize an element:</span></span>  
  
```vb

Module Module1  
    Sub Main()  
        Dim root As XElement = <aw:Root/>  
        Console.WriteLine(root)  
    End Sub  
End Module  
```  
  
 <span data-ttu-id="af08d-119">이 예제는 다음과 같은 출력을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-119">This example produces the following output:</span></span>  
  
```xml  
<aw:Root xmlns:aw="http://www.adventure-works.com" />  
```  
  
## <a name="global-namespaces-and-embedded-expressions"></a><span data-ttu-id="af08d-120">전역 네임스페이스 및 포함 식</span><span class="sxs-lookup"><span data-stu-id="af08d-120">Global Namespaces and Embedded Expressions</span></span>  
 <span data-ttu-id="af08d-121">XML 리터럴에서 선언된 네임스페이스는 포함 식에까지 영향을 미치지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-121">Namespaces that are declared in XML literals do not carry over into embedded expressions.</span></span> <span data-ttu-id="af08d-122">다음 예제에서는 기본 네임스페이스를 선언한 다음</span><span class="sxs-lookup"><span data-stu-id="af08d-122">The following example declares a default namespace.</span></span> <span data-ttu-id="af08d-123">`Child` 요소에 대한 포함 식을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-123">It then uses an embedded expression for the `Child` element.</span></span>  
  
```vb  
Dim root As XElement = _  
    <Root xmlns="http://www.adventure-works.com">  
        <%= <Child/> %>  
    </Root>  
Console.WriteLine(root)  
```  
  
 <span data-ttu-id="af08d-124">이 예제는 다음과 같은 출력을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-124">This example produces the following output:</span></span>  
  
```xml  
<Root xmlns="http://www.adventure-works.com">  
  <Child xmlns="" />  
</Root>  
```  
  
 <span data-ttu-id="af08d-125">보다시피 생성되는 XML에는 기본 네임스페이스의 선언이 포함됩니다. 따라서 `Child` 요소는 네임스페이스에 없습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-125">As you can see, the resulting XML includes a declaration of a default namespace so that the `Child` element is in no namespace.</span></span>  
  
 <span data-ttu-id="af08d-126">포함 식에서 다음과 같이 네임스페이스를 다시 선언할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-126">You could re-declare the namespace in the embedded expression, as follows:</span></span>  
  
```vb  
Dim root As XElement = _  
    <Root xmlns="http://www.adventure-works.com">  
        <%= <Child xmlns="http://www.adventure-works.com"/> %>  
    </Root>  
Console.WriteLine(root)  
```  
  
 <span data-ttu-id="af08d-127">이 예제는 다음과 같은 출력을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-127">This example produces the following output:</span></span>  
  
```xml  
<Root xmlns="http://www.adventure-works.com">  
  <Child xmlns="http://www.adventure-works.com" />  
</Root>  
```  
  
 <span data-ttu-id="af08d-128">그러나 이 방법은 전역 기본 네임스페이스보다 사용하기가 더 불편합니다. 따라서 전역 기본 네임스페이스를 사용하는 것이 더 좋습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-128">However, this is more cumbersome to use than the global default namespace, which is a better approach.</span></span> <span data-ttu-id="af08d-129">전역 기본 네임스페이스를 사용하면 네임스페이스를 선언하지 않고 XML 리터럴을 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-129">With the global default namespace, you can use XML literals without declaring namespaces.</span></span> <span data-ttu-id="af08d-130">생성되는 XML은 전역적으로 선언된 기본 네임스페이스에 있습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-130">The resulting XML will be in the globally-declared default namespace.</span></span>  
  
```vb

Module Module1  
    Sub Main()  
        Dim root As XElement = <Root>  
                                   <%= <Child/> %>  
                               </Root>  
        Console.WriteLine(root)  
    End Sub  
End Module  
```  
  
 <span data-ttu-id="af08d-131">이 예제는 다음과 같은 출력을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-131">This example produces the following output:</span></span>  
  
```xml  
<Root xmlns="http://www.adventure-works.com">  
  <Child />  
</Root>  
```  
  
## <a name="using-namespaces-with-xml-properties"></a><span data-ttu-id="af08d-132">XML 속성과 함께 네임스페이스 사용</span><span class="sxs-lookup"><span data-stu-id="af08d-132">Using Namespaces with XML Properties</span></span>  
 <span data-ttu-id="af08d-133">네임스페이스에 있는 XML 트리로 작업하는 경우 XML 속성을 사용하고 있으면 XML 속성도 올바른 네임스페이스에 있도록 전역 네임스페이스를 사용해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-133">If you are working with an XML tree that is in a namespace, and you use XML properties, then you must use a global namespace so that the XML properties will also be in the correct namespace.</span></span> <span data-ttu-id="af08d-134">다음 예제에서는 네임스페이스에 XML 트리를 선언한 다음</span><span class="sxs-lookup"><span data-stu-id="af08d-134">The following example declares an XML tree in a namespace.</span></span> <span data-ttu-id="af08d-135">`Child` 요소의 수를 출력합니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-135">It then prints the count of `Child` elements.</span></span>

 <span data-ttu-id="af08d-136">이 예제는 `Child` 요소가 없음을 나타냅니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-136">This example indicates that there are no `Child` elements.</span></span> <span data-ttu-id="af08d-137">이 예제의 결과는 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-137">It produces the following output:</span></span>

 <span data-ttu-id="af08d-138">그러나 기본 전역 네임스페이스를 선언하는 경우 XML 리터럴과 XML 속성이 모두 기본 전역 네임스페이스에 있습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-138">If, however, you declare a default global namespace, then both the XML literal and the XML property are in the default global namespace:</span></span>

 <span data-ttu-id="af08d-139">이 예제는 `Child` 요소가 하나 있음을 나타냅니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-139">This example indicates that there is one `Child` element.</span></span> <span data-ttu-id="af08d-140">이 예제의 결과는 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-140">It produces the following output:</span></span>  
  
```  
1
```  

 <span data-ttu-id="af08d-141">접두사가 포함된 전역 네임스페이스를 선언하면 XML 리터럴과 XML 속성 모두에 접두사를 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="af08d-141">If you declare a global namespace that has a prefix, you can use the prefix for both XML literals and XML properties:</span></span>
