## <a name="manners-of-execution"></a><span data-ttu-id="31c20-108">실행 방식</span><span class="sxs-lookup"><span data-stu-id="31c20-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="31c20-109">직접 실행</span><span class="sxs-lookup"><span data-stu-id="31c20-109">Immediate</span></span>  
 <span data-ttu-id="31c20-110">즉시 실행 데이터 소스는 읽기 작업을 수행할 때 코드의 지점에 있는 쿼리가 선언 된을 의미 합니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="31c20-111">단일, 열거할 수 없는 결과 반환 하는 모든 표준 쿼리 연산자는 즉시 실행 됩니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="31c20-112">연기됨</span><span class="sxs-lookup"><span data-stu-id="31c20-112">Deferred</span></span>  
 <span data-ttu-id="31c20-113">지연 된 실행 작업은 수행 하지 않음을 코드의 지점에 있는 쿼리가 선언 된 것을 의미 합니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="31c20-114">예를 들어을 사용 하 여 쿼리 변수 열거 되는 경우에 작업이 수행 되는 `For Each` 문입니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="31c20-115">이 의미 하는 쿼리 실행의 결과에 종속 데이터 원본의 내용을 경우 쿼리가 정의 하는 대신 쿼리를 실행할 때 합니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="31c20-116">쿼리 변수를 여러 번 열거할 경우 그 때마다 결과가 다를 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="31c20-117">반환 형식이 거의 모든 표준 쿼리 연산자 <xref:System.Collections.Generic.IEnumerable%601>또는 <xref:System.Linq.IOrderedEnumerable%601>지연 된 방식에서으로 실행 합니다.</xref:System.Linq.IOrderedEnumerable%601> </xref:System.Collections.Generic.IEnumerable%601></span><span class="sxs-lookup"><span data-stu-id="31c20-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="31c20-118">지연 된 실행을 사용 하는 쿼리 연산자 스트리밍 또는 비 스트리밍으로 분류할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="31c20-119">스트리밍</span><span class="sxs-lookup"><span data-stu-id="31c20-119">Streaming</span></span>  
 <span data-ttu-id="31c20-120">스트리밍 연산자는 요소를 생성 하기 전에 모든 원본 데이터를 읽이 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="31c20-121">실행 시 스트리밍 연산자는 읽기 및 적절 한 경우 요소를 생성 각 소스 요소에 대해 연산을 수행 합니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="31c20-122">스트리밍 연산자에는 소스 요소가 때까지 계속 읽습니다 결과 요소를 생성할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="31c20-123">즉, 한 결과 요소를 만드는 둘 이상의 소스 요소를 읽을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="31c20-124">비 스트리밍</span><span class="sxs-lookup"><span data-stu-id="31c20-124">Non-Streaming</span></span>  
 <span data-ttu-id="31c20-125">비 스트리밍 연산자 결과 요소를 생성 하기 전에 모든 원본 데이터를 읽어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="31c20-126">정렬 또는 그룹화 하는 등의 작업은이 범주에 속합니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="31c20-127">실행 시 비 스트리밍 쿼리 연산자 모든 원본 데이터를 읽고, 데이터 구조에 넣습니다, 그리고 작업을 수행 및 결과 요소를 생성 합니다.</span><span class="sxs-lookup"><span data-stu-id="31c20-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
