# <a name="anonymous-type-property-39ltpropertynamegt39-cannot-be-used-in-the-definition-of-a-lambda-expression-within-the-same-initialization-list"></a><span data-ttu-id="17548-102">익명 형식 속성 &#39; &lt;propertyname&gt; &#39; 동일한 초기화 목록 내의 람다 식 정의에 사용할 수 없습니다</span><span class="sxs-lookup"><span data-stu-id="17548-102">Anonymous type property &#39;&lt;propertyname&gt;&#39; cannot be used in the definition of a lambda expression within the same initialization list</span></span>
<span data-ttu-id="17548-103">무명 형식의 초기화 목록에 정의된 속성은 동일한 목록의 람다 식 정의에 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="17548-103">Properties defined in the initialization list of an anonymous type cannot be part of a lambda expression definition in the same list.</span></span> <span data-ttu-id="17548-104">예를 들어 다음 코드에서 `Num` 속성은 `LambdaFun`의 정의에 포함될 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="17548-104">For example, in the following code, property `Num` cannot be included in the definition of `LambdaFun`.</span></span>  
  
```vb  
' Not valid.  
'Dim anon = New With {.Num = 4, .LambdaFun = Function() .Num > 0}  
```  
  
 <span data-ttu-id="17548-105">**오류 ID:** BC36549</span><span class="sxs-lookup"><span data-stu-id="17548-105">**Error ID:** BC36549</span></span>  

## <a name="to-correct-this-error"></a><span data-ttu-id="17548-106">이 오류를 해결하려면</span><span class="sxs-lookup"><span data-stu-id="17548-106">To correct this error</span></span>  
  
1.  <span data-ttu-id="17548-107">무명 형식을 두 부분으로 분할하는 것이 좋습니다.</span><span class="sxs-lookup"><span data-stu-id="17548-107">Consider splitting the anonymous type into two parts:</span></span>  
  
```vb  
Dim anon1 = New With {.Num = 4}  
Dim anon2 = New With {.LambdaFun = Function() anon1.Num > 0}  
' - or -  
Dim anon3 = New With {.lambdaFun = Function(n As Integer) n > 0}  
Console.WriteLine((anon2.LambdaFun)())  
Console.WriteLine(anon3.lambdaFun(anon1.Num))  
anon1.Num = -5  
Console.WriteLine((anon2.LambdaFun)())  
Console.WriteLine(anon3.lambdaFun(anon1.Num))  
```  
  
<span data-ttu-id="17548-108">`anon1.Num` 을 `Key` 속성으로 선언하면 해당 값을 변경할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="17548-108">Note that if you declare `anon1.Num` as a `Key` property, its value cannot be changed.</span></span>  
  
2.  <span data-ttu-id="17548-109">대신, 일반 함수 문을 사용하여 무명 형식 속성에 액세스해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="17548-109">An alternative is to use a regular function statement to access the anonymous type property:</span></span>  
  
```vb  
Function testNum(ByVal n As Integer) As Boolean  
    Return n > 0  
End Function  
Console.WriteLine(testNum(anon1.Num))  
```  
  
3.  <span data-ttu-id="17548-110">마찬가지로, 무명 형식 외부에 정의된 람다 함수를 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="17548-110">Similarly, you can use a lambda function that is defined outside the anonymous type:</span></span>  
  
```vb  
Dim lambdaFun1 = Function() anon1.Num > 0  
Dim lambdaFun2 = Function(n As Integer) n > 0  
```
