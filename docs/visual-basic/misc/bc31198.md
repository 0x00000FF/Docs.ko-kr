<span data-ttu-id="dc1c3-103">XML 리터럴 선언이 [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] 컴파일러가 인식할 수 없는 식의 위치에 사용되었습니다.</span><span class="sxs-lookup"><span data-stu-id="dc1c3-103">An XML literal declaration is used in an expression in a location that is ambiguous to the [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] compiler.</span></span> <span data-ttu-id="dc1c3-104">즉, [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] 컴파일러에서 보다 작음(<) 문자가 비교 연산자로 사용되었는지 또는 XML 리터럴의 시작 부분으로 사용되었는지 확인할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="dc1c3-104">That is, the [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] compiler cannot determine whether the less-than character (<) is intended as a comparative operator or the start of an XML literal.</span></span> <span data-ttu-id="dc1c3-105">다음 코드는 예제를 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="dc1c3-105">The following code provides an example:</span></span>

```vb  
                  Select element  
```  
  
 <span data-ttu-id="dc1c3-106">**오류 ID:** BC31198</span><span class="sxs-lookup"><span data-stu-id="dc1c3-106">**Error ID:** BC31198</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="dc1c3-107">이 오류를 해결하려면</span><span class="sxs-lookup"><span data-stu-id="dc1c3-107">To correct this error</span></span>  
  
-   <span data-ttu-id="dc1c3-108">다음 예제와 같이 XML 리터럴 선언을 괄호로 묶습니다.</span><span class="sxs-lookup"><span data-stu-id="dc1c3-108">Enclose the XML literal declaration in parentheses, as shown in the following example:</span></span>

-   <span data-ttu-id="dc1c3-109">다음 예제와 같이 기존 XML 리터럴을 가리키도록 코드를 수정합니다.</span><span class="sxs-lookup"><span data-stu-id="dc1c3-109">Modify your code to refer to an existing XML literal, as shown in the following example:</span></span>
