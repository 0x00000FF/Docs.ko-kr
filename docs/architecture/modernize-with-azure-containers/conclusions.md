# <a name="conclusions"></a>결론

- 컨테이너 기반 솔루션은 궁극적으로 비용 절감 혜택을 제공합니다. 컨테이너는 프로덕션 환경에서 종속성이 없는 경우 발생하는 충돌을 제거하기 때문에 배포 문제에 대한 솔루션입니다. 이러한 문제를 제거하여 개발/테스트, DevOps 및 프로덕션 작업을 크게 향상시킬 수 있습니다.

- Docker 컨테이너는 서버 기반 애플리케이션 또는 서비스 배포의 표준 단위가 되고 있습니다.

- 프로덕션 환경에서는 오케스트레이터(예: Kubernetes)를 사용하여 확장 가능한 컨테이너 기반 애플리케이션을 호스트해야 합니다.

- 컨테이너를 호스트하는 Azure VM은 클라우드에서 소규모 개발/테스트 환경을 만드는 빠르고 간단한 방법입니다.

- 기존 애플리케이션에서 Azure로 관계형 데이터베이스를 마이그레이션하는 경우 기본적으로 Azure SQL Database Managed Instance를 권장합니다.

- Visual Studio 2017 및 Image2Docker는 초보자를 위한 학습 곡선을 가속화하여 기존 .NET 애플리케이션을 Windows 컨테이너로 현대화하기 시작할 수 있는 기본 도구입니다.

- 컨테이너화된 애플리케이션을 프로덕션 환경에 배치하는 경우 항상 Azure DevOps Services 또는 Jenkins와 같은 CI/CD 파이프라인에 대한 DevOps 문화 및 DevOps 도구를 만들거나 채택합니다.

- Microsoft Azure는 Windows 컨테이너, 클라우드 인프라 및 PaaS 서비스를 사용하여 기존 .NET Framework 애플리케이션을 현대화하는 가장 포괄적이고 완전한 환경을 제공합니다.
