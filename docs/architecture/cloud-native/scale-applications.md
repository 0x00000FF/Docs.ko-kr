# <a name="scaling-cloud-native-applications"></a>클라우드 네이티브 응용 프로그램 크기 조정

[!INCLUDE [book-preview](../../../includes/book-preview.md)]

클라우드 호스팅 환경으로 전환 하는 경우 가장 자주 발생 하는 touted 이점 중 하나는 확장성입니다. 확장성 또는 응용 프로그램에서 각 사용자에 대 한 성능을 지나치게 저하 시 키 지 않고 추가 사용자 부하를 허용 하는 기능은 응용 프로그램을 각각 필요한 리소스를 제공 하는 작은 부분으로 분할 하 여 달성할 수 있습니다. 이 장에서는 클라우드 네이티브 응용 프로그램을 사용자 수요에 맞게 확장할 수 있는 기술을 소개 합니다. 이러한 기술은 다음과 같습니다.

- 컨테이너
- Orchestrator
- 서버를 사용 하지 않는 컴퓨팅
