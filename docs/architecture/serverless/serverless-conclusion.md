<span data-ttu-id="dbd9e-106">**서버를 사용 하지 않는 이점.**</span><span class="sxs-lookup"><span data-stu-id="dbd9e-106">**Benefits of using serverless.**</span></span> <span data-ttu-id="dbd9e-107">서버를 사용 하지 않는 솔루션은 사용이 종 량 제 모델에서 구현 되기 때문에 비용 절감의 중요 한 이점을 제공 합니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-107">Serverless solutions provide the important benefit of cost savings because serverless is implemented in a pay-per-use model.</span></span> <span data-ttu-id="dbd9e-108">서버를 사용 하지 않으면 응용 프로그램의 개별 구성 요소를 독립적으로 확장, 테스트 및 배포할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-108">Serverless makes it possible to independently scale, test, and deploy individual components of your application.</span></span> <span data-ttu-id="dbd9e-109">서버 리스 서버는 마이크로 서비스 아키텍처를 구현 하 고 DevOps 파이프라인에 완전히 통합 하는 데 고유 합니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-109">Serverless is uniquely suited to implement microservices architectures and integrates fully into a DevOps pipeline.</span></span>

<span data-ttu-id="dbd9e-110">**배포 단위로 코딩 합니다.**</span><span class="sxs-lookup"><span data-stu-id="dbd9e-110">**Code as a unit of deployment.**</span></span> <span data-ttu-id="dbd9e-111">서버 리스 서버를 응용 프로그램에서 제거 합니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-111">Serverless abstracts the hardware, OS, and runtime away from the application.</span></span> <span data-ttu-id="dbd9e-112">서버를 사용 하지 않는 경우 코드의 비즈니스 논리를 배포 단위로 집중할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-112">Serverless enables focusing on business logic in code as the unit of deployment.</span></span>

<span data-ttu-id="dbd9e-113">**트리거 및 바인딩.**</span><span class="sxs-lookup"><span data-stu-id="dbd9e-113">**Triggers and bindings.**</span></span> <span data-ttu-id="dbd9e-114">서버 리스 서버는 저장소, Api 및 기타 클라우드 리소스와의 통합을 용이 하 게 합니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-114">Serverless eases integration with storage, APIs, and other cloud resources.</span></span> <span data-ttu-id="dbd9e-115">Azure Functions는 리소스와 상호 작용 하기 위해 코드 및 바인딩을 실행 하는 트리거를 제공 합니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-115">Azure Functions provides triggers to execute code and bindings to interact with resources.</span></span>

<span data-ttu-id="dbd9e-116">**마이크로 서비스.**</span><span class="sxs-lookup"><span data-stu-id="dbd9e-116">**Microservices.**</span></span> <span data-ttu-id="dbd9e-117">마이크로 서비스 아키텍처는 자치 서비스의 형태로 여러 독립 된 하위 시스템을 기반으로 하는 분산 된 크고 크고 복잡 한 중요 업무용 응용 프로그램에 대 한 선호 되는 방법입니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-117">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications that are based on multiple independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="dbd9e-118">마이크로 서비스 기반 아키텍처에서 응용 프로그램은 독립적으로 개발, 테스트, 버전 관리, 배포 및 확장할 수 있는 서비스 컬렉션으로 빌드됩니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-118">In a microservice-based architecture, the application is built as a collection of services that can be developed, tested, versioned, deployed, and scaled independently.</span></span> <span data-ttu-id="dbd9e-119">서버를 사용 하지 않는 아키텍처는 이러한 서비스를 구축 하는 데 적합 한 아키텍처입니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-119">Serverless is an architecture well-suited for building these services.</span></span>

<span data-ttu-id="dbd9e-120">**서버를 사용 하지 않는 플랫폼.**</span><span class="sxs-lookup"><span data-stu-id="dbd9e-120">**Serverless platforms.**</span></span> <span data-ttu-id="dbd9e-121">서버를 사용 하지 않는 경우에는 코드에만 해당 됩니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-121">Serverless isn't just about the code.</span></span> <span data-ttu-id="dbd9e-122">서버를 사용 하지 않는 아키텍처를 지 원하는 플랫폼에는 서버를 사용 하지 않는 워크플로와 오케스트레이션, 서버를 사용 하지 않는 메시징 및 이벤트 서비스, 서버</span><span class="sxs-lookup"><span data-stu-id="dbd9e-122">Platforms that support serverless architectures include serverless workflows and orchestration, serverless messaging and event services, and serverless databases.</span></span>

<span data-ttu-id="dbd9e-123">**서버를 사용 하지 않는 챌린지.**</span><span class="sxs-lookup"><span data-stu-id="dbd9e-123">**Serverless challenges.**</span></span> <span data-ttu-id="dbd9e-124">서버를 사용 하지 않는 경우 조각화 및 독립적인 데이터 모델, 복원 력, 버전 관리 및 서비스 검색과 같은 분산 응용 프로그램 개발과 관련 된 과제가 발생 합니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-124">Serverless introduces challenges related to distributed application development, such as fragmented and independent data models, resiliency, versioning, and service discovery.</span></span> <span data-ttu-id="dbd9e-125">서버를 사용 하지 않는 경우 더 긴밀 한 결합을 활용 하는 장기 실행 프로세스 또는 구성 요소에 적합 하지 않을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-125">Serverless may not be ideally suited to long running processes or components that benefit from tighter coupling.</span></span>

<span data-ttu-id="dbd9e-126">**도구 상자가 아닌 도구로 서 서버 리스**</span><span class="sxs-lookup"><span data-stu-id="dbd9e-126">**Serverless as a tool, not the toolbox.**</span></span> <span data-ttu-id="dbd9e-127">서버 리스 서버는 응용 프로그램 아키텍처에 대 한 단독 솔루션이 아닙니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-127">Serverless is not the exclusive solution to application architecture.</span></span> <span data-ttu-id="dbd9e-128">기존 계층, 모놀리식 백 엔드 및 컨테이너를 포함할 수 있는 하이브리드 응용 프로그램의 일부로 활용할 수 있는 도구입니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-128">It is a tool that can be leveraged as part of a hybrid application that may contain traditional tiers, monolith back ends, and containers.</span></span> <span data-ttu-id="dbd9e-129">서버 리스 서버를 사용 하 여 기존 솔루션을 향상 시킬 수 있으며 응용 프로그램 개발에 대 한 전체 또는 없음 방법이 아닙니다.</span><span class="sxs-lookup"><span data-stu-id="dbd9e-129">Serverless can be used to enhance existing solutions and is not an all-or-nothing approach to application development.</span></span>
