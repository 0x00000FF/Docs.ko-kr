# <a name="troubleshooting-type-providers"></a>형식 공급자 문제 해결

이 항목에 설명 하 고 가장 형식 공급자를 사용할 때 발생할 수 있는 문제에 대 한 잠재적인 솔루션을 제공 합니다.


## <a name="possible-problems-with-type-providers"></a>형식 공급자와 함께 가능한 문제
형식 공급자를 사용할 때 문제가 발생 하면 가장 일반적인 솔루션에 대 한 다음 테이블을 검토할 수 있습니다.



|문제점|권장된 조치|
|-------|-----------------|
|**스키마 변경 내용을**합니다. 데이터 원본 스키마의 안정적이 지는 경우 가장 잘 공급자 작업을 입력 합니다. 데이터 테이블 또는 열을 추가 하거나 해당 스키마에 대 한 다른 변경 형식 공급자는 이러한 변경 내용을 자동으로 인식 하지 않습니다.|정리 또는 프로젝트를 다시 빌드하십시오. 프로젝트를 정리 하려면 선택 **빌드**, **Clean** *ProjectName* 메뉴 모음에서 합니다. 프로젝트를 다시 빌드하려면 선택 **빌드**, **다시 작성** *ProjectName* 메뉴 모음에서 합니다. 이러한 작업 모든 유형 공급자 상태 다시 설정 하 고 공급자 데이터 원본에 다시 연결 되어 업데이트 된 스키마 정보를 가져옵니다.|
|**연결 실패**합니다. URL 또는 연결 문자열이 잘못 되었습니다., 다운 되는 네트워크 또는 데이터 원본 또는 서비스를 사용할 수 없습니다.|웹 서비스 또는 OData 서비스에 대 한 URL이 올바른지 및 서비스를 사용할 수 있는지 여부를 확인 하려면 Internet Explorer에서 URL을 시도할 수 있습니다. 데이터베이스 연결 문자열에 대 한 데이터 연결 도구를 사용할 수 있습니다 **서버 탐색기** 연결 문자열이 올바른지 및 데이터베이스를 사용할 수 있는지 여부를 확인할 수 있습니다. 연결을 복원한 후 다음 정리 하거나 해야 프로젝트를 다시 빌드해야 형식 공급자에서 네트워크에 다시 연결 합니다.|
|**유효 하지 않은 자격 증명**합니다. 데이터 원본 또는 웹 서비스에 대 한 유효 사용 권한이 있어야 합니다.|SQL 연결에 대 한 사용자 이름 및 연결 문자열 또는 구성 파일에 지정 된 암호는 데이터베이스에 대 한 올바른 이어야 합니다. Windows 인증을 사용 하는 경우에 데이터베이스에 액세스할 수 있어야 합니다. 데이터베이스 관리자에 대 한 필요한 사용 권한을 각 데이터베이스와 데이터베이스 내의 각 요소에 액세스 하는 권한을 식별할 수 있습니다.<br /><br />웹 서비스 또는 데이터 서비스에 대 한 적절 한 자격 증명이 있어야 합니다. 대부분의 형식 공급자는 적절 한 사용자 이름 및 액세스 키와 함께 설정할 수 있는 자격 증명 속성을 포함 하는 DataContext 개체를 제공 합니다.|
|**유효 하지 않은 경로**합니다. 파일의 경로가 올바르지 않습니다.|경로가 올바른지와 파일이 있는지 여부를 확인 합니다. 또한 경로에 모든 backlashes를 적절 하 게 견적 하거나 축 자 문자열 또는 삼중 따옴표가 붙은 문자열을 사용 합니다.|
